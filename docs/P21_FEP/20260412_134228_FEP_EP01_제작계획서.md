## Phase 2: 씬 일러스트 생성 (예상 3-6시간)

**의존**: Phase 1 완료
**핵심**: 세이프 존 규칙 — 핵심 요소를 중앙 세로 1/3에 배치 (9:16 크롭 대비)

15씬 일러스트를 ComfyUI API 배치로 생성. 캐릭터 시트를 레퍼런스로 입력.

**나머지**: 씬 2, 3, 4, 6, 9, 10, 11, 13, 15

**리스크**: 2인 이상 등장 씬(2, 3, 5, 7, 10, 11, 12, 14)에서 캐릭터 일관성 유지가 어려울 수 있음 → Phase 1 직후 테스트 씬 1장 생성으로 검증

**Phase 2 완료 조건**: 15장 일러스트 사용자 승인, `scenes/scene_XX_[name].png` 저장

## Phase 3: TTS 내레이션 생성 (Phase 1-2와 병렬 가능)

**의존**: Phase 0 완료 (edge-tts만 필요, 일러스트 불필요)

### 3A. 롱폼 드래프트 (edge-tts)

15씬 스크립트를 `ko-KR-InJoonNeural` 음성으로 생성:
```bash
edge-tts --voice "ko-KR-InJoonNeural" --text "[스크립트]" --write-media scene_XX.mp3
```

### 3B. 쇼츠 드래프트 (edge-tts)

7편 쇼츠 스크립트 별도 녹음 (롱폼에서 자르지 않음)

### 3C. 타이밍 매니페스트

각 음성 파일 길이를 `ffprobe`로 추출 → 클립 조립 시 사용

### 3D. Typecast 최종 음성 (Phase 7에서 교체)

edge-tts 드래프트로 전체 영상 확정 후, Typecast 베이직 플랜(월 8,900원)으로 최종 음성 교체

**Phase 3 완료 조건**: 22개 음성 파일(롱폼 15 + 쇼츠 7) 생성 완료 + 타이밍 매니페스트

## Phase 4: 롱폼 영상 조립 (예상 1-2시간)

**의존**: Phase 2 + Phase 3A 완료

### 4A. 개별 클립 생성 (15개, 병렬 가능)

```bash
ffmpeg -loop 1 -i scene_XX.png -i scene_XX.mp3 \
  -c:v libx264 -tune stillimage -c:a aac \
  -shortest -pix_fmt yuv420p clip_XX.mp4
```

선택: Ken Burns 효과 (CineLink 패턴 재사용)

### 4B. 클립 연결

`concat_list.txt` 작성 → `ffmpeg -f concat` 으로 연결

### 4C. 후처리

1. 씬 전환 트랜지션 (페이드, 0.5초)
2. BGM (YouTube Audio Library, 내레이션 대비 -18dB)
3. 자막 (SRT 또는 drawtext)

**산출물**: `EP01_forced_compliment/output/EP01_longform_final.mp4` (~6분)
