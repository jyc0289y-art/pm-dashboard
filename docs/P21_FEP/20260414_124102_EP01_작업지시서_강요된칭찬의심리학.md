# 작업지시서: "일상속 경험의 심리학" 채널 — EP.01 강요된 칭찬

## 프로젝트 개요

비치키 스타일의 심리학 해설 YouTube 채널 콘텐츠를 제작한다.
AI 일러스트 + TTS 내레이션을 조합한 영상 포맷이다.

### 채널 정보
- **채널명**: 일상속 경험의 심리학
- **슬로건**: 당연하다고 넘긴 순간, 거기에 심리학이 있었습니다.
- **컨셉**: 누구나 겪어봤지만 아무도 분석하지 않은 일상 에피소드를 심리학적으로 해부하는 채널
- **톤**: 장난스럽지만 핵심은 정확하게
- **타겟**: 한국어 화자, 2030세대 (특정 연령을 한정하지 않는다)

### 시리즈 구조
- 남자 주인공을 중심으로 주변인물들(여동생, 직장동료, 친구, 연인 등)과의 에피소드를 매회 심리학적으로 해석한다.
- 채널명 "일상속 경험의 심리학"은 채널 브랜드로, YouTube 영상 아래에 자동 표시된다. 영상 제목에는 채널명을 반복하지 않는다.
- 영상 제목 구조: **[___의 심리학] — [훅/부제]**
  - EP.01: 강요된 칭찬의 심리학 — "나 예쁘지?" 다섯 글자에 감춰진 심리
  - EP.02 예시: 거절할 수 없는 부탁의 심리학 — "시간 있지?" 한마디의 함정
  - EP.03 예시: 습관적 사과의 심리학 — "아니 괜찮아"의 진짜 뜻
- 채널명도 "___의 심리학", 에피소드 제목도 "___의 심리학" 구조라 통일감을 유지한다.

### 기술 스택
- **일러스트 생성**: ComfyUI + FLUX Kontext Dev (로컬, MacBook Apple Silicon)
- **TTS/더빙**: Typecast
- **영상 편집**: CineLink 파이프라인 (FFmpeg 기반) 또는 수동 편집
- **작업 환경**: Claude Code (MacBook)

채널명 "일상속 경험의 심리학"은 영상 아래에 자동 표시되므로 영상 제목에는 포함하지 않는다. 에피소드별로 "___의 심리학" 구조를 유지하여 채널과의 통일감을 만든다.

### 썸네일 디자인

**구도**: 수빈이 자신만만한 표정으로 정면을 보고 있고, 옆에 민준이 질린 표정으로 서 있는 2인 구도.

**핵심 텍스트 배치**:
- 중앙 상단 (큼직하게): **"나 예쁘지?"**
- 하단 또는 캐릭터 주변에 키워드 태그 스타일로 배치:
  - **가스라이팅 ❌** — 가장 인지도 높은 키워드. "이건 가스라이팅이 아닙니다" 뉘앙스로 호기심 유발. 엑스 표시 또는 취소선으로 "가스라이팅이라고 생각했지?" 느낌.
  - **인지부조화** — 대중적으로 많이 알려진 심리학 용어. 듣자마자 뭔가 불편한 느낌을 환기시킴.
  - **나르시시즘** — 가장 대중적 키워드. 사람들이 주변인에게 "혹시 나르시시스트?" 라고 궁금해하는 맥락과 연결됨.

**키워드 선정 근거**:
- 가스라이팅: 검색량 최상위 심리학 키워드. ❌ 표시로 "당신이 아는 그것과 다르다"는 반전 유도 → 클릭 유발
- 나르시시즘: 자기애성 인격에 대한 대중적 관심 높음. "주변의 그 사람"을 떠올리게 함
- 인지부조화: 교양 심리학에서 가장 유명한 개념 중 하나. 학구적 느낌 + 공감 포인트

**제외한 키워드와 이유**:
- 강요된 동의 (Coerced Compliance): 정확한 용어이나 대중 인지도 낮음. 영상 안에서 설명할 때 효과적이지 썸네일에서는 클릭 유발력 부족.

**색감**: 따뜻한 배경 + 텍스트는 굵은 흰색/노란색, 키워드 태그는 빨간색 또는 파란색 강조

## 1. 에피소드 원본

### 제목
강요된 칭찬의 심리학 — "나 예쁘지?" 다섯 글자에 감춰진 심리

### 상황 설정
아는 여동생이 "나 예쁘지?"라고 묻는다. 사실상 답이 정해진 질문이다. 거기서 그치지 않고, 아기에게도 같은 질문을 한다. 대답할 수 없는 존재에게까지 확인을 구하는 상황. 질문을 받는 쪽은 대답해주기 싫다는 감정을 느낀다.

이 일상적인 에피소드 안에 질문하는 사람, 답변하는 사람, 그리고 답변을 거부하는 사람 — 세 가지 심리가 동시에 작동한다.

### 파트 1: 질문하는 사람의 심리

**확인 추구 행동 (Validation-seeking)**
자기 외모에 대한 확신을 내면이 아닌 외부에서 얻으려는 행동이다. 핵심은 "진짜 궁금한 게 아니라는 것"이다. 답이 정해진 질문을 던지는 순간, 이것은 질문이 아니라 요구가 된다.

**강요된 동의 (Coerced Compliance)**
거절할 수 없는 사회적 압력 속에서 특정 답변을 유도하는 구조다. 관계의 맥락상 "아니"라고 말하기 어렵다는 걸 본인도 알고 있기 때문에, 질문의 형식을 빌렸지만 실질적으로는 동의를 강제하는 것이다.

**나르시시즘적 공급 (Narcissistic Supply)**
타인의 칭찬이나 긍정적 반응을 통해 자존감을 유지하려는 패턴이다. 답변의 진정성은 중요하지 않다. "예쁘다"는 말 자체를 듣는 행위가 목적이다.

**아기에게까지 질문하는 의미**
아기는 동의도 거부도 할 수 없다. 이 지점에서 질문의 본질이 드러난다. 진심 어린 평가가 아니라 의식(ritual)에 가깝다. 대답의 내용이 아니라 대답을 받아내는 행위 자체에서 심리적 만족을 얻는 구조다.

### 파트 2: 대답을 거부하는 사람의 심리

**자율성 침해에 대한 저항**
답이 정해진 질문은 상대의 판단권을 빼앗는다. "네 의견을 물어보는 것 같지만 실은 내가 원하는 말을 해라"라는 구조이기 때문에, 자유의지를 가진 사람으로서 본능적으로 불쾌함을 느낀다.

**도구화에 대한 거부감**
진심이 궁금한 것이 아니라 칭찬 자판기로 쓰이는 느낌이다. 심리학에서 말하는 도구적 관계(instrumental relationship)에 대한 거부감으로, "나"라는 사람이 아니라 "나의 기능(칭찬 제공)"만 필요한 상황에 대한 반발이다.

**인지 부조화**
솔직한 답을 하면 관계가 불편해지고, 원하는 답을 해주면 자기 진정성이 훼손된다. 이 두 가지 사이에 끼이는 불편함 자체가 대답을 꺼리게 만든다.

**심리적 경계(Boundary) 방어**
타인의 감정적 필요를 무조건 충족시켜야 한다는 암묵적 압력을 거부하는 것은 건강한 자기보호다. "이건 진짜 질문이 아니잖아"라는 걸 직감적으로 감지하고 있는 것이다.

### 파트 3: 인지부조화를 느끼면서도 답변해주는 사람의 심리

**관계 유지 동기 (Relational Maintenance)**
"이 한마디면 끝나잖아"라는 계산이다. 솔직함의 가치보다 관계의 평화를 우선시하는 선택이다. 단기적으로는 합리적이지만, 반복되면 관계 안에서 자기 목소리를 잃어가는 패턴이 된다.

**습관적 순응 (Habitual Compliance)**
사회심리학에서 말하는 사회적 바람직성 편향이다. 거절했을 때의 사회적 비용이 크다고 무의식적으로 판단한다. 특히 한국 문화에서 관계적 조화를 중시하는 경향과 맞물리면 더 강하게 작동한다.

**자기 합리화 (Self-justification)**
인지부조화를 해소하는 방법은 두 가지다. 행동을 바꾸는 것(거절)과 신념을 바꾸는 것("뭐 실제로 안 예쁜 것도 아니니까..."). 대부분 후자를 택한다. 답변 후에 오히려 불편함이 줄어드는 이유가 여기에 있다. 본인이 스스로를 설득한 것이다.

**학습된 무력감의 경미한 형태**
"어차피 거절해도 또 물어볼 거잖아"라는 경험이 쌓이면 저항 자체를 포기한다. 에너지 절약 차원에서 응하는 것이다.

**종합**
매번 작은 자기 진정성을 지불하고 관계의 평화를 사는 거래를 하고 있다. 이것이 쌓이면 그 관계 자체에 대한 은근한 피로감으로 나타난다.

### 파트 4: 가스라이팅과의 차이

이 상황을 가스라이팅이라고 부르기 쉽지만 정확하지 않다. 가스라이팅은 상대의 현실 인식 자체를 왜곡시키는 행위다("네가 잘못 기억하는 거야", "그런 일 없었어"). 이 에피소드는 현실 인식을 왜곡하는 것이 아니라, 정해진 답을 강요하는 구조다. 확인 추구 행동 + 강요된 동의가 더 정확한 해석이다.

### 핵심 인사이트

하나의 일상적 질문 안에 세 가지 심리적 역학이 동시에 작동한다.

- **질문자**: 외부 확인을 통한 자존감 유지 (칭찬의 소비자)
- **거부자**: 자율성과 진정성을 지키려는 건강한 경계 (칭찬 거래의 거부자)
- **순응자**: 관계 유지를 위해 자기 진정성을 지불하는 암묵적 거래 (칭찬의 공급자)

"나 예쁘지?"라는 다섯 글자 안에 이만큼의 심리적 구조가 숨어 있다.

## 2. 등장 캐릭터 설계

일러스트의 캐릭터 일관성을 유지하기 위해, 각 캐릭터의 시각적 특징을 고정한다.
캐릭터 시트를 먼저 생성한 뒤, 이후 모든 장면에서 레퍼런스 이미지로 사용한다.

### 시리즈 공통 캐릭터

**주인공 (캐릭터 B)**: 시리즈 전체에 걸쳐 등장하는 남자 주인공. 매 에피소드에서 주변인물과의 일상적 상황을 겪고, 그 안의 심리학적 구조를 경험하는 인물이다. 이후 에피소드에서도 동일한 캐릭터를 유지한다.

### 4-4. TTS 내레이션 생성

#### 추천 방식: 2단계 접근

**1단계 — 초안/타이밍 테스트: edge-tts (무료, Claude Code 자동화)**

edge-tts는 Microsoft Edge의 TTS 엔진을 CLI로 사용할 수 있는 무료 도구다. 카드 등록 불필요, 사용량 제한 없음, Claude Code에서 완전 자동화 가능.

```bash
# 설치
pip install edge-tts

# 한국어 남성 목소리로 생성
edge-tts --voice "ko-KR-InJoonNeural" --text "여러분 주변에도 이런 사람 있지 않나요?" --write-media scene_01.mp3

# 위트 있는 톤에 적합한 한국어 남성 음성 목록:
# ko-KR-InJoonNeural — 차분하면서 약간 위트 있는 남성 (추천)
# ko-KR-HyunsuNeural — 따뜻한 남성
# ko-KR-SunHiNeural — 여성 (비교용)

# 사용 가능한 전체 한국어 음성 확인
edge-tts --list-voices | grep ko-KR
```

Claude Code가 모든 씬의 스크립트를 자동으로 음성 파일로 변환하고, 영상 타이밍을 맞추는 데 사용한다. 최종 품질이 부족하다고 판단되면 2단계로 진행.

**2단계 — 최종 품질: Typecast (수동, 고품질)**

edge-tts로 전체 타이밍과 영상 구조를 확정한 뒤, 최종 음성만 Typecast로 교체하는 방식이다.

Typecast 플랜 현황:
- 무료 플랜: 월 5분 다운로드, 100개 이상 캐릭터, 감정 조절 불가, 카드 등록 불필요
- 베이직 플랜: 월 8,900원(연결제), 60분 다운로드 — EP.01 롱폼(~6분) + 쇼츠(~3분) 제작에 충분

EP.01 예상 분량: 롱폼 약 6분 + 쇼츠 7편 약 3분 = 총 약 9분. 무료 플랜(5분)으로는 부족하므로, 최종 제작 시 베이직 플랜 구독을 권장한다. 정액제이므로 API처럼 예측 불가능한 과금 위험은 없다.

Typecast 작업 방법 (수동):
1. typecast.ai 접속 → 로그인
2. 새 프로젝트 생성
3. 캐릭터 선정: 위트 있는 남성 화자 (추천 캐릭터는 사이트에서 미리듣기로 선별)
4. 아래 씬별 스크립트를 순서대로 입력 → 생성 → 다운로드
5. 파일명은 `scene_01.wav`, `scene_02.wav` ... 형식으로 저장

```
거절할 수 없는 사회적 압력 속에서 특정 답변을 유도하는 구조. 심리학에서는 이걸 강요된 동의라고 부릅니다. 관계의 맥락상 아니라고 말하기 어렵다는 걸 본인도 알고 있기 때문에, 질문의 형식을 빌렸지만 실질적으로는 동의를 강제하는 겁니다.

타인의 칭찬이나 긍정적 반응을 통해 자존감을 유지하려는 패턴. 답변의 진정성은 중요하지 않습니다. 예쁘다는 말 자체를 듣는 행위가 목적이에요.

진심이 궁금한 게 아니라, 칭찬 자판기로 쓰이는 느낌입니다. 나라는 사람이 아니라, 나의 기능만 필요한 상황. 심리학에서 말하는 도구적 관계에 대한 거부감이에요.

[scene_08 — 인지 부조화]
솔직한 답을 하면 관계가 불편해지고, 원하는 답을 해주면 자기 진정성이 훼손됩니다. 이 두 가지 사이에 끼이는 불편함 자체가 대답을 꺼리게 만듭니다.

대답해주기 싫다는 감정. 이건 사실 아주 건강한 심리적 반응입니다. 타인의 감정적 필요를 무조건 충족시켜야 한다는 암묵적 압력을 거부하는 것. 이건 진짜 질문이 아니잖아, 라는 걸 직감적으로 감지하고 있는 겁니다.

[scene_10 — 관계 유지 동기]
그런데 많은 사람들은 이 불편함을 느끼면서도 결국 예쁘다고 답해줍니다. 왜일까요? 이 한마디면 끝나잖아, 라는 계산입니다. 솔직함의 가치보다 관계의 평화를 우선시하는 선택이에요. 단기적으로는 합리적이지만, 반복되면 관계 안에서 자기 목소리를 잃어가는 패턴이 됩니다.

[scene_11 — 자기 합리화]
인지부조화를 해소하는 가장 쉬운 방법은 신념을 바꾸는 겁니다. 뭐, 실제로 안 예쁜 것도 아니니까. 답변 후에 오히려 불편함이 줄어드는 이유가 여기 있어요. 본인이 스스로를 설득한 겁니다.

[scene_12 — 학습된 무력감]
어차피 거절해도 또 물어볼 거잖아. 이런 경험이 쌓이면 저항 자체를 포기합니다. 매번 작은 자기 진정성을 지불하고 관계의 평화를 사는 거래를 하고 있는 거예요. 이게 쌓이면, 그 관계 자체에 대한 은근한 피로감으로 나타납니다.

[scene_15 — 아웃트로]
```

```
    ├── deep_03.wav
    ├── deep_04.wav
    └── deep_05.wav
```

쇼츠 내레이션은 롱폼에서 잘라내는 것이 아니라, 섹션 6-1의 쇼츠 스크립트를 별도로 녹음한다. 쇼츠는 독립 완결형이므로 도입/마무리 톤이 롱폼과 다르다.

### 4-5. 영상 조립 (FFmpeg)

```bash
# 각 씬의 이미지 + 오디오를 결합하여 개별 클립 생성
ffmpeg -loop 1 -i scene_01.png -i scene_01.wav \
  -c:v libx264 -tune stillimage -c:a aac \
  -shortest -pix_fmt yuv420p \
  clip_01.mp4

# 전체 클립을 하나로 이어붙이기
# concat_list.txt:
# file 'clip_01.mp4'
# file 'clip_02.mp4'
# ...
ffmpeg -f concat -safe 0 -i concat_list.txt \
  -c copy final_output.mp4
```

### 4-6. 후처리 (선택)

- 자막 삽입: FFmpeg drawtext 또는 별도 자막 파일(.srt)
- Ken Burns 효과: 정지 이미지에 미세한 줌/팬 적용으로 역동감 부여
- BGM: 저작권 프리 배경음악 추가
- 트랜지션: 씬 전환 시 페이드인/아웃

### 6-1. 롱폼 → 쇼츠 전략

#### 기본 원칙

롱폼 하나의 생명을 일주일 이상으로 연장한다. 단, 쇼츠는 롱폼의 "잘라낸 조각"이 아니라 각각 독립적인 훅과 결론을 가진 자체 완결형 콘텐츠여야 한다. 롱폼 제작 시점부터 쇼츠화를 고려하여 모듈식으로 설계한다.

#### 업로드 스케줄 (예시: 수요일 롱폼 기준)

| 요일 | 콘텐츠 | 유형 | 목적 |
|------|--------|------|------|
| 월 (D-2) | 티저 쇼츠 ① "이 질문, 받아본 적 있으시죠?" | 쇼츠 | 기대감 조성, 롱폼 예고 |
| 화 (D-1) | 티저 쇼츠 ② "이건 가스라이팅이 아닙니다" | 쇼츠 | 호기심 극대화 |
| **수 (D-Day)** | **롱폼: 강요된 칭찬의 심리학 — "나 예쁘지?" 다섯 글자에 감춰진 심리** | **롱폼** | **메인 콘텐츠** |
| 목 (D+1) | 심화 쇼츠 ① 확인 추구 행동 편 | 쇼츠 | 롱폼 유입 유도 |
| 금 (D+2) | 심화 쇼츠 ② 칭찬 자판기 편 | 쇼츠 | 가장 유머러스한 장면 |
| 토 (D+3) | 심화 쇼츠 ③ 인지부조화 편 | 쇼츠 | 심리학 키워드 검색 유입 |
| 일 (D+4) | 심화 쇼츠 ④ 학습된 무력감 편 | 쇼츠 | 공감 포인트 |
| 월 (D+5) | 심화 쇼츠 ⑤ 핵심 정리 편 | 쇼츠 | 롱폼 최종 유입 + 다음 에피소드 예고 |

#### 쇼츠 7편 상세 구성

**티저 쇼츠 ① (15~20초): "이 질문, 받아본 적 있으시죠?"**
- 훅: 수빈이 정면을 보며 "나 예쁘지?" (2초)
- 내용: "답이 정해진 질문. 대답 안 하면 분위기 이상해지고, 해주자니 내 진심이 아닌데." (8초)
- 마무리: "이 불편함의 정체, 심리학에 답이 있습니다." + 롱폼 예고 텍스트 (5초)
- CTA: "수요일 본편에서 전체 분석"

**티저 쇼츠 ② (15~20초): "이건 가스라이팅이 아닙니다"**
- 훅: 화면에 크게 "가스라이팅?" → ❌ 취소선 (2초)
- 내용: "많은 사람들이 이런 상황을 가스라이팅이라고 부르는데, 정확하지 않습니다." (8초)
- 마무리: "그러면 뭐냐고요? 심리학에서는 이걸 '강요된 동의'라고 합니다." (5초)
- CTA: "내일 전체 분석 영상 공개"

**심화 쇼츠 ① (30~45초): 확인 추구 행동**
- 훅: "수빈이 콩이한테도 '나 예쁘지?' 하고 물어봅니다." (3초)
- 내용: 콩이 씬 일러스트 + 확인 추구 행동(Validation-seeking) 핵심 설명 (20~30초)
- 마무리: "대답의 내용이 아니라, 대답을 받아내는 행위 자체가 목적인 거죠." (5초)
- CTA: "전체 분석은 프로필 영상에서"

**심화 쇼츠 ② (30~45초): 칭찬 자판기**
- 훅: 자판기 일러스트 (씬 7) — "나는 칭찬 자판기가 아닌데..." (3초)
- 내용: 도구적 관계(instrumental relationship) 설명 (20~30초)
- 마무리: "진심이 궁금한 게 아니라, 기능만 필요한 상황에 대한 본능적 반발입니다." (5초)

**심화 쇼츠 ③ (30~45초): 인지부조화**
- 훅: 천사/악마 일러스트 (씬 8) — "예쁘다고 해줄까, 말까?" (3초)
- 내용: 인지부조화의 두 가지 해소 방법 (행동 변경 vs 신념 변경) 설명 (25초)
- 마무리: "대부분은 자기 스스로를 설득하는 쪽을 택합니다." (5초)

**심화 쇼츠 ④ (20~30초): 학습된 무력감**
- 훅: 반복되는 잔상 일러스트 (씬 12) — "어차피 또 물어볼 거잖아..." (3초)
- 내용: 매번 자기 진정성을 지불하고 관계의 평화를 사는 거래 (15초)
- 마무리: "이게 쌓이면 관계 자체에 대한 피로감이 됩니다." (5초)

**심화 쇼츠 ⑤ (30~45초): 핵심 정리 + 다음 에피소드 예고**
- 훅: 수빈, 민준, 지우가 나란히 서 있는 일러스트 (씬 14) (3초)
- 내용: "칭찬의 소비자, 거부자, 공급자 — '나 예쁘지?'라는 다섯 글자 안에 이 세 가지가 동시에 작동합니다." (20초)
- 마무리: "다음 에피소드: [다음 주제 예고]" (5초)
- CTA: "구독하면 다음 에피소드를 놓치지 않습니다"

#### 쇼츠 포스팅 시간 (한국 기준)
- 평일: 오후 2~3시 또는 저녁 8~9시
- 주말: 오전 11시 또는 저녁 7~8시

### 6-2. 일러스트 듀얼 포맷 설계 (쇼츠 대응)

롱폼 제작 시점부터 쇼츠 전환을 고려하여 일러스트를 설계한다.

#### 세이프 존 규칙

```
┌─────────────────────────────────┐
│         16:9 (롱폼)              │
│  ┌─────────────────┐            │
│  │                 │            │
│  │   핵심 요소      │            │
│  │   (캐릭터+텍스트) │            │
│  │                 │            │
│  │   ← 세이프 존 →  │            │
│  │   (9:16 크롭 시   │            │
│  │    여기만 남음)   │            │
│  └─────────────────┘            │
└─────────────────────────────────┘
```

- 모든 핵심 요소(캐릭터 얼굴, 말풍선, 키 텍스트)를 화면 중앙 세로 영역에 배치
- 16:9에서는 좌우에 배경이 보이고, 9:16으로 크롭해도 핵심이 잘리지 않음
- FLUX 프롬프트에 "center-composed, main subject in vertical center third" 포함

#### 쇼츠용 추가 작업

- 롱폼 일러스트를 9:16으로 크롭 (FFmpeg: `crop=ih*9/16:ih`)
- 상단/하단에 텍스트 오버레이 공간 확보
- 자막은 화면 하단 1/3에 큰 글씨로 (모바일 가독성)

## 8. Claude Code 작업 시 주의사항

- ComfyUI는 localhost:8188에서 항상 실행 중이어야 함
- 캐릭터 레퍼런스 이미지는 반드시 동일한 파일을 사용할 것 (일관성 핵심)

- 한 씬에서 여러 캐릭터가 등장할 경우, 캐릭터별 레퍼런스 이미지를 모두 입력으로 제공
