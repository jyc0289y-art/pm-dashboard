## 1. FFmpeg 파이프라인

### 1.1 싱크 드리프트 (+20.6초)
- **증상**: 롱폼 뒤쪽으로 갈수록 나레이션과 일러스트가 점점 어긋남
- **원인**: `-shortest` 플래그가 `-loop 1` + `zoompan` 조합에서 1~2초 오버런
- **해결**: 반드시 `-t {duration}` 명시
- **카테고리**: FFmpeg, 싱크
- **심각도**: 🔴 치명적

### 1.2 B컷 세그먼트 이중 포함 (+133초)
- **증상**: 롱폼이 4.5분이어야 하는데 6.7분으로 부풀림
- **원인**: `build_longform()`의 `glob("scene_*.mp4")`가 `_seg0`, `_seg1` 임시 파일도 포함
- **해결**: `_seg/_cut/_bcut/_concat` 패턴을 glob에서 제외하는 필터 추가
- **카테고리**: FFmpeg, 파일 관리
- **심각도**: 🔴 치명적

### 1.3 zoompan 양자화 (픽셀 계단)
- **증상**: 브리딩 효과에서 카메라가 이산적으로 이동 ("양자화")
- **원인**: 1920x1080 해상도에서 1px 이동이 눈에 띔
- **해결**: 내부 2x 업스케일(3840x2160) → Lanczos 다운스케일
- **카테고리**: FFmpeg, 영상 품질
- **심각도**: 🟡 중간

### 1.4 오버레이 빌드가 원본 덮어씀
- **증상**: `--overlay` 플래그로 빌드 시 텍스트가 포함된 영상이 원본을 덮어씀
- **원인**: 오버레이 빌드와 클린 빌드가 같은 output 경로 사용
- **해결**: 오버레이 빌드는 별도 파일명 사용 (예: `_ko` 접미사) [미구현]
- **카테고리**: 파이프라인 설계
- **심각도**: 🟡 중간

## 2. AI 일러스트 생성 (ComfyUI + FLUX)

### 2.1 검은 화면 (scene_13)
- **증상**: 씬 좌측 절반이 완전히 검은색
- **원인**: 프롬프트에 "darker tones, shadowy" → AI가 문자 그대로 해석
- **해결**: 구체적 색상명 사용 ("muted purple-gray background")
- **규칙**: "dark/shadow/black" 직접 사용 금지
- **카테고리**: 프롬프트 엔지니어링
- **심각도**: 🔴 치명적

### 2.2 워터마크/텍스트 누출 (scene_05, 08, 09)
- **증상**: "www.kurome...", "kcat", "춘씨2" 등 불필요한 텍스트
- **원인**: FLUX 모델이 학습 데이터의 워터마크를 재현
- **해결**: "original artwork, no artist signature, no website url" 반드시 추가
- **카테고리**: 프롬프트 엔지니어링
- **심각도**: 🟡 중간

### 2.3 FP8 모델 MPS 불가
- **증상**: Apple Silicon에서 FP8 모델 로드 실패
- **해결**: GGUF Q8 사용 (flux1-kontext-dev-Q8_0.gguf)
- **카테고리**: 환경 설정
- **심각도**: 🟢 1회성

## 3. 브리딩 효과 진화 (v1→v6)

### 3.1 v1: "수전증"
- **파라미터**: drift ±3px, 4.2초 주기
- **피드백**: "카메라를 들고 있는 손이 떨리는 것처럼 보여"
- **교훈**: 빠르고 작은 움직임 = 떨림 느낌

### 3.2 v2: "너무 미세"
- **파라미터**: zoom ±0.3%, 5초 주기
- **피드백**: "변화를 느낄 수 없어"
- **교훈**: 줌만으로는 체감 불가, drift 필요

### 3.3 v3: "이산화 양자화"
- **파라미터**: zoom ±1.2% + drift
- **피드백**: "픽셀 높낮이를 읽는 것처럼 계단이 보여"
- **교훈**: 1080p에서 1px 이동이 시각적으로 감지됨 → 업스케일 필요

### 3.4 v4/v5: "해결"
- **파라미터**: 2x 내부 처리 + 복합 사인파(2주기)
- **피드백**: "훨씬 좋아. 더 역동적이게만 해줘"
- **교훈**: Lanczos 다운스케일로 서브픽셀 부드러움 달성

### 3.5 v6: "랜덤 모션"
- **추가**: 씬별 MD5 해시 → 랜덤 위상/진폭
- **효과**: 씬마다 다른 모션 패턴, 규칙성 제거
- **클램핑**: h ±30px, v ±24px → 구석 클로즈업 방지

## 7. LivePortrait 통합

### 7.1 pkl 템플릿 eye/lip 데이터 누락
- **증상**: named 템플릿(wink.pkl 등)으로 inference 시 `KeyError: 'c_d_eyes_lst'`
- **원인**: named 템플릿은 motion 데이터만 포함, 번호 템플릿(d0~d8)만 eye/lip 키 보유
- **해결**: `live_portrait_pipeline.py`에 zeros 폴백 패치 (numpy zeros 배열)
- **카테고리**: LivePortrait, 호환성
- **심각도**: 🔴 치명적 (모든 씬 처리 차단)

### 7.2 zoompan 비디오 입력 비호환
- **증상**: LP 영상에 zoompan 필터 적용 시 "Failed to configure output pad"
- **원인**: zoompan 필터는 정지 이미지용으로 설계, 비디오 스트림 입력 불가
- **해결**: `scale+crop` 조합으로 미세 줌 구현 (약간 크게 스케일 → 사인파 crop 이동)
- **카테고리**: FFmpeg, LivePortrait
- **심각도**: 🔴 치명적

### 7.3 단순 만화체 얼굴 미감지 (scene_11)
- **증상**: "No face detected in the source image!" — 15개 중 1개 실패
- **원인**: LivePortrait face detector가 심하게 단순화된 만화 스타일을 인식 못함
- **해결**: 해당 씬은 자동으로 정지 이미지+브리딩 폴백 (코드에 폴백 로직 내장)
- **카테고리**: LivePortrait, 폴백
- **심각도**: 🟡 중간

### 7.4 B컷 세그먼트에 잘못된 LP 영상 적용
- **증상**: B컷 구간에 메인 씬의 LP 영상이 루핑됨 (의도하지 않은 캐릭터 등장)
- **원인**: scene_seed에서 씬 이름 추출 시 "bcut" 구분 없이 메인 씬 LP를 반환
- **해결**: `"bcut" in scene_seed` 체크로 B컷 세그먼트는 LP 적용 제외
- **카테고리**: 파이프라인 설계, LivePortrait
- **심각도**: 🟡 중간

## 8. 애니메이션 품질 갭 — 비치키 수준 미달 (미해결)

### 8.1 로컬 무료 도구의 구조적 한계
- **증상**: LP/SadTalker 모두 만화체 얼굴 감지 실패 or 1인만 애니메이션
- **원인**: 실사 얼굴 학습 모델 → 웹툰/만화체 미지원
- **미해결 항목**:
  - 멀티 캐릭터 동시 애니메이션 ❌
  - 대사 기반 립싱크 ❌
  - 머리카락/신체 움직임 ❌
- **카테고리**: 애니메이션 품질
- **심각도**: 🔴 치명적 (비치키 갭의 핵심)

### 8.2 향후 검토 대상 솔루션
- **Kling AI** (image→video): 만화체 OK, 전체 씬 애니메이션, ~$0.3/씬, 15씬≈$5
- **Runway Gen-3**: 유사 기능, 유료
- **Hedra**: 오디오 기반 토킹헤드, 만화체 지원 여부 확인 필요
- **Hailuo (MiniMax)**: 무료 크레딧 있을 수 있음, 확인 필요
- **일러스트 스타일 전환**: 반실사 스타일로 변경 시 LP/SadTalker 모두 작동 가능
- **SadTalker**: 설치 완료(`~/developer/SadTalker/`), 모델 다운 완료, 만화체에서 얼굴 미감지로 사용 불가
- **검토 시점**: EP.02 기획 단계에서 스타일+도구 조합 재결정
