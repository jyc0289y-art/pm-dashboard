# EP01 v3 품질개선 계획

## Context

사용자가 `output/20260416_184151_EP01_v3_lp_bubble.mp4` 프리뷰를 검토한 후 5가지 구체적 컴플레인을 제기. 추가로 코드 분석 중 **멀티세그먼트 클립에서 오버레이가 완전히 누락되는 크리티컬 버그**를 발견. 이 계획은 6단계로 모든 이슈를 해결한다.

## Phase 0: 멀티세그먼트 오버레이 렌더링 버그 수정 (P0 Critical)

**문제**: `build_video_v3.py` 라인 1184-1221의 멀티세그먼트 경로가 `build_bubble_overlay_video()`를 호출하지 않아, 8개 멀티세그먼트 클립의 **모든 말풍선/텍스트 오버레이가 무시됨**.

**영향 클립** (16개 중 8개):
- 00_prologue, 01_siblings, 02_baby, 03_validation
- 05_minjun_pov, 08_dissonance_boundary, 09_jiwoo_intro, 11_helplessness

**수정 방법**:
- 파일: `EP01_forced_compliment/build_video_v3.py` 라인 1184-1221
- 멀티세그먼트 concat 완료 후, 싱글세그먼트와 동일하게 `build_bubble_overlay_video()` 호출 추가
- 각 세그먼트의 시간 오프셋을 오버레이 타이밍에 반영해야 함 (세그먼트별 start_time 누적 계산)

**TRAP 등록**: TRAP-034 (멀티세그먼트 오버레이 누락)

## Phase 1: 타이틀 카드 노출 시간 증가

**사용자 컴플레인 #1**: "강요된 칭찬의 심리학 제목 텍스트가 나오는 장면이 짧아서 정보가 전달되지 않아"

**수정**:
- 파일: `build_video_v3.py` → `V2_SEQUENCE` 딕셔너리
- 클립 `00_prologue`의 S-02 ratio: `0.08` → `0.13` (약 3초 이상 확보)
- S-01 ratio: `0.50` → `0.45` (S-02 증가분 흡수)
- 나머지 (S-00=0.08, S-23=0.34) 유지

## Phase 2: 40초 부근 씬 전환 타이밍 조정

**사용자 컴플레인 #2**: "이 순간에서 민준 모습으로 넘어가는 것보다 '민준이의 머릿속~' 내레이션에서 넘어가는 게 자연스러워"

**수정**:
- 파일: `build_video_v3.py` → `V2_SEQUENCE`
- 클립 `01_siblings`의 비율 조정:
  - S-04 (수빈+동생 → 민준 전환 지점): `0.05` → `0.08`
  - S-05 (민준 내면): `0.34` → `0.31`
  - S-03, S-06 유지 (0.20, 0.41)
- "민준이의 머릿속" 텍스트가 S-05 시작 시점과 일치하도록 비율 계산

## Phase 3: 대사 간 간격 삽입 ("응/응이뭐야" 문제)

**사용자 컴플레인 #3**: "응이 나오자마자 응이뭐야를 붙여놔서 한 사람이 응응이뭐야 라고 하는 것처럼 들려"

**방법**: edge-tts `--write-subtitles` 워드 타이밍 활용 + FFmpeg 후처리

**수정 단계**:
1. `generate_tts_v2.sh`에서 `S03_S06_siblings.mp3` 재생성 시 `--write-subtitles` 옵션 추가
2. VTT 파일에서 "응" 종료 → "응이 뭐야" 시작 사이 타이밍 확인
3. FFmpeg `apad`/무음 삽입으로 대사 간 0.3~0.5초 간격 추가
4. 다른 TTS 클립에도 각본의 비트/정적 지시가 있는 부분 검토하여 동일 적용
   - 각본에 22개 pause 지시 확인됨 (0.3초~2.0초)
   - 우선순위: 화자 전환 지점의 간격 확보

**대상 파일**: `EP01_forced_compliment/audio/longform_v2/S03_S06_siblings.mp3` (및 기타 대사 전환이 빠른 클립)

## Phase 4: 말풍선 + 주석 오버레이 전수 보강

**사용자 컴플레인 #4**: "대사가 나오는 모든 부분에 말풍선을 달아줘"
**사용자 컴플레인 #5**: "각본에서 달기로 했던 주석들도 검토해야 해"

### 4-A: 말풍선 전수 추가

**현황 분석**:
- 현재 manifest에 말풍선(Type A) 있는 씬: S-06 (5개)
- 각본에 대사가 있지만 말풍선이 없는 씬: S-03, S-04, S-05, S-08, S-09, S-10, S-15, S-16, S-17, S-19, S-20, S-21, S-23, S-24

**작업**:
- 파일: `EP01_forced_compliment/text_overlay_v2_manifest.json`
- 각본의 모든 캐릭터 대사 지점에 Type A (speech bubble) 오버레이 추가
- 내면 독백은 Type B (thought bubble) 오버레이
- 화자별 말풍선 위치: `PORTRAIT_RECIPES`의 캐릭터 position 좌표 + `SPEAKER_NAME_MAP` 연동
- 말풍선 꼬리 방향: `speaker_x_ratio` 기반 자동 결정 (기존 로직 활용)

### 4-B: 주석/라벨 전수 감사

**각본 대비 누락 확인된 항목**:
- S-15: Type E "2막B" 구조 라벨 누락
- S-19: 반복 대사 패턴 ("괜찮아" 반복) Type D 킬링라인 누락
- S-23: 프롤로그 bookend — 폰트 크기 확대 필요
- 기타: 각본의 `[자막 주석]` 12건 전수 대조

**작업**: manifest에 누락 오버레이 추가 + 기존 항목 폰트/위치 검증

## Phase 5: 전체 리빌드 + QA 검증

**빌드 순서**:
1. Phase 0 코드 수정 (멀티세그먼트 오버레이)
2. Phase 1-2 비율 수정 (V2_SEQUENCE)
3. Phase 3 TTS 재생성 (간격 삽입)
4. Phase 4 manifest 업데이트 (말풍선/주석)
5. 전체 빌드: `python build_video_v3.py --liveportrait`
6. Layer 1-5 자체 QA 실행

**QA 체크포인트**:
- Layer 1: ffprobe 스펙 (1920×1080, 30fps, AAC)
- Layer 2: 16 클립 전수 존재 + duration 합산 검증
- Layer 3: 말풍선 가독성, LP 크로마키 스필, 씬 전환점 시각 검수
- Layer 4: 각본 26씬 매핑 + 킬링라인 전달 확인
- Layer 5: v2 대비 회귀 비교 (개선/유지/퇴보)

## 검증 방법

1. Phase 0 검증: 멀티세그먼트 클립(예: 01_siblings)에서 말풍선이 렌더링되는지 프레임 샘플링
2. Phase 1 검증: S-02 타이틀 카드가 3초 이상 표시되는지 타임스탬프 확인
3. Phase 2 검증: 40초 부근에서 "민준이의 머릿속" 내레이션과 씬 전환 동기화 확인
4. Phase 3 검증: "응" 후 0.3초+ 간격 → "응이 뭐야" 재생 확인 (오디오 파형)
5. Phase 4 검증: 전체 대사 씬에 말풍선 표시 + 각본 주석 12건 전수 렌더링
6. Phase 5: 전체 QA Layer 1-5 + 회귀 비교 보고서 제출 후 사용자 검수 요청
