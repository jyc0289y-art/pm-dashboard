# 웹 기반 사진 편집기 프로젝트 지침서

## 1. 프로젝트 개요

Lightroom 수준의 컬러 그레이딩 + 포토스케이프 수준의 실용 기능을 결합한 **100% 클라이언트 사이드 웹 사진 편집기**를 구축한다.

### 핵심 원칙
- **서버 연산 없음**: 모든 이미지 처리는 브라우저(WebGL/Canvas)에서 수행. 서버는 정적 파일 호스팅만 담당.
- **실시간 프리뷰**: 슬라이더 조작 시 즉시 반영. WebGL GLSL 셰이더 기반으로 GPU 가속 활용.
- **JPEG 입력 기준 설계**: RAW 파일 처리는 스코프 외. JPEG/PNG 입력 → JPEG/PNG 출력.
- **최신 디바이스 타겟**: WebGL 2.0 지원 브라우저 (전 세계 95%+ 커버리지). 스마트폰/PC 모두 대응.

### 레퍼런스 서비스
- **Photopea**: 100% 클라이언트 사이드 이미지 편집의 검증된 아키텍처 모델

---

## 2. 기술 스택

| 레이어 | 기술 | 이유 |
|---|---|---|
| UI 프레임워크 | React | 상태 관리, 컴포넌트 구조화, 슬라이더 UI |
| 이미지 처리 (실시간 프리뷰) | WebGL 2.0 + GLSL 셰이더 | GPU 가속 필수. Canvas CPU 루프는 고해상도에서 너무 느림 |
| 이미지 처리 (최종 내보내기) | Canvas API | `canvas.toBlob()` 으로 JPEG/PNG 최종 렌더링 |
| 상태 관리 | React useState/useReducer 또는 Zustand | 편집 파라미터 상태 관리 |
| 스타일링 | Tailwind CSS | 빠른 UI 구축 |

### WebGL 셰이더 아키텍처

모든 이미지 보정은 **GLSL fragment shader 파이프라인**으로 처리한다:

```
원본 텍스처 → [셰이더 패스 1: 기본 보정] → [셰이더 패스 2: HSL] → [셰이더 패스 3: 스플릿토닝] → [셰이더 패스 4: 클래리티] → [셰이더 패스 5: 그레인] → 최종 출력
```

멀티패스가 필요한 이유: 클래리티(가우시안 블러)는 별도 패스가 필요하고, 나머지는 단일 셰이더에 합칠 수 있지만 가독성과 유지보수를 위해 분리를 권장한다.

## 3. 기능 명세

### 3-A. 기본 기능 (포토스케이프 수준)

#### 크롭 (Crop)
- 자유 비율 크롭 + 고정 비율 프리셋 (1:1, 4:3, 16:9, 3:2, 9:16)
- 드래그 앤 드롭 크롭 영역 지정
- Canvas API `drawImage(sx, sy, sw, sh, dx, dy, dw, dh)` 로 구현
- 크롭 영역 시각적 오버레이 (어두운 마스크 + 밝은 선택 영역)

#### 리사이즈 (Resize)
- 너비/높이 픽셀값 직접 입력
- 비율 잠금 토글 (aspect ratio lock)
- 퍼센트 기반 리사이즈 (50%, 75% 등)
- Canvas API로 구현

#### 용량 최적화 (Export Optimization)
- JPEG 품질 슬라이더 (1~100)
- `canvas.toBlob('image/jpeg', quality)` 로 구현
- 예상 파일 크기 실시간 표시
- 출력 포맷 선택: JPEG / PNG
- 긴 변 기준 리사이즈 옵션 (예: 긴 변 1920px로 제한 — SNS 전송용)

#### 회전/뒤집기
- 90도 회전 (시계/반시계)
- 좌우 반전 / 상하 반전
- Canvas transform으로 구현

### 3-B. 색보정 기능 (Lightroom 수준)

#### 노출 (Exposure)
- 범위: -3.0 ~ +3.0 EV
- 구현: 감마 커브 조절. `pow(color, 1.0 / (1.0 + exposure))` 또는 단순 밝기 곱 `color * pow(2.0, exposure)`
- 후자가 실제 EV 스톱에 더 가까움

#### 콘트라스트 (Contrast)
- 범위: -100 ~ +100
- 구현: S-커브 적용. 중간톤(0.5) 기준으로 밝은 부분은 더 밝게, 어두운 부분은 더 어둡게.
- `color = (color - 0.5) * (1.0 + contrast/100.0) + 0.5`

#### 하이라이트 / 섀도 (Highlights / Shadows)
- 각각 범위: -100 ~ +100
- 구현: luminance 기준으로 밝은 영역/어두운 영역을 분리하여 각각 밝기 조절
- JPEG 기준이므로 RAW 수준의 복구는 불가하지만, 근사치로 충분히 실용적
- 하이라이트: `if (luminance > threshold) apply adjustment`
- 섀도: `if (luminance < threshold) apply adjustment`
- threshold는 smooth step으로 부드럽게 전환

#### 색온도 (White Balance / Color Temperature)
- 범위: 2000K ~ 10000K (슬라이더)
- 구현: **Planckian locus 기반 RGB 보정**
  - 색온도 K값 → RGB 곱셈 계수 변환
  - Tanner Helland의 색온도-RGB 변환 공식 사용 가능:
    ```
    if temp <= 6600:
      R = 255
      G = 99.4708 * ln(temp/100) - 161.1196
      B = 138.5177 * ln(temp/100 - 10) - 305.0448
    else:
      R = 329.698 * pow(temp/100 - 60, -0.1332)
      G = 288.122 * pow(temp/100 - 60, -0.0755)
      B = 255
    ```
  - 원본 대비 상대적 보정으로 적용 (기준 색온도 대비 목표 색온도의 비율)

#### 채도 (Saturation)
- 범위: -100 ~ +100
- 구현: RGB → HSL 변환 후 S 채널 스케일링
- `S_new = S * (1.0 + saturation/100.0)`
- 또는 luminance 기반: `color = mix(vec3(luminance), color, 1.0 + saturation/100.0)`

#### 자연스러운 채도 (Vibrance)
- 범위: -100 ~ +100
- Saturation과의 차이: 이미 채도가 높은 색은 덜 올리고, 채도가 낮은 색을 더 올림
- 구현: 현재 채도에 반비례하는 가중치 적용
- `weight = 1.0 - currentSaturation; S_new = S + vibrance * weight`

### 3-C. 고급 색보정 기능

#### HSL 개별 채널 조절
- 6개 또는 8개 색상 채널: Red, Orange, Yellow, Green, Aqua/Cyan, Blue, Purple, Magenta
- 각 채널별로 Hue(색상 시프트), Saturation(채도), Luminance(밝기) 독립 조절
- 구현:
  1. RGB → HSL 변환
  2. 현재 픽셀의 Hue가 어떤 채널 범위에 속하는지 판별
  3. 해당 채널의 H/S/L 조절값 적용
  4. 인접 채널 경계에서는 smooth interpolation으로 부드럽게 전환
  5. HSL → RGB 역변환

**이 기능이 핵심 — 아래 기능들이 HSL 기반으로 동작:**

#### 스플릿 토닝 (Split Toning)
- 섀도 색상 (Hue) + 섀도 채도
- 하이라이트 색상 (Hue) + 하이라이트 채도
- 밸런스 슬라이더 (섀도/하이라이트 경계 조절)
- 구현:
  1. 픽셀의 luminance 계산
  2. luminance가 낮으면(섀도) → 섀도 색상 틴트 블렌딩
  3. luminance가 높으면(하이라이트) → 하이라이트 색상 틴트 블렌딩
  4. 밸런스 값으로 경계 위치 조절
  5. `tintColor = hslToRgb(selectedHue, selectedSat, pixelLuminance)`
  6. `result = mix(originalColor, tintColor, strength)`

#### 선택적 컬러 (Selective Color / 부분 탈색)
- 특정 Hue 범위만 채도 유지, 나머지는 모노크롬 처리
- 유지할 색상 범위 선택 (예: 시안+블루+마젠타만 유지 → 네온만 살리기)
- 구현: HSL 변환 후, 선택되지 않은 Hue 범위의 Saturation을 0으로
- 경계는 feathering으로 부드럽게

#### 클래리티 (Clarity / Local Contrast)
- 범위: -100 ~ +100
- **단순 샤프닝이 아님** — "대형 반경 언샤프 마스크"로 로컬 콘트라스트 강화
- 구현 (WebGL 멀티패스):
  1. **패스 1**: 원본 텍스처에 큰 반경(예: 50~100px) 가우시안 블러 적용 → 블러 텍스처 생성
     - 가우시안 블러는 **separable filter**로 구현: 수평 패스 + 수직 패스 = 2패스
     - 큰 커널도 2패스면 충분. 모바일 GPU에서도 수십 ms 수준
  2. **패스 2**: `result = original + clarity * (original - blurred)`
     - original - blurred = 로컬 디테일
     - clarity > 0 이면 로컬 콘트라스트 강화, < 0 이면 소프트 효과
- **성능 참고**: 2400만 화소 이미지 기준, 최신 모바일 GPU(Adreno 740, Mali-G720급)에서 실시간 프리뷰 가능. 데스크톱은 여유.

#### 그레인 (Film Grain)
- 강도 (Amount) + 크기 (Size)
- 구현: GLSL에서 pseudo-random 노이즈 생성 → luminance에 곱하거나 더함
- `noise = fract(sin(dot(uv, vec2(12.9898, 78.233))) * 43758.5453)`
- 크기는 UV 스케일링으로 조절

## 6. 파라미터 전체 목록 및 기본값

```typescript
interface EditParams {
  // 기본 보정
  exposure: number;       // -3.0 ~ +3.0, 기본 0
  contrast: number;       // -100 ~ +100, 기본 0
  highlights: number;     // -100 ~ +100, 기본 0
  shadows: number;        // -100 ~ +100, 기본 0

  // 색보정
  colorTemp: number;      // 2000 ~ 10000 (K), 기본 5800 (Daylight)
  saturation: number;     // -100 ~ +100, 기본 0
  vibrance: number;       // -100 ~ +100, 기본 0

  // HSL (8채널 × 3속성)
  hsl: {
    [channel: 'red'|'orange'|'yellow'|'green'|'aqua'|'blue'|'purple'|'magenta']: {
      hue: number;        // -30 ~ +30, 기본 0
      saturation: number; // -100 ~ +100, 기본 0
      luminance: number;  // -100 ~ +100, 기본 0
    }
  };

  // 스플릿 토닝
  splitToning: {
    shadowHue: number;      // 0 ~ 360, 기본 0
    shadowSat: number;      // 0 ~ 100, 기본 0
    highlightHue: number;   // 0 ~ 360, 기본 0
    highlightSat: number;   // 0 ~ 100, 기본 0
    balance: number;        // -100 ~ +100, 기본 0
  };

  // 효과
  clarity: number;        // -100 ~ +100, 기본 0
  grain: {
    amount: number;       // 0 ~ 100, 기본 0
    size: number;         // 0 ~ 100, 기본 50
  };

  // 선택적 컬러
```

## 7. GLSL 셰이더 구현 가이드

### 7-A. 기본 보정 셰이더 (단일 패스)

```glsl
// fragment shader — 기본 보정 + 색온도 + 채도/자연채도 + 콘트라스트
uniform sampler2D uTexture;
uniform float uExposure;      // -3 ~ +3
uniform float uContrast;      // -100 ~ +100
uniform float uHighlights;    // -100 ~ +100
uniform float uShadows;       // -100 ~ +100
uniform float uColorTemp;     // 보정 계수 (K값에서 변환)
uniform float uSaturation;    // -100 ~ +100
uniform float uVibrance;      // -100 ~ +100
varying vec2 vUv;

// RGB to HSL, HSL to RGB 함수 포함 필요
// Luminance 계산: dot(color, vec3(0.2126, 0.7152, 0.0722))

void main() {
  vec4 texColor = texture2D(uTexture, vUv);
  vec3 color = texColor.rgb;

  // 1. 노출
  color *= pow(2.0, uExposure);

  // 2. 콘트라스트
  color = (color - 0.5) * (1.0 + uContrast / 100.0) + 0.5;

  // 3. 하이라이트/섀도
  float lum = dot(color, vec3(0.2126, 0.7152, 0.0722));
  float highlightMask = smoothstep(0.5, 1.0, lum);
  float shadowMask = 1.0 - smoothstep(0.0, 0.5, lum);
  color += color * highlightMask * (uHighlights / 200.0);
  color += color * shadowMask * (uShadows / 200.0);

  // 4. 색온도 (RGB 곱 보정)
  color *= uColorTempRGB; // vec3, K값에서 사전 계산된 RGB 계수

  // 5. 채도
  float gray = dot(color, vec3(0.2126, 0.7152, 0.0722));
  color = mix(vec3(gray), color, 1.0 + uSaturation / 100.0);

  // 6. 자연채도 (Vibrance)
  float currentSat = length(color - vec3(gray)) / (gray + 0.001);
  float vibranceWeight = 1.0 - clamp(currentSat, 0.0, 1.0);
  color = mix(vec3(gray), color, 1.0 + uVibrance / 100.0 * vibranceWeight);

  color = clamp(color, 0.0, 1.0);
  gl_FragColor = vec4(color, texColor.a);
}
```

### 7-B. HSL 개별 채널 셰이더

```
// 8채널 HSL 조절
// 각 채널의 Hue 중심값: Red=0, Orange=30, Yellow=60, Green=120, Aqua=180, Blue=240, Purple=270, Magenta=300
// 각 채널 폭: ~30도 (인접 채널과 smooth interpolation)

for each pixel:
  convert RGB → HSL
  determine which channel(s) the hue falls into
  apply weighted H/S/L adjustments from matching channel(s)
  convert HSL → RGB
```

### 7-C. 스플릿 토닝 셰이더

```
// luminance 기반 섀도/하이라이트 분리 후 색상 틴트 적용
for each pixel:
  lum = luminance(color)
  shadowWeight = smoothstep(balance, 0.0, lum) * shadowSat
  highlightWeight = smoothstep(1.0-balance, 1.0, lum) * highlightSat
  shadowTint = hslToRgb(shadowHue, 1.0, lum)
  highlightTint = hslToRgb(highlightHue, 1.0, lum)
  result = mix(color, shadowTint, shadowWeight)
  result = mix(result, highlightTint, highlightWeight)
```

### 7-D. 클래리티 셰이더 (멀티패스)

```
// 패스 1a: 수평 가우시안 블러 (separable)
// 패스 1b: 수직 가우시안 블러 (separable)
// → 블러 텍스처 생성

// 패스 2: 언샤프 마스크 적용
result = original + clarity * (original - blurred)
```

커널 크기: 반경 50~100px 권장. Separable filter이므로 수평/수직 각각 1D 커널만 연산하면 됨.

### 7-E. 그레인 셰이더

## 11. 제약사항 및 스코프 외

- **RAW 파일 처리**: 스코프 외. dcraw/LibRaw WASM 컴파일은 별도 프로젝트 수준.
- **레이어/마스크**: 스코프 외. 단일 이미지 편집에 집중.
- **배치 처리**: Phase 4 이후 확장 가능하나, 초기 스코프에서는 단일 이미지.
- **하이라이트/섀도 복구 한계**: JPEG 입력이므로 RAW 수준의 복구는 원리적으로 불가. 근사치로 구현하되 UI에서 과도한 기대를 유발하지 않도록 범위 제한.

## 12. 핵심 참고 자료

### 색온도 → RGB 변환
- Tanner Helland: "How to Convert Temperature (K) to RGB" — 웹 검색으로 공식 확인

### HSL ↔ RGB 변환
- 표준 수학 공식. GLSL에서의 구현은 Shader Toy 등에서 검증된 코드 다수 존재.

### 가우시안 블러 최적화
- Separable filter (수평+수직 2패스)
- 큰 반경은 다운샘플 → 블러 → 업샘플 기법도 고려 가능

### WebGL 프레임버퍼
- 멀티패스 셰이더를 위해 Framebuffer Object (FBO) 사용. 패스 간 텍스처를 ping-pong.
