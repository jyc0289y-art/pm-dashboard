# Claude Code 텔레그램 원격 제어 봇 기획서

## 개요

맥북에서 실행 중인 Claude Code CLI를 텔레그램 메시지로 원격 제어하는 Python 봇.
모바일에서 텔레그램으로 명령을 보내면, 맥북이 `claude -p` 명령을 실행하고 결과를 텔레그램으로 반환한다.

## 시스템 구조

```
[모바일 텔레그램]
    ↓ 메시지/파일 전송
[텔레그램 Bot API]
    ↓
[맥북: Python 봇 서버 (상주 실행)]
    ↓ subprocess로 실행
[claude -p "명령" --project-dir /path]
    ↓ stdout 캡처
[결과 텍스트를 텔레그램으로 반환]
```

## 핵심 기능

### 1. 기본 질문/응답
- 텔레그램에서 텍스트 메시지를 보내면 `claude -p "메시지내용"` 실행
- stdout 결과를 텔레그램 메시지로 반환
- 텔레그램 메시지 길이 제한(4096자) 초과 시 텍스트 파일로 전송

### 2. 프로젝트 디렉토리 관리
- `/setdir /path/to/project` — 작업 디렉토리 변경
- `/getdir` — 현재 작업 디렉토리 확인
- `/ls` — 현재 디렉토리의 파일 목록 표시
- 설정된 디렉토리는 `claude -p` 실행 시 `cwd` (current working directory)로 사용
- 기본값: `~/Projects` (환경변수 `CLAUDE_WORK_DIR`로 오버라이드 가능)

### 3. 파일 첨부 지원
- **수신 (텔레그램 → 맥북):** 사진/문서를 텔레그램으로 보내면 작업 디렉토리의 `_inbox/` 하위 폴더에 저장. 캡션이 있으면 해당 파일을 컨텍스트로 포함하여 Claude에 전달
- **송신 (맥북 → 텔레그램):** `/getfile filename` 명령으로 작업 디렉토리 내 파일을 텔레그램으로 전송

### 4. 보안 — 사용자 화이트리스트
- 허용된 텔레그램 User ID 목록을 설정 파일에 관리
- 미인증 사용자의 메시지는 무시 + 경고 로그 기록
- `/myid` 명령은 인증 없이 동작 (본인 ID 확인용)

## 명령어 정리

| 명령어 | 설명 | 인증 필요 |
|--------|------|-----------|
| (일반 텍스트) | Claude Code에 질문 전달 | ✅ |
| (파일 첨부) | 파일 저장 + 캡션이 있으면 Claude에 전달 | ✅ |
| `/setdir <경로>` | 작업 디렉토리 변경 | ✅ |
| `/getdir` | 현재 작업 디렉토리 확인 | ✅ |
| `/ls` | 현재 디렉토리 파일 목록 | ✅ |
| `/getfile <파일명>` | 파일을 텔레그램으로 전송 | ✅ |
| `/myid` | 본인 텔레그램 User ID 확인 | ❌ |
| `/help` | 명령어 도움말 | ✅ |

## 설정 방식

### 환경변수 (`.env` 파일)
```env
TELEGRAM_BOT_TOKEN=your_bot_token_here
CLAUDE_WORK_DIR=~/Projects
ALLOWED_USERS=123456789,987654321
CLAUDE_TIMEOUT=300
```

### 파일 구조
```
claude-telegram-bot/
├── bot.py              # 메인 봇 스크립트
├── .env                # 환경변수 (gitignore 대상)
├── .env.example        # 환경변수 템플릿
├── requirements.txt    # python-telegram-bot, python-dotenv
└── README.md           # 설치 및 실행 가이드
```

## 기술 스택

- **Python 3.11+**
- **python-telegram-bot** (v20+, async 기반)
- **python-dotenv** — 환경변수 로딩
- **subprocess** (asyncio.create_subprocess_exec) — Claude CLI 실행

## 주요 구현 사항

### Claude CLI 호출
```
claude -p "사용자 메시지" --output-format text
```
- `cwd`를 현재 설정된 프로젝트 디렉토리로 지정
- timeout 적용 (기본 300초), 초과 시 프로세스 kill + 타임아웃 알림
- 실행 중임을 알리는 "⏳ 처리 중..." 메시지를 먼저 전송

### 파일 첨부 처리
- 텔레그램에서 파일 수신 시 `{작업디렉토리}/_inbox/` 에 저장
- 파일명: 원본 파일명 유지 (중복 시 타임스탬프 접미사)
- 캡션이 있는 경우: 저장된 파일 경로를 포함하여 Claude에 전달
  - 예: `claude -p "이 파일을 분석해줘: /path/to/_inbox/data.csv"`
- 캡션이 없는 경우: 저장 완료 메시지만 반환

### 응답 길이 처리
- 4096자 이하: 일반 텔레그램 메시지로 전송
- 4096자 초과: `.txt` 파일로 만들어 Document로 전송
- 마크다운 파싱 모드 사용 (코드블록 등 서식 유지)

### 에러 처리
- Claude CLI가 stderr 출력 시 → 에러 내용을 사용자에게 전달
- 프로세스 비정상 종료 시 → exit code와 함께 알림
- 네트워크 끊김 시 → 자동 재연결 (python-telegram-bot 기본 지원)

## 실행 및 상주 방법

### 직접 실행
```bash
python3 bot.py
```

### macOS 자동 시작 (LaunchAgent)
`~/Library/LaunchAgents/com.claude.telegram-bot.plist` 생성하여 로그인 시 자동 시작 설정 권장. 기획서에는 plist 예시도 README에 포함할 것.

## 보안 고려사항

- `.env` 파일은 반드시 `.gitignore`에 포함
- `ALLOWED_USERS`가 비어있으면 봇 시작을 거부 (실수 방지)
- `/setdir`로 설정 가능한 경로를 홈 디렉토리 하위로 제한 (경로 탈출 방지)
- 첨부파일 크기 제한: 20MB (텔레그램 Bot API 제한)
