# MarketLink (마켓링크) 프로젝트 기획서

> 최종 업데이트: 2026-03-16
> 버전: 3
> 변경사항 (v2→v3): ① 브랜드명 Market Memory → MarketLink 확정 (SeouLink/SoLink과 브랜드 일관성), ② DWVA 매매시스템 연동 — 매크로 예측 보조지표 모듈 설계 (방어적 감속이 아닌 공격적 배분 최적화), ③ 상관관계 엔진 공유 구조 설계 (MarketLink ↔ DWVA)
> 상태: 초기 기획 (컨셉 정의 + 학술 기반 조사 + DWVA 연동 설계 완료)

## 1. 프로젝트 개요

### 1.1 핵심 컨셉

MarketLink는 **세 개의 축**으로 구성된 금융 시장 인텔리전스 플랫폼이다.

**축 1 — 역사적 이벤트 매핑 (Historical Event Mapping)**
S&P 500 등 주요 시장 지수의 장기 시계열 그래프 위에 정치경제학적·지정학적·지경학적 사건들을 다층 레이어로 매핑하는 인터랙티브 시각화.

**축 2 — 실시간 이벤트 감지 (Real-time Event Detection)**
최근 5년 구간에 대해 뉴스를 실시간 모니터링하고, 시장에 유의미한 영향을 주는 사건을 자동 감지·분류하여 차트에 즉시 반영. "살아있는 역사 차트".

**축 3 — 크로스에셋 디커플링 탐색 (Cross-Asset Decoupling Finder)**
다양한 지수·자산군 간의 상관관계를 동적으로 추적하고, 디커플링(탈동조화)되어 독립적으로 움직이거나 역방향으로 움직이는 자산 쌍을 탐색. "돈이 쉬지 않고 일하게 만드는" 자산 로테이션 도구. 축 1·2의 이벤트 매핑과 결합하면 "왜 지금 이 두 자산이 디커플링되었는가"까지 설명 가능.

> 세 축의 융합이 MarketLink의 핵심 차별점이다.
> 기존 도구들은 축 1(과거 주석), 축 3(상관계수)을 각각 단편적으로 제공하지만,
> 세 축을 하나의 플랫폼에서 연결하는 서비스는 존재하지 않는다.

### 1.2 왜 이것을 만드는가

전 세계 투자자들의 심리는 S&P 500, NASDAQ, DJIA에 의해 지대한 영향을 받는다. 이 지표들은 사실상 근현대 경제사의 **실시간 투표 기록**이다. 그러나:
- 이 데이터를 역사적 맥락 속에서 탐색할 수 있는 도구는 없거나 극히 제한적이다.
- "지금 무슨 일이 시장에 영향을 주고 있는가"를 실시간으로 구조화하여 보여주는 도구가 없다.
- 자산 간 상관관계 변화를 이벤트 맥락과 연결하여 설명하는 도구가 없다.

### 1.3 브랜드 포지셔닝

- **모기업**: SL Corporation (에스엘코퍼레이션)
- **브랜드**: **MarketLink (마켓링크)** — 확정
  - SeouLink, SoLink과 "Link" 키워드 일관성 유지
  - "시장(Market)과 사건(Event)을 연결(Link)한다"는 의미
- **포지션**: SeouLink(한국), SoLink(생존/아웃도어)와 독립된 **금융 리서치/투자 도구** 브랜드 라인
- **내부 활용**: DWVA 매매시스템의 매크로 이슈 조기경보 모듈로도 기능 (섹션 4.5 참조)

### 1.4 우선 구현 범위

- **1차**: S&P 500 단일 지수 (1920~현재) + 최근 5년 상세 이벤트 + 실시간 감지
- **2차**: DJIA 추가 (1896~현재) + 원유(WTI/Brent), 금 등 원자재 지수 추가 → 디커플링 분석 도입
- **3차**: NASDAQ, 채권(10Y Treasury), VIX, 달러인덱스, 비트코인 등 추가 → 디커플링 분석 본격화

## 2. 기존 학술 연구 및 시도 분석

### 2.1 주요 선행 연구

#### ① GPR Index — Caldara & Iacoviello (2022)

- **출처**: American Economic Review, Vol. 112(4), pp. 1194-1225
- **방법론**: 미국·영국·캐나다 주요 10개 신문에서 지정학적 긴장 관련 키워드를 자동 텍스트 검색하여 월별 지수화
- **범위**: 1900~현재 (Historical GPR Index), 1985~현재 (Benchmark GPR Index)
- **분류 체계**: 8개 카테고리 (전쟁 위협, 평화 위협, 군비 증강, 핵 위협, 테러 위협, 전쟁 개시, 전쟁 확대, 테러 행위)
- **핵심 발견**: GPR 상승 → 투자 감소, 고용 감소, 재난 확률 상승, 하방 리스크 확대
- **데이터 공개**: matteoiacoviello.com/gpr.htm (무료)
- **하위 지수**: GPR Threats (위협), GPR Acts (실현), 18개국 국가별 GPR

#### ② EPU Index — Baker, Bloom & Davis (2016)

- **출처**: Quarterly Journal of Economics, Vol. 131(4), pp. 1593-1636
- **방법론**: 주요 10개 미국 신문에서 "경제/불확실성/정책" 3중 키워드 조합 빈도 측정
- **범위**: 1900~현재 (Historical), 1985~현재 (Benchmark)
- **핵심 발견**: EPU 상승 → 주가 변동성 증가, 정책 민감 섹터 투자·고용 감소
- **데이터 공개**: policyuncertainty.com (무료, Bloomberg/FRED/Haver/Reuters 채택)
- **확장**: 12개국 EPU, 카테고리별 EPU (통화정책, 재정정책, 무역정책, 의료정책 등)

#### ③ Event Study Methodology

- **핵심 참조**: Fama et al. (1969), 이후 반세기간 발전
- **방법론**: 특정 이벤트 전후 비정상 수익률(Abnormal Returns)을 측정하여 이벤트의 시장 영향을 통계적으로 검증
- **최근 적용 사례**: 러시아-우크라이나 전쟁(2022), 이스라엘-하마스 분쟁(2023), 미국-이란 분쟁(2026)
- **한계**: 개별 이벤트의 단기(수일~수개월) 영향만 측정, 복합 이벤트의 누적 효과 미반영

#### ④ 실무 시각화 사례

- **Landmark Wealth Management**: S&P 500 차트에 1932~현재 군사 분쟁 어노테이션
- **LPL Research**: 2차 대전 이후 지정학적 충격 후 S&P 500 평균 반응 데이터 (평균 -5%, 3주 내 바닥, 1-2개월 회복)
- **Citigroup Global Wealth Report 2024**: 1940년 이후 20대 지정학적 이벤트의 S&P 500 영향 순위
- **Hartford Funds**: 무력 분쟁 후 S&P 500이 1년 후 상승한 비율 약 70%

#### ⑤ 크로스에셋 상관관계 분석 도구 — 신규 조사

- **Portfolio Visualizer**: ETF/뮤추얼 펀드 간 상관계수 매트릭스와 롤링 상관관계 시각화 제공. 무료/유료. 단, 이벤트 맥락 연결 없음.
- **Guggenheim Investments Asset Class Correlation Map**: 자산군 간 상관관계 히트맵. 정적.
- **Permutable AI Trading Co-Pilot**: AI 기반으로 원자재 간 디커플링 탐지. 기관 투자자 대상 고가 솔루션.
- **J.P. Morgan Private Bank**: 주식-채권 상관관계의 시대별 변화를 분석하지만, 개인 투자자 대상이 아님.
- **공통 한계**: 상관관계 "변화"는 보여주지만, **왜** 변화했는지 (이벤트 맥락)를 설명하지 못함. MarketLink의 축 1·2와 결합하면 이 간극을 메울 수 있음.

### 2.2 기존 연구/시도의 한계 — MarketLink의 차별화 근거

| 기존 접근 | 한계 | MarketLink의 차별화 |
|-----------|------|----------------------|
| **GPR/EPU 지수** | 추상적 숫자. 스파이크가 "무엇" 때문인지 직관적으로 알 수 없음 | 지수 위에 **구체적 사건 이름, 맥락, 카테고리**를 직접 매핑. GPR/EPU를 보조 레이어로 중첩 |
| **투자 리서치 차트** | 군사 분쟁만 단편적으로 주석. 다른 축 미통합 | **6대 카테고리**의 다층 레이어 + 실시간 업데이트 |
| **이벤트 스터디** | 개별 이벤트의 단기 영향만 정량화 | 장기 시계열 위에 **시간적 밀도**와 **인과 체인**을 시각화 |
| **일반 차트 도구** | 사전 큐레이션 이벤트 DB 없음. 수동 주석만 가능 | **전문 큐레이션 이벤트 DB** + 실시간 자동 감지 |
| **상관관계 도구** (Portfolio Visualizer 등) | 숫자만 보여줌. "왜" 상관관계가 변했는지 설명 없음 | 디커플링 변화 시점에 **이벤트를 자동 매칭**하여 원인 설명 |
| **기존 모든 접근** | 역사 매핑, 실시간 감지, 상관관계 분석이 각각 분리 | **세 축의 융합**: 과거 맥락 + 실시간 감지 + 자산 간 관계를 하나의 플랫폼에서 |

### 2.3 학술적 포지셔닝

MarketLink는 기존 학술 연구를 "소비"하는 도구이지, 기존 연구를 "대체"하려는 것이 아니다.

- GPR Index와 EPU Index의 데이터를 **보조 레이어로 통합**
- Event Study의 방법론을 **각 이벤트의 정량 영향 표시**에 차용
- 크로스에셋 상관관계 분석의 학술 기반 (DCC-GARCH, 롤링 상관계수) 활용
- 이 위에 **직관적 시각화 + 실시간 감지 + 사건 간 인과 네트워크 + 디커플링 탐색**이라는 새로운 가치를 추가

## 3. 이벤트 택소노미 (Event Taxonomy) 상세 설계

### 3.1 설계 원칙

1. **MECE**: 모든 시장 영향 사건이 하나의 카테고리에 분류될 수 있어야 하고, 카테고리 간 중복은 최소화
2. **계층 구조**: 대분류 → 중분류 → 소분류의 3단 구조
3. **다중 태깅 허용**: 1개 대분류 + 크로스레퍼런스 태그
4. **시간 해상도 구분**: "점 사건(Point Event)"과 "기간 사건(Period Event)"
5. **실시간 호환**: 자동 감지된 이벤트도 동일 스키마로 즉시 분류 가능한 구조

### 3.2 대분류 6개 카테고리

```
┌─────────────────────────────────────────────────────┐
│                 MarketLink 이벤트 택소노미          │
├─────────────────────────────────────────────────────┤
│  L1. 지정학 (Geopolitics)                            │
│  L2. 지경학 (Geoeconomics)                           │
│  L3. 통화·재정 정책 (Monetary & Fiscal Policy)        │
│  L4. 금융 시스템 (Financial System)                   │
│  L5. 기술·산업 구조 (Technology & Industry)            │
│  L6. 사회·자연 (Society & Nature)                     │
└─────────────────────────────────────────────────────┘
```

### 3.3 각 카테고리 상세 (대분류 → 중분류 → 소분류 예시)

#### L1. 지정학 (Geopolitics)

| 중분류 | 소분류 (예시) | 대표 사건 |
|--------|-------------|----------|
| **전쟁/무력 분쟁** | 세계대전, 지역전쟁, 내전, 대리전 | WWI, WWII, 한국전쟁, 베트남전, 걸프전, 이라크전, 우크라이나전, 이란전(2026) |
| **핵 위기** | 핵 대치, 핵 확산, 핵 실험 | 쿠바 미사일 위기(1962), 북한 핵실험(2006~), 이란 핵합의(2015)/탈퇴(2018) |
| **테러리즘** | 대규모 테러, 생화학 테러 | 9/11(2001), 런던 폭탄(2005), 파리 테러(2015) |
| **정권 교체/혁명** | 쿠데타, 민주화 운동, 정권 붕괴 | 이란 혁명(1979), 아랍의 봄(2011), 미얀마 쿠데타(2021) |
| **동맹 재편** | NATO 확장, 군사동맹 형성/해체 | NATO 동진, AUKUS(2021), 중러 밀착 |
| **영토 분쟁** | 영유권 분쟁, 해양 분쟁 | 대만해협 위기, 남중국해, 크림반도 병합(2014) |

#### L2. 지경학 (Geoeconomics)

| 중분류 | 소분류 (예시) | 대표 사건 |
|--------|-------------|----------|
| **무역전쟁/관세** | 보호무역, 관세 전쟁, 수출 통제 | 스무트-홀리 관세법(1930), 미중 무역전쟁(2018~), 트럼프 관세(2025~) |
| **경제 제재** | 국가 제재, 금융 제재, 기술 제재 | 이란 제재, 러시아 SWIFT 차단(2022), 중국 반도체 수출통제 |
| **에너지 충격** | OPEC 결정, 석유 금수, 에너지 위기 | OPEC 석유 금수(1973), 유가 폭락(1986, 2014, 2020) |
| **공급망 충격** | 글로벌 공급망 교란, 핵심 자원 독점 | 수에즈 위기(1956), 코로나 공급망 위기(2021), 희토류 독점 |
| **통화 체계 변동** | 기축통화 전환, 통화 블록 형성 | 브레턴우즈 체제(1944), 닉슨 쇼크(1971), 유로 출범(1999) |
| **국제기구/협정** | WTO, IMF, G7/G20, 무역협정 | GATT(1947), WTO 출범(1995), TPP/CPTPP, RCEP |

#### L3. 통화·재정 정책 (Monetary & Fiscal Policy)

| 중분류 | 소분류 (예시) | 대표 사건 |
|--------|-------------|----------|
| **금리 정책** | 인상 사이클, 인하 사이클, 긴급 조치 | 볼커 쇼크(1979-82), 코로나 제로금리(2020) |
| **양적완화/긴축** | QE, QT, 대차대조표 정책 | QE1/2/3(2008-2014), 무제한 QE(2020) |
| **재정 정책** | 재정 부양, 긴축, 부채한도 | 뉴딜(1933), CARES Act(2020), 부채한도 위기(2011, 2023) |
| **규제 정책** | 금융규제, 산업규제, 반독점 | 글래스-스티걸(1933)/폐지(1999), 도드-프랭크(2010) |
| **인플레이션/디플레이션** | 스태그플레이션, 디플레이션 | 대공황 디플레이션(1930s), 스태그플레이션(1970s) |

#### L4. 금융 시스템 (Financial System)

| 중분류 | 소분류 (예시) | 대표 사건 |
|--------|-------------|----------|

### 3.7 실시간 이벤트 감지 시스템 (Real-time Event Detection) — 신규

#### 아키텍처 개요

```
뉴스 소스 (RSS/API)          경제 캘린더
        ↓                        ↓
   NLP 파이프라인          스케줄 기반 수집
   (FinBERT + LLM)
        ↓                        ↓
   이벤트 후보 추출         예정 이벤트 매칭
        ↓                        ↓
        └──────── 통합 ────────┘
                   ↓
          택소노미 자동 분류 (LLM)
                   ↓
          시장 영향도 추정 (significance scoring)
                   ↓
     ┌─────────────┼─────────────┐
     ↓             ↓             ↓
  MAJOR →       SIGNIFICANT →  MINOR →
  즉시 반영     검수 후 반영    배치 반영
```

#### 뉴스 소스

| 소스 유형 | 예시 | 수집 방식 |
|-----------|------|----------|
| 주요 통신사 | Reuters, AP, AFP | RSS / API |
| 금융 전문 매체 | Bloomberg (무료 부분), CNBC, WSJ (무료 부분) | RSS |
| 정부/기관 발표 | Fed, ECB, BLS, Census Bureau | 공식 RSS / API |
| 경제 캘린더 | FOMC 일정, CPI 발표일, 고용 보고서 | 사전 등록 + 발표 시 수집 |

#### NLP 처리 파이프라인

1. **기사 수집**: 5분 간격으로 RSS 피드 수집
2. **중복 제거**: 동일 사건에 대한 복수 기사 클러스터링
3. **이벤트 추출**: FinBERT/LLM으로 금융 이벤트 해당 여부 판정
4. **택소노미 매핑**: 추출된 이벤트를 L1~L6 자동 분류
5. **시장 영향도 추정**: 과거 유사 이벤트의 영향을 기반으로 significance 자동 추정
6. **상관관계 영향 사전 분석**: 해당 이벤트가 어떤 자산군 간 상관관계에 영향을 줄 수 있는지 예측

#### 이벤트 반영 정책

| significance | 반영 방식 | 지연 시간 |
|-------------|----------|----------|
| MAJOR (전쟁 발발, 금리 결정, 대형 파산 등) | 자동 즉시 반영 + 관리자 알림 | 수분~수십분 |
| SIGNIFICANT (주요 경제지표, 제재, 기업 실적 등) | AI 분류 후 관리자 검수 → 반영 | 수시간~1일 |
| MINOR (소규모 이벤트, 개별 기업 뉴스 등) | 일일 배치 처리 | 1일 |

#### 경제 캘린더 통합

최근 5년 구간에서는 정기적 경제 이벤트도 중요한 시장 영향 요인이므로 별도로 관리:
- FOMC 금리 결정 (연 8회)
- CPI/PPI 발표 (월간)
- 고용 보고서 (월간)
- GDP 발표 (분기)
- 어닝 시즌 (분기)
- 주요국 선거 (비정기)

이들은 사전에 달력에 등록되어 있고, 발표 시점에 실제 수치와 시장 반응이 자동으로 채워진다.

## 4. 크로스에셋 디커플링 분석 도구 — 신규

### 4.1 핵심 개념

> **"내 돈이 쉬지 않고 일하게 만든다."**
>
> 커플링된 자산들만 이용하면 하나에서 수익을 내고 나왔을 때 다른 곳도 이미 움직여 버렸다.
> 디커플링되어 독립적으로 움직이거나, 더 좋은 것은 **역방향으로 움직이는 자산 쌍**을 찾아야
> 자산 로테이션으로 연속적인 수익 기회를 포착할 수 있다.

MarketLink의 디커플링 분석 도구는 다음을 제공한다:

1. **상관관계 매트릭스 히트맵**: 주요 자산군 간 실시간 상관계수 시각화
2. **롤링 상관관계 차트**: 특정 자산 쌍의 상관계수가 시간에 따라 어떻게 변화하는지 추적
3. **디커플링 알림**: 역사적으로 커플링되어 있던 자산 쌍이 디커플링되기 시작하면 알림
4. **이벤트 매칭**: 상관관계 급변 시점에 MarketLink의 이벤트 DB에서 원인 이벤트를 자동 매칭
5. **역상관 자산 탐색**: 현재 보유 중인 자산과 역방향으로 움직이는 자산 후보 추천

### 4.3 분석 방법론

#### 롤링 상관계수 (Rolling Correlation)
- 기본 윈도우: 30일, 60일, 90일 (사용자 조절 가능)
- Pearson 상관계수 기본, Spearman 순위 상관계수 옵션

#### 디커플링 감지 알고리즘
```
1. 장기 평균 상관계수 산출 (예: 250거래일 = 약 1년)
2. 현재 롤링 상관계수와 장기 평균의 차이(deviation) 계산
3. deviation이 ±2σ를 초과하면 "디커플링 발생" 신호
4. 해당 시점의 MarketLink 이벤트 DB를 검색하여 원인 후보 매칭
5. 알림 발생: "S&P500과 WTI 원유의 상관관계가 역사적 평균(+0.32)에서
   -0.45로 급변. 원인 후보: [이란 전쟁 발발 (2026-02)]"
```

#### MarketLink 이벤트와의 융합

이것이 기존 상관관계 도구와의 결정적 차별점:

```
┌────────────────────────────────┐
│  디커플링 감지                   │
│  "S&P500 ↔ Oil 역전됨"         │
└──────────┬─────────────────────┘
           ↓ 이벤트 매칭 쿼리
┌────────────────────────────────┐
│  MarketLink 이벤트 DB        │
│  → 이란 전쟁 발발 (L1, 2026-02)│
│  → OPEC 감산 결정 (L2)          │
│  → Fed 금리 동결 (L3)           │
└──────────┬─────────────────────┘
           ↓ 유사 과거 패턴 탐색
┌────────────────────────────────┐
│  역사적 유사 사례                │
│  → 1973 욤키푸르 전쟁 + OPEC 금수│
│    당시 S&P↔Oil: +0.35 → -0.62 │
│    회복까지: 약 8개월             │
│  → 1990 걸프전                  │
│    당시 S&P↔Oil: +0.28 → -0.41 │
│    회복까지: 약 3개월             │
└────────────────────────────────┘
```

사용자는 "지금의 디커플링이 과거 어떤 사례와 유사한지", "그때 어떻게 회복되었는지"를 즉시 확인할 수 있다.

### 4.4 투자자 활용 시나리오

**시나리오: "돈이 쉬지 않고 일하게 만들기"**

1. 사용자가 S&P 500 레버리지 ETF(예: TQQQ)에서 수익 실현
2. MarketLink 디커플링 대시보드 확인
3. 현재 S&P 500과 역상관인 자산 확인 → 예: Gold (+0.71 역상관)
4. Gold 관련 자산으로 자금 이동
5. S&P 500이 조정받는 동안 Gold이 상승 → 추가 수익
6. 디커플링 해소 알림 수신 → 다시 S&P 500으로 로테이션

> 이 시나리오가 MarketLink의 유료 고객 전환의 핵심 동력이 된다.
> "과거를 보는 도구"에서 "투자 판단을 돕는 도구"로의 전환.

### 4.5 DWVA 매매시스템 연동 — 매크로 예측 보조지표 모듈 — 신규

> **상세 설계서**: MarketLink_DWVA_조기경보_프로토콜_v2.md 참조

#### 문제 정의: DWVA에 부재한 "맥락"

DWVA(Dynamic Weighted Value Averaging) 전략은 "하락 시 가중 분할매수"라는 명확한 규칙 기반 시스템이다. 하락이 깊을수록 더 많이 사고, 회복 시 가중치를 줄이다 수익이 나면 매수를 중단한다. **하락장이 기회인 전략**이므로 조기경보가 매수를 방해하면 전략의 존재 이유가 부정된다.

그러나 DWVA에는 **"왜 떨어지는가, 얼마나 깊이/오래 떨어질 것인가"에 대한 정보가 없다.**

#### MarketLink의 올바른 역할: 매수를 방해하지 않고, 배분을 최적화

MarketLink는 DWVA에 **매크로 예측 데이터(MacroForecast)**를 전달한다. "경보"가 아니라 "예측"이다. DWVA는 이 데이터를 기반으로 세 가지를 자율적으로 최적화한다:

**① 가중치 곡선 리셰이핑**: 예상 하락 깊이에 맞춰 가중치 시작점/최대점/최대배수를 동적 조정. 총 투입금은 불변. 초반에 자금을 아끼고 저점에서 더 공격적으로 매집.

**② 매집 기간 동적 조정**: CRASH(급락) → 기간 단축하여 빠르게 투입. GRIND(완만) → 기간 연장하여 천천히 분산. 같은 자금의 시간적 배분을 최적화.

**③ 진입 시점 보정**: "지금이 초반(EARLY_PHASE)인지 저점(DEEP_VALUE)인지"를 과거 유사 사건 데이터로 판단하여 가중치에 보정 계수(0.7x~1.3x)를 적용. 강제가 아닌 권고. 비활성화 가능.

#### 연동 아키텍처 (재설계)

```
MarketLink 실시간 이벤트 감지
    │
    ├─ [B2C] 웹 서비스 사용자에게 차트 + 이벤트 표시
    │
    └─ [내부] MacroForecast 생성
         │
         ▼
    ┌─────────────────────────────────────────────┐
    │  MacroForecast 데이터                         │
    │  ・하락 깊이 예측 (p25/p50/p75)                │
    │  ・바닥 도달 기간 예측                          │
    │  ・하락 형태 (CRASH/GRIND/STEP)                │
    │  ・그룹별 영향 방향 (NEGATIVE/POSITIVE/NEUTRAL) │
    │  ・과거 유사 사건 목록                          │
    └──────────────────┬──────────────────────────┘
                       │
                       ▼
    ┌─────────────────────────────────────────────┐
    │  DWVA DWVAStrategy.apply_macro_forecast()    │
    │                                             │
    │  ① 가중치 곡선 리셰이핑                       │
    │  ② 매집 기간 조정                            │
    │  ③ 진입 시점 보정 계수 적용                    │
    │  ④ 유니버스 추천에 영향 방향 반영              │
    │                                             │
    │  ★ 총 투입금 변경 없음. 매수를 막지 않음.       │
    └─────────────────────────────────────────────┘
```

#### 수혜 그룹 적극 식별

하락 그룹만 보는 것이 아니라, **위기 시 오히려 수혜를 받는 그룹을 적극 식별**하여 포트폴리오 전체 관점에서 기회를 탐색한다.

예: 중동 전쟁 → A_tech/B_broad 하락(매집 기회) + C_energy/D_gold/G_defense 수혜(매수 대상 아님, 수익실현 검토)

#### 상관관계 엔진 공유 구조

DWVA의 `CorrelationAnalyzer`와 MarketLink의 디커플링 분석 도구는 동일한 코어 엔진을 공유한다.

```
┌─────────────────────────────────────────────┐
│  Shared Correlation Engine                   │
│  ・롤링 상관계수, 디커플링 감지, Tail Correlation │
├─────────────────────────────────────────────┤
│     ┌──────────┐      ┌──────────┐          │
│     │MarketLink│      │  DWVA    │          │
│     │(B2C 서비스)│      │(개인 도구) │          │
│     │・웹 시각화 │      │・Universe │          │
│     │・이벤트매칭│      │ Scanner  │          │
│     │・디커플링  │      │・Capital  │          │
│     │ 알림     │      │ Allocator│          │
│     └──────────┘      └──────────┘          │
└─────────────────────────────────────────────┘
```

## 6. 기술 아키텍처 개요

### 6.1 프론트엔드

- **프레임워크**: React (Next.js)
- **차트 라이브러리**: D3.js (커스텀 인터랙티브 차트, 이벤트 오버레이, 인과 체인 네트워크)
- **상관관계 시각화**: D3.js 히트맵 + 롤링 라인 차트
- **반응형**: 모바일 퍼스트

### 6.2 백엔드

- **API**: Node.js / FastAPI
- **DB**: PostgreSQL (이벤트 메타데이터) + TimescaleDB (시계열 지수 + 상관관계 데이터)
- **인증/과금**: Stripe + Supabase Auth
- **실시간**: WebSocket (실시간 이벤트 피드)

### 6.3 데이터 파이프라인

- **지수 데이터**: Macrotrends.net (역사적), Yahoo Finance API (현재, 일별), FRED
- **원자재/채권/통화 데이터**: Yahoo Finance API, FRED, Alpha Vantage
- **GPR/EPU 데이터**: 공개 데이터 월별 자동 수집
- **뉴스 피드**: RSS 수집기 (5분 간격) → NLP 파이프라인
- **경제 캘린더**: FRED 릴리즈 캘린더 API + 수동 등록
- **상관관계 계산**: 일일 배치 (전체) + 실시간 (최근 데이터)

### 6.4 AI/NLP 파이프라인

- **뉴스 이벤트 추출**: FinBERT (금융 특화 감성/이벤트 분류) + LLM (GPT-4o mini / Claude Haiku 등) 택소노미 매핑
- **시장 영향도 추정**: 과거 유사 이벤트 기반 회귀 모델
- **디커플링 원인 매칭**: 이벤트 DB 시맨틱 검색 (벡터 유사도)
- **유사 패턴 탐색**: 상관관계 시계열 DTW (Dynamic Time Warping) 매칭
- **이벤트 요약 생성**: LLM 기반 다국어 생성

## 9. 리스크 및 고려사항

### 9.1 데이터 관련
- **지수 데이터 저작권**: Yahoo Finance API ToS 상업 사용 조건 검토 필요
- **GPR/EPU 데이터 라이선스**: 학술 공개 데이터의 상업 서비스 내 재배포 허가 확인
- **실시간 뉴스 저작권**: RSS 피드 수집 시 원문 재배포 금지. NLP 요약만 사용
- **금융 데이터 지연**: 무료 데이터는 15~20분 지연 — 실시간 분석에 한계. 유료 API 필요 시점 판단

### 9.2 서비스 관련
- **이벤트 큐레이션 편향**: 미국 중심 편향 불가피. 글로벌 사건의 미국 시장 영향 관점으로 통일
- **정치적 중립성**: 사건 서술에서 가치 판단 배제
- **자동 감지 오류**: NLP 오분류 리스크 → MAJOR는 자동, SIGNIFICANT 이하는 검수 프로세스로 완화
- **투자 조언 규제**: 디커플링 분석이 "투자 조언"으로 간주될 수 있는 법적 리스크 → "정보 제공 도구"로 명확히 면책 고지

### 9.3 경쟁 관련
- TradingView 등 기존 차트 플랫폼이 유사 기능 추가 가능성
- Portfolio Visualizer 등 상관관계 도구와의 경쟁
- **방어 전략**: 세 축의 융합(이벤트 매핑 + 실시간 감지 + 디커플링)이 핵심 해자. 개별 기능은 모방 가능하나 융합은 어려움

### 9.4 기술 관련
- **NLP 파이프라인 비용**: LLM API 호출 비용이 뉴스 볼륨에 비례 → 비용 최적화 필요 (경량 모델 1차 필터링 → LLM 2차 분류)
- **상관관계 계산 부하**: 자산 수 N에 대해 N(N-1)/2 쌍의 상관관계 → 자산 30개면 435쌍. 일별 배치는 문제없으나 실시간은 설계 주의
