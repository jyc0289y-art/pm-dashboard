## 2. 실제 장비 검증 결과

### 2.1 검증 환경

- 장비: Ludlum Model 3030-2, FW 39015N02
- 연결: USB-Serial 어댑터 → `/dev/tty.usbserial-A9RTCS37`
- PC: macOS (Apple M5), Python 3.12.6, pyserial 3.5

### 2.2 검증 항목 실행 결과

| 검증 항목 | 결과 | 비고 |
|-----------|------|------|
| 1. 기본 연결 (F, RD, D4) | 통과 | FW=39015N02, D4=BQ |
| 3. 파라미터 읽기 | 통과 | 아래 표 참조 |
| 6. C5 카운트 | 통과 | 2단계 응답 발견 (아래 상세) |

검증 항목 2(모드별 RR), 4(BG/XT 반영), 5(Printer Mode)는 장비 설정 변경이 필요하여 미실행. 추후 LMI 소프트웨어로 설정 변경 후 실행 가능.

### 2.3 실제 장비 파라미터 (대리점 세팅)

| 파라미터 | 명령 | 응답 | 환산값 |
|----------|------|------|--------|
| Alpha 효율 | RE1 | 379 | 37.9% |
| Beta 효율 | RE2 | 328 | 32.8% |
| Alpha Background | RB1 | 000000 | 0.0 cpm |
| Beta Background | RB2 | 000390 | 39.0 cpm |
| B→A Crosstalk | RX1 | 024 | 2.4% |
| A→B Crosstalk | RX2 | 000 | 0.0% |
| BG Subtract | B2 | ON | - |
| Crosstalk Corr. | X2 | ON | - |
| High Voltage | RV | 0600 | 600 Vdc |
| Display Mode | D4 | BQ | Becquerel |

### 2.4 매뉴얼과 다른 실제 동작 (중요)

#### C4/C5 2단계 응답

**매뉴얼 기술**: C4/C5는 카운트 시작 + 완료 시 결과 자동 반환

**실제 동작**:
1. 명령 전송 → **즉시 "OK" 반환** (카운트 시작 확인)
2. 카운트 진행 (60초)
3. 카운트 완료 → **RR 포맷 결과 자동 반환**

**대응**: `serial_comm.py`의 `_count_and_wait()` 메서드에서 Phase 1(OK)을 읽고 버린 뒤, Phase 2(결과)를 대기하도록 구현.

#### RR 모드 코드

**매뉴얼 기술 (Appendix A)**: 3030-2에서 모드 코드 B(Becquerel), P(CPS) 사용

**실제 동작**: D4="BQ"인 상태에서도 **RR 응답의 모드 코드는 "D"**(DPM). alpha/beta 값은 항상 **raw integer scaler count**.

**해석**: RR은 base 3030 프로토콜을 그대로 사용. Appendix A의 모드 코드(B/P)는 RS(저장 샘플) 또는 LCD 표시에만 적용되는 것으로 추정. 우리 소프트웨어는 항상 raw count로부터 자체 계산하므로 문제 없음.

## 3. 설계 결정 사항

### 3.1 장비 설정값 변경 금지

**배경**: 대리점(동양이엔피테크)이 교정한 설정값을 보호하기 위해, 직원이 실수로 설정을 변경하는 것을 방지.

**구현**:
- READ 명령(R*, F, D4, B2, X2)과 COUNT 명령(C4, C5, C2, C3)만 사용
- SET 명령(D0-D3, B0/B1, X0/X1, 모든 S*) 래퍼 미제공
- CLI sniff 모드에서 SET 명령 입력 시 경고 표시 + 확인 요청

### 3.2 모드 크롤링 대체

모드 변경(D0-D3) 금지로 Phase 2 모드 크롤링은 불가능. 대신:
- RR에서 항상 raw scaler count를 수신
- `calculator.py`에서 CPM, net CPM, Bq를 자체 계산
- 장비 파라미터(효율, BG, crosstalk)는 READ 명령으로 읽어와서 적용

이 방식이 데이터 무결성 관점에서도 우수: 효율 재교정 시 과거 raw count로 소급 재계산 가능.

### 3.3 MDA 계산

매뉴얼 p.32 공식 구현. p.42 테이블과 대조 검증 시 **Tb=1분** 사용이 정확함을 확인 (매뉴얼 p.29: "Background count time is always one minute").

## 5. 사용법

```bash
cd ludlum-3030-logger

# 테스트 실행
python3 -m unittest discover tests -v

# 스니퍼 (장비 연결 필요)
python3 tools/sniffer.py                          # 자동 포트 탐색
python3 tools/sniffer.py --port /dev/tty.usbserial-A9RTCS37
python3 tools/sniffer.py --mock                   # mock 모드
python3 tools/sniffer.py --verify                 # 검증 스위트

# CLI (장비 연결 필요)
python3 -m src.cli
python3 -m src.cli --port /dev/tty.usbserial-A9RTCS37
python3 -m src.cli --mock                         # mock 모드

# CLI 명령
> count       # 측정 시작 (장비 설정 카운트 시간)
> count1      # 1분 측정
> params      # 장비 파라미터 표시
> refresh     # 파라미터 재읽기
> history 10  # 최근 10건 이력
> edit 1      # 측정 #1 메타데이터 편집
> export      # CSV 내보내기
> search DU   # 시료명/측정자 검색
> sniff       # raw 명령 모드
> quit        # 종료
```
