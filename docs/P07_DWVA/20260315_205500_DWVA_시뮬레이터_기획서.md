## 1. 프로젝트 개요

### 1.1 목적

DWVA(Dynamic Weighted Value Averaging, 동적 가중 분할매수) 전략의 **시뮬레이션부터 실전 자동 집행까지** 전체 파이프라인을 구축한다.

전체 워크플로우:

```
[시뮬레이터로 전략 설계]  →  [파라미터 확정]  →  [시장 모니터링 대기]  →  [트리거 발동]  →  [API 자동 매수 집행]
     ↓                          ↓                     ↓                    ↓                    ↓
 백테스팅/Monte Carlo      진입시점/가중치곡선      실시간 가격 감시      하락률 조건 충족     KIS/업비트 주문
 리스크/수익 트레이드오프   매집기간/리스크한도      고점 대비 하락률      자동으로 매수 개시   가중치 자동 계산
 파라미터 최적화           기대수익률 범위 설정     Telegram 알림 대기    사전 확정 규칙대로   매수중단까지 자율운영
```

세 가지 모듈로 구성된다:
- **시뮬레이터**: 역사적 백테스팅 + Monte Carlo 시뮬레이션으로 파라미터 사전 검증
- **모니터링**: 실시간 가격 감시 → 진입 트리거 감지 → 알림
- **자동 집행**: KIS API(주식) 또는 업비트 API(암호화폐)를 통한 DWVA 규칙 기반 자동 매수

### 1.2 왜 시뮬레이터가 필요한가

DWVA 전략의 결정적 변수들(진입 시점, 가중치 곡선, 매집 기간, 총 투입금 등)이 최종 성과에 미치는 영향이 크다. 하지만 이 변수들의 조합이 무수히 많아서 직관만으로는 최적 설정을 찾기 어렵다.

예를 들어:
- 고점 대비 -20%에서 진입하면 안전하지만 매집 기회를 놓칠 수 있고
- 고점 대비 -10%에서 진입하면 기회는 많지만 추가 하락 리스크가 크다
- 가중치를 너무 공격적으로 설정하면 초반에 자금이 소진되고
- 너무 보수적이면 저점에서 충분히 매수하지 못한다

이 트레이드오프를 **숫자로 보여주는 것**이 시뮬레이터의 목적이다.

### 1.3 기존 사례와의 차이

| 기존 도구 | 한계 | DWVA 시뮬레이터 차별점 |
|-----------|------|----------------------|
| DCA 백테스터 (GitHub 54개+) | 고정 금액 정기 매수만 지원 | 하락 깊이 연동 동적 가중치 |
| Value Averaging 계산기 | 목표 가치 경로 기반, 하락장 특화 아님 | 평단가 기준 매수 강도 자동 전환 |
| Monte Carlo 포트폴리오 시뮬레이터 | 자산배분 최적화 목적 | 단일 전략의 파라미터 최적화 목적 |
| backtesting.py / Backtrader | 범용 프레임워크, DWVA 로직 없음 | DWVA 전용 로직 + 시나리오 생성기 |

## 2. DWVA 전략 정형화

### 2.1 전략 파라미터 정의

시뮬레이터에서 조절 가능한 모든 파라미터를 정의한다.

#### 진입 조건 파라미터

| 파라미터 | 기호 | 설명 | 기본값 | 범위 |
|---------|------|------|--------|------|
| 진입 트리거 | `entry_trigger` | 고점 대비 하락률 (%) | -15% | -5% ~ -40% |
| 고점 산정 기간 | `peak_lookback` | 최근 N일 고점 기준 | 60일 | 20 ~ 200일 |
| 진입 확인 기간 | `entry_confirm_days` | 트리거 후 N일 유지 확인 | 3일 | 0 ~ 10일 |

#### 매수 파라미터

| 파라미터 | 기호 | 설명 | 기본값 | 범위 |
|---------|------|------|--------|------|
| 총 투입 자금 | `total_capital` | 전체 매수 예산 | 1000만원 | 사용자 설정 |
| 매집 기간 | `accumulation_months` | 기본 매집 기간 (개월) | 6개월 | 2 ~ 12개월 |
| 기본 매수 금액 | `base_amount` | 매수 1회당 기본 금액 | 총자금 / (매집기간 × 매수빈도) | 자동 계산 |
| 매수 빈도 | `buy_frequency` | 매수 간격 | 주 1회 | 일/주/격주/월 |

#### 가중치 파라미터 (핵심)

| 파라미터 | 기호 | 설명 | 기본값 | 범위 |
|---------|------|------|--------|------|
| 가중치 함수 유형 | `weight_type` | 가중치 곡선 형태 | linear | linear/exponential/step |
| 최대 가중치 배수 | `max_weight` | 기본 금액 대비 최대 배수 | 3.0x | 1.0 ~ 10.0x |
| 가중치 기준 | `weight_basis` | 가중치 산정 기준 | drawdown | drawdown/vs_avg_cost |
| 하락 깊이 기준점 | `weight_base_drop` | 이 하락률부터 가중치 증가 시작 | -15% | -5% ~ -30% |
| 가중치 최대점 | `weight_max_drop` | 이 하락률에서 최대 가중치 도달 | -40% | -20% ~ -60% |

#### 평단가 연동 파라미터

| 파라미터 | 기호 | 설명 | 기본값 | 범위 |
|---------|------|------|--------|------|
| 평단 도달 시 가중치 | `at_avg_weight` | 현재가 = 평단가일 때 가중치 | 1.0x | 0.5 ~ 1.5x |
| 매수 중단 수익률 | `stop_buy_profit` | 이 수익률 이상이면 매수 중단 | +35% | +20% ~ +50% |
| 가중치 축소 시작 수익률 | `reduce_weight_at` | 이 수익률부터 가중치 축소 시작 | 0% (평단 도달) | -5% ~ +10% |

#### 자금 보전 파라미터

| 파라미터 | 기호 | 설명 | 기본값 | 범위 |
|---------|------|------|--------|------|
| 시간 연장 트리거 | `extend_trigger_months` | 매집기간 초과 시 연장 발동 | 매집기간 × 1.0 | 사용자 설정 |
| 연장 배수 | `extend_multiplier` | 남은 자금의 소진 기간 연장 | 2.0x | 1.5 ~ 4.0x |
| 최소 1회 매수금액 | `min_buy_amount` | 이 금액 미만이면 매수 건너뜀 | 5만원 | 사용자 설정 |

#### 매도 파라미터 (선택 — 매도 규칙 체계화 시)

| 파라미터 | 기호 | 설명 | 기본값 | 범위 |
|---------|------|------|--------|------|
| 목표 수익률 매도 | `target_profit_sell` | 이 수익률 도달 시 분할 매도 시작 | +50% | +30% ~ +100% |
| Trailing Stop | `trailing_stop` | 고점 대비 하락 시 매도 | -10% | -5% ~ -20% |
| 분할 매도 비율 | `sell_portion` | 매도 시 보유량의 N% 매도 | 30% | 10% ~ 100% |

### 2.2 가중치 함수 정의

세 가지 가중치 곡선을 지원한다. `drop`은 현재 하락률(고점 대비), `base_drop`은 가중치 시작점, `max_drop`은 최대점이다.

**Linear (선형)**:
```
progress = (drop - base_drop) / (max_drop - base_drop)   # 0~1 사이 클램핑
weight = 1.0 + (max_weight - 1.0) × progress
```

**Exponential (지수)**:
```
progress = (drop - base_drop) / (max_drop - base_drop)
weight = 1.0 + (max_weight - 1.0) × progress²
```
→ 초반 완만, 하락 깊어질수록 급격히 증가. 보수적이면서 저점에서 공격적.

**Step (단계)**:
```
-15% ~ -25%: 1.5x
-25% ~ -35%: 2.0x
-35% ~ -45%: 2.5x
-45% 이하:   3.0x
```
→ 단계별 명확한 가중치. 직관적.

### 2.3 평단가 연동 로직

```
현재가 < 평단가:
    → 가중치 = 하락깊이 기반 가중치 (2.2 공식)
    → 매수 지속

현재가 ≥ 평단가 AND 수익률 < stop_buy_profit:
    → 가중치 = max(at_avg_weight × (1 - 수익률/stop_buy_profit), min_weight)
    → 수익률 높아질수록 가중치 축소

수익률 ≥ stop_buy_profit:
    → 매수 완전 중단
```

### 2.4 시간 연장 로직

```
현재 경과 기간 > accumulation_months AND 남은 자금 > 0:
    → 남은 자금을 (남은 자금 / min_buy_amount × extend_multiplier) 회에 걸쳐 분배
    → 매수 1회당 금액 재계산
    → 최악의 경우에도 자금이 즉시 고갈되지 않음
```

## 3. 시뮬레이터 아키텍처

### 3.1 두 가지 모드

#### Mode A: 역사적 백테스팅 (Historical Backtest)

실제 과거 가격 데이터를 사용하여 DWVA 전략이 어떤 성과를 냈을지 계산한다.

**입력**:
- 종목 티커 + 기간 (예: SOXL, 2022-01-01 ~ 2023-06-30)
- DWVA 파라미터 세트

**과정**:
1. yfinance/pykrx로 과거 OHLCV 데이터 로드
2. 고점 산정 → 진입 트리거 감지
3. 매수 시뮬레이션: 매수 빈도마다 가중치 계산 → 매수량 결정
4. 일별 포트폴리오 가치, 평단가, 투입금 추적

**출력**:
- 누적 수익률, 최종 평단가, 투입 대비 수익
- 매수 타임라인 시각화 (가격 차트 위에 매수 시점/금액 표시)
- 가중치 변화 추이
- 최대 미실현 손실 (MDD)
- 매수 중단 시점 및 이유

**활용 사례**:
- "2022년 SOXL 하락장에서 이 파라미터로 했으면 결과가 어땠을까?"
- "2020년 코로나 급락에서는?"
- "2018년 BTC 하락장에서는?"

#### Mode B: Monte Carlo 시뮬레이션 (Scenario Generation)

가상의 하락장 시나리오를 수천 개 생성하여 전략의 확률적 성과 분포를 예측한다.

**입력**:
- DWVA 파라미터 세트
- 하락장 시나리오 파라미터 (아래 3.2 참조)
- 시뮬레이션 횟수 (기본 10,000회)

**과정**:
1. 하락장 가격 경로를 확률적으로 생성 (GBM 또는 역사적 리샘플링)
2. 각 경로에 DWVA 전략을 적용하여 성과 계산
3. 10,000개 결과의 분포 분석

**출력**:
- 수익률 분포: 중앙값, 평균, 5/25/75/95 백분위
- 최대 손실 분포: "최악의 5% 시나리오에서 손실은 N% 이내"
- 자금 소진 확률: "매집 기간 내 자금이 바닥날 확률 N%"
- 목표 수익률 달성 확률: "수익률 30% 이상 달성 확률 N%"
- 매집 기간 초과 확률: "6개월 안에 끝나지 않을 확률 N%"
- Equity Curve 팬 차트: 수천 개 시뮬레이션 경로를 겹쳐서 시각화

### 3.2 하락장 시나리오 생성기

Monte Carlo용 가상 하락장 가격 경로를 생성한다.

#### 방법 1: Geometric Brownian Motion (GBM)

수학적으로 가격 경로를 생성한다. 변동성과 드리프트(추세)를 조절하여 다양한 하락장을 시뮬레이션한다.

**조절 가능한 파라미터**:

| 파라미터 | 설명 | 예시 |
|---------|------|------|
| 최대 하락폭 | 고점 대비 최대 하락률 | -30%, -50%, -70% |
| 하락 기간 | 고점→저점까지 기간 | 2개월, 6개월, 12개월 |
| 하락 형태 | 급락/완만/V자/L자/W자 | 선택 가능 |
| 변동성 | 일일 변동성 | 1%, 3%, 5% |
| 회복 기간 | 저점→원래 가격까지 기간 | 3개월, 6개월, 회복 없음 |
| 회복 형태 | V자 급반등 / 완만 회복 / 횡보 후 회복 | 선택 가능 |

#### 방법 2: 역사적 리샘플링 (Historical Resampling)

실제 역사적 하락장 데이터를 기반으로 가격 경로를 생성한다. 실제 발생했던 패턴을 재현하므로 현실성이 높다.

**내장 시나리오 템플릿**:

| 시나리오 | 기반 데이터 | 최대 하락 | 기간 | 특징 |
|---------|-----------|----------|------|------|
| 코로나 급락 | 2020.02~2020.04 | -35% (S&P) | 1개월 | 급락 + V자 회복 |
| 2022 인플레 하락 | 2022.01~2022.10 | -77% (SOXL) | 10개월 | 완만하고 긴 하락 |
| 2018 암호화폐 겨울 | 2018.01~2018.12 | -84% (BTC) | 12개월 | 급락 + L자 횡보 |
| 2008 금융위기 | 2008.09~2009.03 | -57% (S&P) | 6개월 | 급락 + 느린 회복 |
| 닷컴 버블 | 2000.03~2002.10 | -78% (NASDAQ) | 30개월 | 매우 긴 하락 |
| 커스텀 | 사용자 정의 | 사용자 설정 | 사용자 설정 | 파라미터 직접 입력 |

각 템플릿의 일별 수익률 분포를 추출하고, 이를 리샘플링하여 "비슷하지만 다른" 가격 경로를 대량 생성한다.

### 3.3 파라미터 최적화

시뮬레이터의 핵심 기능 중 하나. 주어진 시나리오(또는 시나리오 세트)에서 최적의 파라미터 조합을 탐색한다.

**최적화 대상 (사용자가 선택)**:
- 최대 수익률 (공격적)
- 최소 MDD (방어적)
- 최대 Sharpe Ratio (균형)
- 자금 소진 확률 최소화 (안전 우선)

**최적화 방법**:
- Grid Search: 파라미터 범위를 격자로 나누어 전수 탐색. 직관적이지만 느림
- Random Search: 무작위로 N개 조합을 탐색. Grid보다 효율적
- Optuna (Bayesian Optimization): 이전 결과를 참고하여 유망한 영역을 집중 탐색. 가장 효율적

**출력**:
- 최적 파라미터 세트 + 해당 성과 지표
- 파라미터 민감도 분석: "이 파라미터를 10% 바꾸면 결과가 얼마나 변하는가?"
- 파라미터 히트맵: 두 파라미터의 조합별 성과를 색상으로 시각화

## 4. 성과 지표

### 4.1 핵심 지표

| 지표 | 설명 | 계산 방법 |
|------|------|----------|
| **총 수익률** | 투입 대비 최종 수익 | (최종 포트폴리오 가치 - 총 투입금) / 총 투입금 |
| **평균 매수단가** | 가중 평균 매수가 | 총 투입금 / 총 매수 수량 |
| **최대 미실현 손실** | 매수 기간 중 최악의 평가손실 | min(일별 평가액 - 일별 투입 누적) |
| **자금 소진 시점** | 투입 자금이 바닥나는 날 | 누적 투입 = 총 자금 되는 시점 |
| **투입 효율** | 실제 투입된 자금 비율 | 실제 투입 / 총 예산 (매수 중단 시 100% 미만) |
| **평단 도달 일수** | 매수 시작 후 BEP까지 기간 | 현재가 ≥ 평단가 최초 도달일 |
| **목표 수익 도달 일수** | 수익률 30%+ 도달까지 기간 | — |

### 4.2 Monte Carlo 전용 지표

| 지표 | 설명 |
|------|------|
| **수익률 중앙값 (P50)** | 시뮬레이션 중앙값 |
| **수익률 P5 / P95** | 최악의 5% ~ 최선의 5% 범위 |
| **목표 수익률 달성 확률** | 수익률 ≥ X% 되는 시뮬레이션 비율 |
| **원금 손실 확률** | 최종 평가액 < 총 투입금인 비율 |
| **자금 조기 소진 확률** | 매집 기간 내 자금 바닥나는 비율 |
| **시간 연장 발동 확률** | 매집 기간을 초과하는 비율 |

### 4.3 벤치마크 비교

모든 시뮬레이션 결과를 아래 벤치마크와 비교한다:
- **일시불 투자 (Lump Sum)**: 진입 시점에 총 투입금 일시 투입
- **단순 DCA**: 동일 금액 정기 매수 (가중치 없음)
- **Buy & Hold**: 기간 시작 시점에 매수 후 보유

## 5. 시각화

### 5.1 매수 타임라인 차트

가격 차트 위에 매수 시점과 금액을 함께 표시한다.
- X축: 시간
- Y축 (좌): 가격 (캔들 또는 라인)
- Y축 (우): 매수 금액 (바 차트)
- 평단가 수평선 (점선)
- 손익 구간 색상: 손실 구간 빨강, 수익 구간 초록
- 매수 중단 시점 표시

### 5.2 가중치 변화 추이

시간에 따른 매수 가중치 변화를 보여준다.
- 가격 하락에 따른 가중치 증가 → 평단가 접근 시 축소 → 수익 구간 매수 중단

### 5.3 Equity Curve 팬 차트 (Monte Carlo)

수천 개의 시뮬레이션 경로를 겹쳐서 표시한다.
- 중앙선: P50 (중앙값)
- 짙은 밴드: P25 ~ P75
- 연한 밴드: P5 ~ P95
- 개별 경로: 반투명 선으로 배경에 표시

### 5.4 파라미터 민감도 히트맵

두 파라미터의 조합별 성과를 2D 히트맵으로 표시한다.
- 예: X축 = 진입 트리거 (-10% ~ -30%), Y축 = 최대 가중치 (1x ~ 5x), 색상 = Sharpe Ratio

### 5.5 시나리오별 비교 대시보드

여러 하락장 시나리오에서의 전략 성과를 나란히 비교한다.
- 행: 시나리오 (코로나, 2022 인플레, 2018 BTC 등)
- 열: 핵심 지표 (수익률, MDD, 평단 도달 일수 등)

```python
class DWVAStrategy:
    """DWVA 전략 로직. 시뮬레이터와 실전 집행 모두에서 공용으로 사용한다."""
    
    def __init__(self, params: DWVAParams):
        """파라미터로 전략 초기화."""
        
    def should_enter(self, price: float, peak: float) -> bool:
        """진입 조건 충족 여부 판단."""
        
    def calculate_weight(self, current_price: float, peak: float, avg_cost: float) -> float:
        """현재 상태에 따른 매수 가중치 계산."""
        
    def calculate_buy_amount(self, weight: float, remaining_capital: float) -> float:
        """실제 매수 금액 계산 (자금 보전 로직 포함)."""
        
    def should_stop_buying(self, unrealized_profit_pct: float) -> bool:
        """매수 중단 조건 확인."""


class BacktestEngine:
    """역사적 데이터로 DWVA 전략 백테스팅."""
    
    def run(self, price_data: pd.Series, strategy: DWVAStrategy) -> BacktestResult:
        """일별 시뮬레이션 실행."""


class MonteCarloEngine:
    """Monte Carlo 시뮬레이션 엔진."""
    
    def run(self, strategy: DWVAStrategy, scenario: ScenarioConfig, 
            n_simulations: int = 10000) -> MonteCarloResult:
        """N회 시뮬레이션 실행 및 통계 분석."""


class ScenarioGenerator:
    """하락장 가격 경로 생성기."""
    
    def generate_gbm(self, params: GBMParams, n_paths: int) -> np.ndarray:
        """GBM 기반 가격 경로 생성."""
        
    def generate_historical(self, template: str, n_paths: int) -> np.ndarray:
        """역사적 리샘플링 기반 가격 경로 생성."""


class Optimizer:
    """파라미터 최적화."""
    
    def optimize(self, engine: BacktestEngine | MonteCarloEngine,
                 param_ranges: dict, objective: str) -> OptimizationResult:
        """최적 파라미터 탐색."""
```

```python
class DWVASession:
    """하나의 DWVA 매집 사이클 전체를 관리하는 세션 객체.
    
    생명주기: WATCHING → TRIGGERED → ACCUMULATING → REDUCING → COMPLETED
    
    - WATCHING: 시장 모니터링 중. 진입 트리거 대기.
    - TRIGGERED: 트리거 발동. 확인 기간(entry_confirm_days) 경과 대기.
    - ACCUMULATING: 매집 진행 중. 가중치 매수 실행.
    - REDUCING: 평단가 도달. 가중치 축소하며 매수.
    - COMPLETED: 매수 중단 조건 도달 또는 자금 소진. 매도 대기.
    """
    
    def __init__(self, session_id: str, strategy: DWVAStrategy, 
                 live_config: LiveConfig):
        """세션 생성. 확정된 파라미터와 실전 설정을 받는다."""
    
    @property
    def state(self) -> SessionState:
        """현재 세션 상태."""
    
    def update(self, current_price: float, timestamp: datetime) -> Optional[BuyOrder]:
        """가격 업데이트를 받아 매수 판단을 수행한다.
        매수가 필요하면 BuyOrder를 반환, 아니면 None."""
    
    def get_status(self) -> SessionStatus:
        """현재 세션 상태 요약 (평단가, 투입금, 수익률, 남은 자금 등)."""
    
    def save(self, path: str):
        """세션 상태를 파일로 저장 (프로세스 재시작 시 복구용)."""
    
    @classmethod
    def load(cls, path: str) -> 'DWVASession':
        """저장된 세션 상태에서 복구."""


class PriceMonitor:
    """실시간 가격 모니터링 + 진입 트리거 감지.
    
    WebSocket 또는 REST polling으로 가격을 수신하고,
    등록된 세션들의 진입 조건을 실시간으로 체크한다.
    """
    
    def __init__(self, market: str):  # 'kis' or 'upbit'
        """거래소별 모니터 초기화."""
    
    def watch(self, symbol: str, session: DWVASession):
        """종목+세션을 감시 목록에 등록."""
    
    def start(self):
        """모니터링 시작 (비동기 루프)."""
    
    def on_price_update(self, symbol: str, price: float):
        """가격 업데이트 콜백. 등록된 세션들에 전파."""


class BaseExecutor(ABC):
    """거래소 주문 집행 추상 클래스."""
    
    @abstractmethod
    def buy_market(self, symbol: str, amount: float) -> OrderResult:
        """시장가 매수. amount는 원화(KRW) 금액."""
    
    @abstractmethod
    def get_balance(self) -> dict:
        """잔고 조회."""
    
    @abstractmethod  
    def get_current_price(self, symbol: str) -> float:
        """현재가 조회."""


class KISExecutor(BaseExecutor):
    """한국투자증권 KIS API를 통한 주문 집행.
    
    모의투자 / 실전투자 모드를 config로 전환.
    해외 ETF의 경우 미국 장 시간 자동 대응.
    """
    
    def __init__(self, config: KISConfig):
        """API 인증 초기화. Access Token 자동 발급/갱신."""
    
    def buy_market(self, symbol: str, amount: float) -> OrderResult:
        """국내: 시장가 매수 / 해외: 지정가 매수 (시장가 미지원 종목 대응)."""
    
    def is_market_open(self, market: str = 'domestic') -> bool:
        """장 운영 시간 확인 (국내/해외/서머타임 자동 판단)."""


class UpbitExecutor(BaseExecutor):
    """업비트 API를 통한 주문 집행. 24시간 운영."""
    
    def __init__(self, config: UpbitConfig):
        """pyupbit 인증 초기화."""
    
    def buy_market(self, symbol: str, amount: float) -> OrderResult:
        """시장가 매수."""


class BuyScheduler:
    """DWVA 매수 스케줄 관리.
    
    매수 빈도(일/주/격주/월)에 따라 매수 시점을 관리하고,
    장 운영 시간을 고려하여 적절한 타이밍에 매수를 실행한다.
    """
    
    def __init__(self, executor: BaseExecutor, session: DWVASession):
        """스케줄러 초기화."""
    
    def start(self):
        """스케줄링 시작."""
    
    def execute_scheduled_buy(self):
        """스케줄된 매수 실행: 세션에 가격 전달 → 매수 판단 → 주문 실행."""


class Reporter:
    """알림 및 보고서."""
    
    def send_buy_notification(self, order: OrderResult, session: DWVASession):
        """매수 체결 시 Telegram 알림."""
    
    def send_daily_report(self, session: DWVASession):
        """일일 상태 보고 (수익률, 평단가, 남은 자금, 다음 매수 예정)."""
    
    def send_trigger_alert(self, symbol: str, drop_pct: float):
        """진입 트리거 발동 알림."""
    
    def send_error_alert(self, error: Exception):
        """오류 발생 시 긴급 알림."""
```

## 8. 실전 자동 집행 상세 설계

### 8.1 전체 운영 흐름

```
┌─────────────────────────────────────────────────────────────────┐
│                        DWVA 실전 운영 흐름                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  [1단계: 사전 설계]                                               │
│  시뮬레이터로 종목/파라미터 확정                                     │
│  → 진입 트리거, 가중치 곡선, 매집 기간, 리스크 한도, 기대 수익률       │
│           ↓                                                     │
│  [2단계: 세션 생성]                                               │
│  확정된 파라미터로 DWVASession 생성                                 │
│  → live_config.yaml에 종목/API모드/알림 설정                       │
│  → 세션 상태: WATCHING                                           │
│           ↓                                                     │
│  [3단계: 모니터링 대기]                                            │
│  PriceMonitor가 실시간 가격 감시                                   │
│  → 고점 대비 하락률 추적                                           │
│  → 진입 트리거 도달 시 Telegram 알림                                │
│  → entry_confirm_days 경과 확인                                   │
│  → 세션 상태: WATCHING → TRIGGERED → ACCUMULATING                │
│           ↓                                                     │
│  [4단계: 자동 매수 집행]                                           │
│  BuyScheduler가 매수 빈도에 따라 자동 실행                           │
│  → DWVAStrategy.calculate_weight() 호출                          │
│  → 가중치 × 기본금액 = 매수금액 산출                                │
│  → KISExecutor / UpbitExecutor로 실제 주문                        │
│  → PositionTracker로 평단가/수익률 실시간 갱신                      │
│  → 매수 체결마다 Telegram 알림                                     │
│           ↓                                                     │
│  [5단계: 동적 전환]                                               │
│  현재가 ≥ 평단가 → 가중치 축소 시작 (REDUCING)                      │
│  수익률 ≥ stop_buy_profit → 매수 완전 중단 (COMPLETED)             │
│  매집기간 초과 → 시간 연장 발동 (자금 재배분)                         │
│           ↓                                                     │
│  [6단계: 매도 대기]                                               │
│  매수 완료 후 보유. 매도는 별도 규칙 또는 수동 판단.                   │
│  → Trailing Stop / 목표 수익률 매도 (설정 시)                       │
│  → 일일 보고서로 수익률 모니터링                                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 8.2 DWVA 세션 생명주기

| 상태 | 설명 | 전환 조건 |
|------|------|----------|
| **WATCHING** | 시장 모니터링 중. 진입 트리거 대기 | 고점 대비 하락률 ≥ entry_trigger |
| **TRIGGERED** | 트리거 발동. 확인 기간 대기 | entry_confirm_days 연속 유지 |
| **ACCUMULATING** | 매집 진행. 가중치 매수 실행 | 현재가 < 평단가 구간 |
| **REDUCING** | 가중치 축소 매수. 평단가 상회 구간 | 현재가 ≥ 평단가 AND 수익률 < stop_buy_profit |
| **COMPLETED** | 매수 중단. 보유 중 | 수익률 ≥ stop_buy_profit OR 자금 소진 |
| **PAUSED** | 일시 정지 (수동 개입 또는 오류) | 사용자 명령 또는 시스템 오류 |

세션 상태는 `sessions/active/` 디렉토리에 JSON으로 영속 저장된다. 프로세스 재시작 시 자동으로 마지막 상태에서 복구된다.

## 9. 종목 유니버스 및 자동 추천 시스템

### 9.1 설계 원칙

DWVA 전략은 하락장에서만 매수하므로, 모든 종목이 동시에 하락하면 자금이 한꺼번에 몰리고 동시에 상승하면 자금이 놀게 된다. **서로 다른 시점에 하락하는(디커플링된) 종목들**을 확보해야 자금이 항상 일한다.

디커플링을 만드는 3대 축:
- **지역 축**: 미국, 중국, 유럽, 신흥국은 경기 사이클이 다름
- **섹터 축**: 기술, 에너지, 헬스케어, 금융, 방산은 서로 다른 경제적 힘에 반응
- **자산 축**: 주식, 금, 원자재는 구조적으로 다른 요인에 반응

이 세 축을 조합하여 "어떤 거시경제 환경에서도 어딘가는 하락 매집 기회가 있는" 유니버스를 구축한다.

### 9.2 종목 선별 기준

DWVA에 적합한 종목의 필수 조건:

| 조건 | 이유 | 필터 |
|------|------|------|
| **지수 추종** | 개별 기업 리스크 없음. 하락 후 회복 확실성 최대 | ETF만 대상 |
| **충분한 유동성** | 슬리피지 최소화, 시장가 주문 가능 | 일평균 거래대금 $10M+ |
| **충분한 역사** | 백테스팅/Monte Carlo에 최소 3년치 데이터 필요 | 상장 3년+ |
| **KIS API 매매 가능** | 한국투자증권에서 거래 가능해야 함 | 해외주식 거래 가능 종목 |
| **레버리지 버전 존재 (우대)** | DWVA 수익 극대화 가능. 필수는 아님 | 2x/3x ETF 존재 여부 |

### 9.3 전체 유니버스 (9개 그룹)

#### 그룹 A — 미국 기술/반도체
- **움직이는 힘**: AI/반도체 투자 사이클, 금리, 기술 혁신
- 1x: QQQ (나스닥100), SOXX (반도체), XLK (기술섹터)
- 3x: TQQQ (나스닥 3x), SOXL (반도체 3x)
- **DWVA 적합성**: 변동성 크고 하락 후 강한 반등. 매집 효과 극대화

#### 그룹 B — 미국 광범위 시장

### 9.4 DWVA 시나리오별 자금 흐름 예시

**시나리오: 금리 인상 → 기술주 하락기 (2022년형)**
```
A 기술/반도체: ▼▼▼ 급락 → DWVA 매집 시작 (자금 투입)
B 광범위:      ▼▼   하락 → DWVA 매집 시작 (자금 투입, A보다 적은 가중치)
C 에너지:      ▲▲▲ 급등 → DWVA 대기 (이전 사이클 수익 실현 중)
D 금:          ▲    상승 → DWVA 대기
E 중국:        ▼▼   하락 → DWVA 매집 시작 (독립적 원인: 중국 규제)
F 헬스케어:    →    횡보 → DWVA 대기
G 방산:        ▲▲   상승 → DWVA 대기 (우크라이나 전쟁)
```
→ 자금이 A, B, E에 분산 투입. C, D, G에서 수익 실현 중. **자금이 쉬지 않음.**

**시나리오: 글로벌 경기침체 (2008년형)**
```
A~B 미국 전체: ▼▼▼ → DWVA 매집 (가장 큰 자금)
C 에너지:      ▼▼   → DWVA 매집 (유가 폭락 동반)
D 금:          ▲▲   → DWVA 대기 (안전자산 급등)
E 중국:        ▼▼▼ → DWVA 매집
I 신흥국:      ▼▼▼ → DWVA 매집
```
→ 대부분 하락하지만 금(D)이 방어. 이런 극단적 시나리오에서는 **총 투입 속도를 자동으로 늦추는 로직** 필요 (동시 활성 세션 수 제한).

### 9.5 상관관계 분석 모듈

시뮬레이터에 내장하는 상관관계 분석 기능.

#### 기능 1: 정적 상관관계 매트릭스

과거 N년간의 일별 수익률로 종목 간 피어슨 상관계수를 계산한다.

**입력**: 유니버스 전체 종목 티커 목록, 분석 기간
**출력**: 상관관계 히트맵 (9×9 그룹 레벨 + 개별 종목 레벨)
**활용**: 디커플링이 약한 종목 쌍 식별, 유니버스 구성 검증

#### 기능 2: 롤링 상관관계

상관관계는 시간에 따라 변한다. 60일/120일/252일(1년) 롤링 윈도우로 상관관계 추이를 추적한다.

**출력**: 시간 축 상관관계 추이 차트
**활용**: "이 두 종목이 최근 상관관계가 높아지고 있다" → DWVA 동시 매집 위험 경고

#### 기능 3: 위기 시 상관관계 (Tail Correlation)

일반적 시장에서의 상관관계와 위기 시(하락 상위 5%) 상관관계는 다르다. 위기 시 대부분의 자산이 동시에 하락하는 경향이 있다. 이를 별도로 측정한다.

**출력**: 정상 시 상관관계 vs 위기 시 상관관계 비교 테이블
**활용**: "정상 시에는 디커플링이지만 위기 시에는 동조화된다" → 글로벌 급락 시나리오 대비

### 9.6 자동 추천 시스템

시뮬레이터가 현재 활성 DWVA 세션 상태를 고려하여 새로운 매집 기회를 자동 추천한다.

#### 추천 로직

```
1. 전체 유니버스 스캔: 각 종목의 고점 대비 하락률 실시간 계산

2. 진입 트리거 근접 종목 필터링:
   → 고점 대비 -10% 이상 하락한 종목 목록 생성
   → 각 종목의 DWVA 등급별 트리거까지 남은 하락폭 계산

3. 현재 활성 세션과의 상관관계 체크:
   → 새 후보 종목과 현재 매집 중인 종목들의 상관관계 확인
   → 상관관계 0.7 이상이면 경고: "현재 SOXL 매집 중. TQQQ는 상관관계 0.92로 자금 집중 위험"
   → 상관관계 0.3 이하면 추천: "현재 SOXL 매집 중. XLE는 상관관계 0.15로 디커플링 양호"

4. 자금 여력 체크:
   → 현재 활성 세션들의 남은 투입 예정 자금 합산
   → 미배분 자금 확인
   → 자금 여력 부족 시: "신규 매집 가능하나 자금 여력 부족. 기존 세션 축소 또는 추가 자금 필요"

5. 추천 보고서 생성:
   → 종목, 현재 하락률, 트리거까지 남은 하락폭, 활성 세션과의 상관관계, 추천 파라미터
   → Telegram으로 주간 스캔 결과 발송
```

### 9.7 동시 활성 세션 관리

여러 종목에서 동시에 DWVA 세션이 활성화될 때의 자금 배분 규칙.

| 규칙 | 내용 |
|------|------|
| **동시 세션 최대 수** | 5개 (자금 규모에 따라 조정) |
| **단일 세션 최대 자본** | 총 자본의 30% |
| **상관관계 높은 종목(>0.7) 동시 매집** | 합산 자본 25% 이내 제한 |
| **글로벌 급락 감지** | 유니버스 60%+ 동시 하락 시 전체 매수 속도 50% 감속 |
| **자금 예약 제도** | 각 세션 생성 시 총 투입 예정금을 "예약". 미배분 자금에서 차감 |

```python
    def rolling_correlation(self, ticker_a: str, ticker_b: str, windows: list[int]) -> pd.DataFrame:
        """롤링 윈도우 상관관계 추이 계산."""
    
    def tail_correlation(self, tickers: list[str], percentile: int = 5) -> pd.DataFrame:
        """위기 시(하락 상위 N%) 상관관계 계산."""
    
    def plot_heatmap(self, matrix: pd.DataFrame) -> Figure:
        """상관관계 히트맵 시각화."""
    
    def plot_rolling(self, rolling_data: pd.DataFrame) -> Figure:
        """롤링 상관관계 추이 차트."""


class CapitalAllocator:
    """동시 활성 세션 간 자금 배분 관리."""
    
    def __init__(self, total_capital: float, rules: SessionRules):
        """총 자본 및 규칙 설정."""
    
    def reserve(self, session: DWVASession) -> bool:
        """신규 세션의 투입 예정 자금 예약. 규칙 위반 시 False 반환."""
    
    def release(self, session: DWVASession):
        """완료된 세션의 미사용 자금 반환."""
    
    def get_available(self) -> float:
        """현재 미배분(사용 가능) 자금."""
    
    def check_crash_mode(self, scan_result: ScanResult) -> bool:
        """글로벌 급락 감지 → 매수 속도 감속 여부 판단."""
```

### 10.3 구성 요소

#### Streamlit 웹 대시보드 (맥북 로컬)

Streamlit은 Python만으로 인터랙티브 웹앱을 만들 수 있는 프레임워크이다. 별도 프론트엔드(React/Vue) 개발 없이 기존 시뮬레이터/트레이딩 코드를 그대로 UI로 노출할 수 있다.

**대시보드 페이지 구성:**

| 페이지 | 기능 | 접근 수준 |
|--------|------|----------|
| **홈 / 개요** | 전체 포트폴리오 상태, 활성 세션 요약, 일일 손익 | 읽기 전용 |
| **세션 관리** | 활성 세션 목록, 상태(WATCHING~COMPLETED), 실시간 수익률/평단가 | 읽기 + 일시정지/재개 |
| **시뮬레이터** | 백테스팅 실행, Monte Carlo 실행, 파라미터 입력/결과 시각화 | 실행 가능 |
| **유니버스 스캔** | 9개 그룹 고점 대비 하락률, 상관관계 히트맵, 자동 추천 | 읽기 전용 |
| **파라미터 최적화** | Optuna 최적화 실행, 히트맵, 민감도 분석 | 실행 가능 |
| **거래 내역** | 매수/매도 이력, 체결 상세, 일별 성과 로그 | 읽기 전용 |
| **보고서** | 일일/주간/월간 보고서 조회 및 다운로드 | 읽기 전용 |
| **설정** | 세션 파라미터 조회 (수정은 yaml 직접 편집 — 보안) | 조회만 가능 |

**보안 조치:**
- Streamlit 자체에 비밀번호 인증 적용 (`st.secrets` 또는 `streamlit-authenticator`)
- API 키, 거래소 비밀번호 등은 대시보드에 표시하지 않음
- 주문 실행은 대시보드에서 직접 불가 — 세션 생성/일시정지/재개만 가능
- 신규 세션 생성 시 Telegram 2차 확인 필수

#### Tailscale VPN (맥북 ↔ 외부 디바이스)

맥북의 Streamlit 대시보드에 외부에서 안전하게 접근하기 위한 개인용 VPN.

**왜 Tailscale인가:**
- 설치가 매우 간단 (앱 설치 + 로그인만으로 완료)
- 무료 (개인용, 디바이스 100대까지)
- WireGuard 기반 암호화 (군사급 보안)
- 포트포워딩, 공유기 설정, DDNS 불필요
- NAT 환경에서도 자동 터널링
- macOS, iOS, Android 모두 지원

**설정 방법:**
```
1. 맥북: Tailscale 설치 → 로그인 → IP 확인 (예: 100.x.y.z)
2. 스마트폰: Tailscale 설치 → 동일 계정 로그인
3. 맥북에서 Streamlit 실행: streamlit run dashboard.py --server.port 8501
4. 스마트폰 브라우저에서: http://100.x.y.z:8501 접속
```

**맥북 잠자기 방지:**
원격 접근이 항상 가능하려면 맥북이 꺠어있어야 한다.
- `caffeinate -s` 명령으로 잠자기 방지 (전원 연결 시)
- 또는 시스템 설정 → 에너지 절약 → "네트워크 접근 시 깨우기" 활성화
- 외부 모니터 없이도 clamshell 모드에서 동작 가능 (전원+외장키보드 연결 시)

#### Telegram Bot (알림 + 간단한 명령)

실시간 알림과 외부에서의 긴급 명령을 처리한다. 대시보드에 접속하지 않아도 핵심 정보를 받고 대응할 수 있다.

**알림 (Bot → 사용자):**
- 트리거 발동/매집 개시/매수 체결/가중치 전환/매수 중단
- 일일/주간 보고서
- 오류/긴급 상황

**명령 (사용자 → Bot):**

| 명령 | 기능 | 보안 |
|------|------|------|
| `/status` | 전체 활성 세션 상태 요약 | — |
| `/session SOXL` | 특정 세션 상세 (평단가, 수익률, 잔여자금) | — |
| `/universe` | 유니버스 스캔 결과 (트리거 근접 종목) | — |
| `/pause SOXL` | 특정 세션 일시 정지 | 확인 메시지 후 실행 |
| `/resume SOXL` | 일시 정지된 세션 재개 | 확인 메시지 후 실행 |
| `/pause_all` | 전체 세션 일시 정지 (긴급) | 확인 메시지 후 실행 |
| `/report daily` | 최근 일일 보고서 재발송 | — |
| `/sim SOXL -30% 6m` | 간단 시뮬레이션 실행 (결과 요약 텍스트로 반환) | — |

**보안 조치:**
- Bot은 사전 등록된 Telegram Chat ID에서만 명령 수신
- 세션 정지/재개 등 실행 명령은 "정말 실행하시겠습니까?" 확인 단계 필수
- 주문 직접 실행은 Telegram에서 불가 (세션 상태 변경만 가능)

### 10.4 보안 레이어 정리

| 데이터/기능 | 위치 | 외부 접근 | 보호 방식 |
|------------|------|----------|----------|
| API Key / Secret | 맥북 .env 파일 | ❌ 불가 | 파일 시스템 권한, .gitignore |
| 주문 실행 로직 | 맥북 트레이딩 엔진 | ❌ 불가 | 로컬 프로세스만 호출 가능 |
| 세션 데이터 (JSON) | 맥북 sessions/ | ❌ 불가 | 대시보드에서 읽기만 가능 |
| 대시보드 UI | 맥북 Streamlit | ✅ Tailscale + 비밀번호 | VPN 암호화 + 앱 인증 |
| 알림/명령 | Telegram Bot | ✅ 등록된 Chat ID만 | Chat ID 화이트리스트 |
| 시뮬레이션 실행 | 맥북 로컬 | ✅ 대시보드에서 트리거 | VPN + 인증 경유 |
| 설정 변경 (yaml) | 맥북 파일 직접 편집 | ❌ 불가 | SSH 또는 로컬에서만 |

**핵심**: 돈이 움직이는 모든 것(주문, API 키, 설정 변경)은 맥북 로컬에서만 가능하다. 외부에서는 보기/일시정지/시뮬레이션만 가능하다.

## 12. 면책

본 시스템은 투자 전략의 사전 검증 및 자동 집행을 위한 도구이며, 시뮬레이션 결과나 자동 매매 성과가 미래 투자 수익을 보장하지 않습니다. 모든 투자 결정 및 손실에 대한 책임은 전적으로 투자자 본인에게 있습니다. Monte Carlo 시뮬레이션은 가정에 기반한 확률적 추정이며, 실제 시장은 모델의 가정과 다르게 움직일 수 있습니다. 자동 매매 시스템은 기술적 오류, 네트워크 장애, 거래소 장애 등으로 예상과 다르게 작동할 수 있으므로 반드시 모의투자로 충분한 검증 후 사용하시기 바랍니다.
