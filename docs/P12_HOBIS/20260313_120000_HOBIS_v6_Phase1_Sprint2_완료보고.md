### 3. IndexedDB 마이그레이션 (`b02d215`)

| 수정 파일 | 변경 내용 |
|-----------|-----------|
| `js/store.js` | localStorage(5MB) → IndexedDB(50MB+) 전면 교체 |
| `js/core.js` | `storeLoad().then()` 비동기 초기화 체인 |

- **문제**: 반복 임포트로 localStorage 5MB 초과 → 저장 실패
- **해결**: IndexedDB `hobis_v6_db` 백엔드, localStorage 폴백, 자동 마이그레이션
- `storeLoad()` → Promise 반환 (비동기)
- `storeSave()` → fire-and-forget IndexedDB 쓰기

### 4. 임포트 중복 방지 (`b02d215`)

- `flowId` 기반 1차 dedup (Flow 임포트 이벤트)

- 임포트 시 프로젝트 인라인 생성 (건별 storeSave → 1회 일괄 저장)
- `storeDedup()` 앱 시작 시 자동 실행 (기존 중복 데이터 정리)

## 기술 변경 사항

### 데이터 레이어 아키텍처 변경

```
[Sprint 1]  localStorage(5MB) — 동기식 read/write
    ↓
[Sprint 2]  IndexedDB(50MB+) — 비동기 Promise 기반
            + localStorage 자동 폴백
            + 최초 실행 시 localStorage → IndexedDB 자동 마이그레이션
```

### 새 Store API

| 함수 | 역할 |
|------|------|
| `storeAddComment(eventId, comment)` | 댓글 추가 (cm_ ID 자동 생성) |
| `storeUpdateComment(eventId, commentId, changes)` | 댓글 수정 |
| `storeDeleteComment(eventId, commentId)` | 댓글 삭제 |
| `storeDedup()` | 중복 이벤트/프로젝트 정리 |
| `storeReplaceData(newData)` | 전체 데이터 교체 (v6 임포트용) |
