# HOBIS v6 Phase 1 Sprint 1 — 구현 계획

## 배경

HOBIS v5는 사이버펑크 UI의 바닐라 JS SPA로, 방사선 방호 계산기 + **읽기 전용** Flow 캘린더 뷰어로 구성되어 있다. v6에서는 이를 프로젝트/일정 관리 시스템으로 진화시킨다 — CRUD, localStorage 영속성, Flow JSON 가져오기 호환.

**Sprint 1 목표**: 읽기 전용 Flow 캘린더를 프로젝트 관리, 일정 CRUD, 로컬 데이터 영속성을 갖춘 완전한 캘린더로 전환

**로컬 경로**: `$LOCALAPPDATA/Temp/hobis-v5-check/` (hobis-v6 GitHub 레포)

## 신규 생성 파일 (의존성 순서)

### 1. `js/store.js` (~250줄) — 데이터 레이어
localStorage 래퍼. **모든 다른 모듈이 의존함.**

- **데이터 스키마**: `{ projects: [], events: [], meta: { version, lastModified } }`
- **주요 함수**:
  - `generateId(prefix)` → `"ev_" + 랜덤hex` 또는 `"pj_" + 랜덤hex`
  - `storeLoad()` — localStorage 키 `hobis_v6`에서 읽기, 없으면 빈 기본값 반환
  - `storeSave(data)` — `meta.lastModified` 타임스탬프와 함께 저장
  - `storeGetProjects()` / `storeGetEvents()` — 편의 getter
  - `storeCreateProject({name, color})` → UUID 포함 새 프로젝트 반환
  - `storeUpdateProject(id, changes)` / `storeDeleteProject(id)`
  - `storeCreateEvent({title, description, projectId, start, end, location, attendees, author, comments})` → 새 이벤트 반환
  - `storeUpdateEvent(id, changes)` / `storeDeleteEvent(id)`
  - `storeImportFlowJson(flowArray)` — Flow 형식 변환 (프로젝트명 → projectId), 없는 프로젝트 자동 생성, 이벤트 병합
  - `storeExportAll()` — 전체 데이터를 JSON 문자열로 반환
  - `storeGetUsage()` — localStorage 사용량 (바이트/퍼센트) 반환

### 2. `js/projects.js` (~180줄) — 프로젝트 관리 UI
프로젝트 목록, 컬러 피커, 생성/수정/삭제 모달.

- **색상 팔레트**: `V6_PROJECT_COLORS` — 사이버펑크 8색 `{name, hex, bg}`
- **주요 함수**:
  - `projRenderList(container)` — 컬러 도트 + 이벤트 수와 함께 프로젝트 목록 렌더링
  - `projRenderDropdown(selectEl, selectedId)` — 이벤트 폼용 `<select>` 채우기
  - `projShowCreateModal()` / `projShowEditModal(id)` — 이름 입력 + 컬러 그리드 모달
  - `projDoCreate()` / `projDoUpdate(id)` / `projDoDelete(id)` — store 호출, UI 갱신
  - `projGetColor(projectId)` → hex 색상 문자열
  - `projGetName(projectId)` → 프로젝트명 문자열

### 3. `js/event-form.js` (~200줄) — 일정 생성/수정 모달
모든 필드가 포함된 일정 생성/수정 모달 폼.

- **폼 필드**: 제목, 프로젝트(드롭다운), 시작 일시, 종료 일시, 장소, 설명, 참석자(쉼표 구분), 작성자
- **주요 함수**:
  - `efShowCreate(prefillDate)` — 빈 폼 열기, 캘린더 클릭 시 날짜 자동 채움
  - `efShowEdit(eventId)` — 기존 이벤트 데이터로 채워진 폼 열기
  - `_efRenderModal()` — 사이버펑크 스타일 폼 필드의 모달 HTML 생성
  - `efDoSave()` — 유효성 검증, store 호출, 모달 닫기, 캘린더 새로고침
  - `efDoDelete()` — 확인 다이얼로그, storeDeleteEvent 호출, 모달 닫기
  - `efClose()` — 모달 닫기, 정리

### 4. `js/events.js` (~950줄) — 리팩토링된 캘린더 모듈
**`flow-calendar.js` → `events.js`로 이름 변경**. 가장 큰 변경 — 기존 검색/캘린더/상세 UI 유지하면서 CRUD 추가.

- **핵심 마이그레이션**: `fcData[id]` (배열 인덱스) → `fcData.find(e => e.id === id)` (UUID 조회)
  - 531번줄: `const ev = fcData[id]` → UUID find로 변경
  - 253/257번줄: filter 연산은 이미 안전 (반복문 사용)
  - 733, 877번줄: forEach/filter — 이미 안전
- **주요 변경사항**:
  - `evInitApp()` — `fcInitApp()` 대체, 파일 업로드 대신 store에서 로드
  - `evGetAllEvents()` / `evGetEvent(id)` — store 기반 getter
  - `evBuildProjectMap()` — 이벤트 스캔 대신 store 프로젝트에서 구축
  - `evRefresh()` — store에서 데이터 재로드, 현재 뷰 재렌더링
  - 기존 `fcDoSearch()`, `fcRenderCalendar()`, `fcShowDetail()` 함수 모두 유지
  - 헤더에 "일정 생성" 버튼, 상세 뷰에 "수정"/"삭제" 버튼 추가
  - 캘린더 날짜 셀 클릭 핸들러 → `efShowCreate(clickedDate)`
  - 이벤트 칩에 `event.stopPropagation()` 추가 (셀 클릭 방지)
  - `fcHandleFile()` → `storeImportFlowJson()` 호출로 변경
  - 파일 업로드 박스는 보조 기능("Flow JSON 가져오기")으로 변경

### 5. `js/import-export.js` (~120줄) — JSON 내보내기/가져오기
형식 자동 감지 기능이 포함된 내보내기/가져오기.

- **주요 함수**:
  - `ioExportJson()` — Blob URL로 `hobis_v6_data.json` 다운로드
  - `ioImportFile(event)` — JSON 파일 읽기, 형식 자동 감지
  - `_ioDetectFormat(parsed)` — `'v6'` (meta.version 존재) 또는 `'flow'` (flow 이벤트 배열) 반환
  - `ioShowImportConfirm(data, format)` — 병합/교체 전 확인 다이얼로그

## 기존 파일 수정

### 6. `js/core.js` (84줄 → ~100줄)
- `currentMainTab = 'decay'` → `'calendar'`로 변경
- `setMainTab()`에 `'calendar'`를 `'flowcal'` 별칭으로 추가
- **`event.target` 버그 수정**: `setMainTab(t)`이 암시적 `event.target` 사용 → 프로그래밍 호출 시 깨짐. 수정: 두 번째 파라미터 `setMainTab(t, btn)` 추가, `btn || event?.target` 사용
- `window.onload`에 `storeLoad()` → `evInitApp()` 호출 추가

### 7. `index.html` (711줄)
- 버전 텍스트 변경: `HOBIS v4.15 patch3` → `HOBIS v6.0`
- **탭 순서 변경**: CALENDAR을 첫 번째로, 이후 DECAY, SHIELD, LOGISTICS
  ```html
  <button class="tab-btn active" onclick="setMainTab('calendar', this)">CALENDAR</button>
  <button class="tab-btn" onclick="setMainTab('decay', this)">DECAY</button>
  <button class="tab-btn" onclick="setMainTab('shield', this)">SHIELD</button>
  <button class="tab-btn" onclick="setMainTab('logistics', this)">LOGISTICS</button>
  ```
- flowCalPanel 헤더 영역에 도구 버튼 추가:
  - `+ 새 일정` 버튼 → `efShowCreate()`
  - `프로젝트` 버튼 → `projShowCreateModal()`
  - `내보내기` 버튼 → `ioExportJson()`
  - `가져오기` 버튼 (기존 file input 유지, 스타일 변경)
- **스크립트 로드 순서** (새 스크립트를 events.js 앞에):
  ```html
  <script src="js/store.js"></script>
  <script src="js/projects.js"></script>
  <script src="js/event-form.js"></script>
  <script src="js/import-export.js"></script>
  <script src="js/events.js"></script>
  ```
  (`flow-calendar.js` 참조 제거, `events.js`로 대체)

### 8. `css/hobis.css` — 신규 스타일 추가
~150줄의 새 CSS 추가:

- **이벤트 폼** (`.ef-*`): 모달 오버레이, 사이버펑크 스타일 폼 필드, 날짜/시간 선택기
- **프로젝트 UI** (`.proj-*`): 컬러 피커 그리드, 프로젝트 목록 아이템, 컬러 도트
- **확인 다이얼로그** (`.v6-confirm-*`): 삭제 확인 오버레이
- **빈 상태** (`.v6-empty-*`): "일정이 없습니다" 플레이스홀더 + 생성 버튼
- **저장소 표시** (`.v6-storage-*`): localStorage 사용량 바
- **툴바** (`.v6-toolbar`): 캘린더 헤더의 버튼 행
- 모든 신규 스타일은 기존 CSS 변수 사용 (`--hobis-green`, `--hobis-cyan`, `--hobis-bg`, `--hobis-panel`, `--hobis-border`)
