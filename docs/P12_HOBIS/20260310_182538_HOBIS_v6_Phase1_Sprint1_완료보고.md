## 구현된 기능

### 프로젝트 관리
- 프로젝트 생성/수정/삭제 (PROJECTS 버튼)
- 8색 사이버펑크 컬러 팔레트 선택
- 프로젝트 삭제 시 연결된 일정은 유지 (프로젝트 연결만 해제)

### 일정 CRUD
- 새 일정 생성 (+ NEW 버튼 또는 캘린더 날짜 클릭)
- 일정 수정 (상세 뷰에서 EDIT 버튼)
- 일정 삭제 (확인 다이얼로그 포함)
- 제목, 프로젝트, 시작/종료 일시, 장소, 작성자, 설명, 참석자 필드

### 데이터 영속성
- localStorage 기반 자동 저장 (키: `hobis_v6`)
- 브라우저 닫고 다시 열어도 데이터 유지
- 사용량 표시 바 (5MB 기준 퍼센트)
- **임포트 파일 정보 영속성**: 임포트한 파일명, 시간, 건수를 `meta.lastImport`에 저장하여 새로고침 후에도 마지막 임포트 상태 표시

### Flow JSON 호환
- Flow 크롤링 JSON 파일 임포트 시 자동 형식 감지
- 프로젝트명 → projectId 자동 변환, 없는 프로젝트 자동 생성
- v6 형식 JSON 임포트 시 데이터 교체 확인

### 내보내기
- EXPORT 버튼으로 전체 데이터 JSON 다운로드
- 타임스탬프 파일명 (`hobis_v6_YYYYMMDD_HHmm.json`)

### UI 변경
- CALENDAR 탭이 기본 활성 탭으로 변경
- 탭 순서: CALENDAR > DECAY > SHIELD > LOGISTICS
- 캘린더 날짜 셀 클릭 → 해당 날짜로 새 일정 생성 폼 열림
- 이벤트 칩 클릭 시 셀 클릭과 충돌 방지 (stopPropagation)
- 상세 뷰에 EDIT / DELETE 액션 버튼 추가

## 기술 변경 사항

### 아키텍처
- **배열 인덱스 → UUID 기반 ID**: `fcData[id]` → `storeFindEvent(id)` (문자열 ID 조회)
- **전역 배열 → store 래퍼**: `fcData = JSON.parse(...)` → `storeLoad()` + `storeGetEvents()`
- **프로젝트명 → projectId 참조**: 이벤트에 `projectId` 필드 추가, `projGetName()` / `projGetColor()` 함수로 조회

### 스크립트 의존성 체인
```
store.js → projects.js → event-form.js → events.js
                                        ↗
                        import-export.js
```
`core.js`는 모든 v6 모듈 이후에 로드되어 `window.onload`에서 `storeLoad()` → `evInitApp()` 호출
