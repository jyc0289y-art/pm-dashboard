# 항공권 종합 서비스 기획서

> 최종 수정: 2026-04-10 (v2 — API 전략 재편, 아키텍처 보강, MVP 세분화)

## 프로젝트 개요

항공권 가격 검색/모니터링 + 실시간 비행편 추적 + 예약 항공편 출도착/지연/게이트 정보를 하나의 앱에서 제공하는 종합 항공 서비스.

현재 Chrome MCP 기반 스크래핑 프로토타입을 운영 중이며, 이를 공식 API 기반 웹 서비스로 확장하는 것이 목표.

---

## 핵심 기능 3가지

### 1. 항공권 가격 검색 & 모니터링
- 출발지/도착지(또는 "어디든") + 날짜 범위로 항공편 검색
- 관심 노선 등록 → 주기적 가격 모니터링 (cron)
- 가격 추세 시각화 (차트)
- 임계값 이하 시 알림 (텔레그램/이메일/카카오톡)

### 2. 실시간 비행편 위치 추적 (FlightRadar24 스타일)
- 지도 위 항공기 실시간 위치 표시
- 항공편 번호로 특정 비행편 추적
- 항공기 고도/속도/경로 정보

### 3. 예약 항공편 관리
- PNR 번호 또는 항공편 번호 입력으로 예약 항공편 등록
- 출발/도착 시간, 지연 상태, 게이트 정보 실시간 업데이트
- 변경사항 발생 시 푸시 알림

## API 구성

### API 역할 배분 (v2 — Rate Limit 최적화)

| 역할 | API | 용도 | 비용 | Rate Limit |
|------|-----|------|------|------------|
| **메인 검색/가격** | **Kiwi.com Tequila API** | 항공편 검색, "어디든" 탐색, 가격 모니터링 | 무료 | 넉넉 (분당 200건 이상) |
| 보조 검색/검증 | Amadeus Self-Service API | GDS 데이터 교차 검증, 정밀 검색 | 무료 월 2,000건, 초과 시 종량제 | 월 2,000건 (무료) |
| 예매 연결 | Tequila Affiliate | 예매 페이지 리다이렉트 (커미션) | 무료 (커미션 수익) | — |
| 항공편 상태 | AviationStack API | 출도착/지연/게이트 정보 | 무료 월 500건, 유료 $49.99/월 | 월 500건 (무료) |
| 실시간 위치 추적 | OpenSky Network API | ADS-B 기반 항공기 위치 데이터 | 무료 (비상업적) | 인증 시 5초/1건 |

> **v1 대비 변경**: Tequila를 메인 검색 엔진으로 승격. Amadeus는 월 2,000건 제한으로 모니터링 서비스의 핵심 엔진으로 부적합 → GDS 데이터 교차 검증 및 정밀 검색 보조 용도로 전환.

### API별 Rate Limit 관리 전략

| API | 무료 한도 | 예상 사용량 (MVP) | 대응 |
|-----|----------|-------------------|------|
| Tequila | 분당 200+ | 노선 50개 × 4회/일 × 30일 = 6,000건/월 | 캐시 30분 TTL로 중복 제거 |
| Amadeus | 월 2,000건 | 교차 검증 전용: ~500건/월 | 캐시 6시간 TTL, 쿼터 90% 시 중단 |
| AviationStack | 월 500건 | 추적 10편 × 6회/일 × 30일 = 1,800건 | Phase 3 시점에 유료 전환 또는 대안 검토 |
| OpenSky | 인증 시 5초/1건 | 지도 갱신 30초마다 1건 | 서버 캐시 + WebSocket 브로드캐스트 |

## 기술 스택

### 아키텍처 다이어그램 (v2)

```
[Frontend - Next.js 14+]
    |           ↑
    | REST      | WebSocket (실시간 지도/알림)
    v           |
[Backend - FastAPI (Python)]
    |
    ├── [캐시 레이어 - Redis 또는 TTLCache]
    |       ├── 검색 결과 캐시 (TTL 30분)
    |       ├── API 쿼터 카운터
    |       └── 실시간 위치 캐시 (TTL 30초)
    |
    ├── [API 클라이언트 (서킷브레이커 내장)]
    |       ├── Tequila (메인 검색)
    |       ├── Amadeus (보조 검증)
    |       ├── AviationStack (항공편 상태)
    |       └── OpenSky (실시간 위치)
    |
    ├── [SQLite → 가격 히스토리, 사용자 설정, 모니터링 노선]
    |
    └── [APScheduler]
            ├── 가격 체크 (주기적)
            └── 알림 발송 (텔레그램/이메일)
```

### Backend
- **Python 3.11+ / FastAPI**: 비동기 API 서버
- **APScheduler 또는 Celery**: 주기적 가격 체크 스케줄러
- **SQLite (→ 추후 PostgreSQL 마이그레이션 가능)**: 가격 히스토리, 사용자 설정 저장
- **Pydantic**: 데이터 검증
- **cachetools (TTLCache)**: 인메모리 캐시 (MVP) → 추후 Redis 전환
- **aiohttp / httpx**: 비동기 HTTP 클라이언트 (서킷브레이커 패턴 내장)

### Frontend
- **Next.js 14+ (App Router)**: React 기반 프론트엔드
- **Recharts 또는 Chart.js**: 가격 추세 차트
- **Leaflet + OpenStreetMap**: 비행편 지도 시각화
- **Tailwind CSS**: 스타일링
- **반응형 설계**: 모바일 퍼스트 (Phase 1부터)

### 실시간 통신
- **WebSocket (FastAPI WebSocket)**: 실시간 지도 위치 업데이트, 알림 푸시
- **SSE (Server-Sent Events)**: 가격 알림 등 단방향 실시간 이벤트 (경량 대안)

### 인프라
- **Hetzner VPS**: €7.49/월 (2 vCPU, 4GB RAM)
- **Docker + docker-compose**: 배포 컨테이너화
- **GitHub Actions**: CI/CD (테스트 → 빌드 → VPS 배포)
- **도메인**: 연 ~$12
- **SSL**: Let's Encrypt (무료)
- **Nginx**: 리버스 프록시 + WebSocket 업그레이드

### 알림
- **Telegram Bot API**: 무료, 구현 간단
- **이메일**: SMTP (Gmail 또는 Resend.com 무료 티어)
- **카카오톡 알림**: 카카오 알림톡 API (사업자 등록 필요, Phase 4)

## 데이터 모델

```python
# 사용자 (MVP: 디바이스/세션 기반, Phase 4에서 인증 전환)
class User:
    id: int
    device_id: str           # UUID, 브라우저 localStorage에 저장
    nickname: str | None     # 선택적 닉네임
    timezone: str            # 기본 "Asia/Seoul"
    created_at: datetime

# 모니터링 노선
class WatchedRoute:
    id: int
    user_id: int             # FK → User
    origin: str              # IATA 코드 (예: ICN)
    destination: str         # IATA 코드 또는 "anywhere"
    departure_date_from: date
    departure_date_to: date
    return_date_from: date | None
    return_date_to: date | None
    max_price: int           # 알림 임계값 (KRW)
    currency: str            # 기본 "KRW" — 환율 변환 기준
    check_interval: int      # 분 단위
    is_active: bool
    created_at: datetime

# 가격 히스토리
class PriceRecord:
    id: int
    route_id: int            # FK → WatchedRoute
    airline: str
    price_original: float    # API 원본 가격
    currency_original: str   # API 원본 통화 (EUR, USD 등)
    price_krw: int           # KRW 변환 가격
    exchange_rate: float     # 변환 시점 환율
    departure_at: datetime
    arrival_at: datetime
    stops: int
    duration_minutes: int
    source: str              # "tequila" | "amadeus"
    booking_url: str | None  # 어필리에이트 예매 링크
    fetched_at: datetime

# 추적 항공편
class TrackedFlight:
    id: int
    user_id: int             # FK → User
    flight_number: str       # 예: KE001
    departure_date: date
    pnr: str | None
    origin: str
    destination: str
    origin_timezone: str     # 출발지 타임존 (예: "Asia/Seoul")
    dest_timezone: str       # 도착지 타임존 (예: "Asia/Tokyo")
    scheduled_departure: datetime  # UTC
    scheduled_arrival: datetime    # UTC
    actual_departure: datetime | None
    actual_arrival: datetime | None
    gate: str | None
    terminal: str | None
    status: str              # "scheduled" | "departed" | "arrived" | "delayed" | "cancelled"
    delay_minutes: int | None
    last_updated: datetime

# 알림 설정
class AlertConfig:
    id: int
    user_id: int             # FK → User
    channel: str             # "telegram" | "email" | "kakao"
    target: str              # 텔레그램 chat_id 또는 이메일 주소
    is_active: bool

# API 사용량 추적
class ApiUsage:
    id: int
    api_name: str            # "tequila" | "amadeus" | "aviationstack" | "opensky"
    endpoint: str
    called_at: datetime
    response_status: int
    response_time_ms: int
    quota_remaining: int | None  # API가 제공하는 경우
```

## API 엔드포인트 설계

```
# 항공편 검색
GET  /api/search/flights?origin=ICN&dest=NRT&date=2026-05-01&return=2026-05-05

# 모니터링
POST /api/watch                    # 노선 모니터링 등록
GET  /api/watch                    # 등록된 노선 목록
DEL  /api/watch/{id}               # 모니터링 해제
GET  /api/watch/{id}/history       # 가격 히스토리 조회
GET  /api/watch/{id}/chart         # 차트용 데이터

# 항공편 추적
POST /api/track                    # 항공편 추적 등록
GET  /api/track                    # 추적 중인 항공편 목록
GET  /api/track/{flight_number}    # 특정 항공편 실시간 상태
WS   /ws/track/live/map            # 지도용 실시간 위치 (WebSocket)

# 알림
POST /api/alerts/config            # 알림 채널 설정
GET  /api/alerts/history           # 발송된 알림 내역

# 시스템 (내부/관리)
GET  /api/system/api-usage         # API 사용량 대시보드
GET  /api/system/health            # 헬스체크
```

## 에러 핸들링 & 폴백 전략

```
[API 호출]
    |
    ├── 성공 → 캐시 저장 + 응답
    |
    ├── 일시 오류 (5xx, timeout)
    |       └── 재시도 (최대 3회, 지수 백오프)
    |               └── 실패 → 서킷 오픈 (60초 차단)
    |                       └── 캐시 폴백 (이전 결과 반환 + "cached" 플래그)
    |
    └── 쿼터 초과 (429)
            └── 쿼터 카운터 업데이트 + 해당 API 비활성화
                    └── 대안 API로 자동 전환 (Tequila ↔ Amadeus)
```

| 장애 시나리오 | 폴백 |
|--------------|------|
| Tequila 다운 | Amadeus로 검색 (쿼터 허용 시) + 캐시된 결과 |
| Amadeus 쿼터 초과 | 교차 검증 비활성화, Tequila 단독 운영 |
| AviationStack 다운 | 최근 캐시 반환 + "업데이트 지연" 표시 |
| OpenSky 다운 | 지도 "최근 위치" 표시 + 갱신 중단 안내 |

## 주의사항

1. **스크래핑 금지**: 프로덕션에서는 반드시 공식 API만 사용할 것. 스크래핑은 ToS 위반 + 불안정.
2. **API Rate Limit 관리**: 캐시 레이어 + 쿼터 카운터로 관리. 쿼터 90% 도달 시 자동 경고.
3. **OpenSky 비상업적 제한**: 상업적 서비스 전환 시 별도 라이선스 필요.
4. **예매 직접 중계 불가**: IATA 인가(보증금 $50,000+) + 한국 여행업 등록 필요. MVP에서는 어필리에이트 모델로 우회.
5. **개인정보보호**: 사용자 이메일/PNR 등 수집 시 개인정보처리방침 필요.
6. **타임존 처리**: 모든 시간은 UTC로 저장, 출발지/도착지 로컬 시간은 표시 시 변환. pytz 또는 zoneinfo 사용.
7. **통화 변환**: API 응답 원본 통화 + KRW 변환가 모두 저장. 환율은 실시간 조회 (하루 1회 갱신).
