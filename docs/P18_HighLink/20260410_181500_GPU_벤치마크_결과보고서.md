## 5. 결론 및 권고

### 5-1. 15fps 목표 달성 여부
- YOLO11L@1280: **달성** (28.1 fps, 목표의 1.87배)
- YOLO11L@640: **달성** (57.1 fps, 목표의 3.81배)
- YOLO11S@1280: **달성** (65.8 fps, 목표의 4.39배)
- YOLO11S@640: **달성** (97.4 fps, 목표의 6.49배)

### 5-2. VRAM 여유도
- YOLO11L@1280 단일: 0.33GB / 8GB (4.1%)
- 멀티스트림 x2: 0.68GB / 8GB (8.5%)
- **RTX 4060 8GB로 동시 처리 가능한 최대 스트림 수 추정**: 약 **10~12대**
  - VRAM 기준: 0.33GB/스트림 x 12 = 3.96GB (여유 충분)
  - 연산 기준: 단일 28.1fps → 배치 효율 고려 시 약 10~12스트림에서 15fps/스트림 유지 가능 추정
  - 단, 실제로는 영상 디코딩(FFmpeg) CPU 부하와 전후처리 오버헤드가 추가됨

### 5-3. 운영 환경 GPU 사양 권고

| 동시 카메라 수 | 권장 GPU | 근거 |
|---------------|----------|------|
| 3대 (Phase 1) | **RTX 4060 8GB** | 충분 — 단일 GPU로 28fps 여유 |
| 10~20대 (Phase 2) | **RTX 4090 24GB** 또는 **A4000 16GB** | VRAM 3배 + 연산 2~3배 필요 |
| 50~100대 (Phase 3) | **A100 40/80GB** 또는 **L40S 48GB** | 다중 GPU 또는 Triton 멀티인스턴스 |

- RTX 4060은 Phase 1 개발/PoC용으로 충분하며, 운영 환경에서는 카메라 수에 따라 상위 GPU 필요
- VRAM 사용량이 매우 낮은 것이 특징 — 모델 추론보다 **배치 스케줄링 최적화**가 멀티스트림 성능의 관건

### 5-4. 기타 관찰 사항

1. **테스트 영상 해상도 제한**: 640x360 (YouTube 무료 다운로드 제한). 실제 ITS CCTV 영상(720p~1080p)에서는 FPS가 다소 하락할 수 있음 → ITS API 인증 후 재검증 필요
2. **VRAM 사용량이 예상보다 낮음**: 0.33GB (모델 로드 + 추론). 이는 640x360 입력 영상이 작기 때문. 1080p 입력 시 VRAM 사용량 증가 예상
3. **소형 객체 비율 56%**: sample_test.py 결과 소형 객체(32x32px 미만)가 전체 감지의 56%를 차지 → **SAHI 도입이 필수적**이나, FPS 감소(4~6배) 고려 필요
4. **MPS 대비 2.9배 성능**: RTX 4060이 M시리즈 대비 약 3배 빠름. 개발 중 빠른 이터레이션에 유리
5. **멀티스트림 배치 효율**: 2스트림 배치에서 총 FPS가 단일과 거의 동일(27.2 vs 28.1) → 배치 오버헤드가 거의 없음. 추가 스트림 확장에 긍정적 신호
6. **total_mem API 변경**: PyTorch 2.11+에서 `torch.cuda.get_device_properties().total_mem` → `total_memory`로 변경됨. 기존 스크립트 수정 적용함
