# HighLink — Streamlit 데모 서버 재시작 지시서

> **이 문서는 Claude Code가 읽고 실행하기 위한 자동화 지시서입니다.**

## 배경

SUV/밴 → truck 오분류 보정 로직이 `engine/detection/detector.py`와
`app/streamlit/app.py`에 추가되었습니다. Streamlit 서버를 재시작하여 변경사항을 반영합니다.

## ⚠️ 보안 필수 준수사항

**아래 규칙은 반드시 지켜야 합니다. 위반 시 API 키/소스코드 유출 위험.**

### 1. API 키는 secrets.toml에만 저장

```
.streamlit/secrets.toml   ← 여기에만 저장 (Git 미포함)
```

- **절대 코드 파일(.py)에 API 키를 하드코딩하지 마세요**
- **절대 커밋 메시지, 결과보고서, 로그 파일에 키를 포함하지 마세요**
- 환경변수 `set HIGHLINK_ITS_API_KEY=...`도 사용 가능하지만, secrets.toml이 우선 권장

### 2. Git 커밋 전 확인

```bash
# 커밋 전 반드시 확인: secrets.toml이 스테이징에 포함되지 않았는지
git status
# .streamlit/secrets.toml이 보이면 안 됨
# .gitignore에 포함되어 있으므로 정상적으로는 나타나지 않음
```

- `git add .` 또는 `git add -A` 사용 금지 → 파일을 명시적으로 지정하여 add
- `.env`, `.streamlit/secrets.toml`, 키가 포함된 어떤 파일도 커밋 금지

### 3. 외부 접속 보안

- Streamlit 비밀번호 인증이 반드시 활성화된 상태에서 Cloudflare Tunnel 실행
- Tunnel URL을 공개 채널(SNS, 오픈 채팅 등)에 게시 금지 → 파트너에게만 DM으로 전달
- 테스트 완료 후 사용하지 않는 Tunnel은 종료

### 4. 소스코드 보호

- 파트너는 브라우저 접속만 허용 → 서버 SSH/원격 접속 제공 금지
- Streamlit은 서버 사이드 렌더링이므로 Python 코드는 브라우저에 노출되지 않음
- 그러나 **에러 메시지에 파일 경로/코드 스니펫이 노출**될 수 있으므로, 운영 시 Streamlit 디버그 모드 비활성화 권장:
  ```bash
  streamlit run app/streamlit/app.py --server.port 8501 --logger.level error
  ```

### 보안 체크 요약

| 위협 | 대응 | 확인 |
|------|------|------|
| API 키 Git 노출 | `.gitignore`에 `secrets.toml` 포함 | 커밋 전 `git status` 확인 |
| API 키 브라우저 노출 | 서버 사이드에서만 사용, 클라이언트 미전송 | 코드에서 `st.secrets` 사용 |
| 비인가 접속 | Streamlit 비밀번호 + Cloudflare HTTPS | 인증 없이 접속 불가 확인 |
| 소스코드 노출 | 서버 사이드 렌더링, SSH 미제공 | F12에서 Python 코드 미노출 확인 |
| 에러 메시지 경로 노출 | `--logger.level error` 옵션 | 에러 발생 시 경로 미표시 확인 |

## 1단계: 코드 풀 + API 키 설정

```bash
cd {프로젝트 루트}
git pull
```

### API 키를 secrets.toml에 설정 (Git에 포함되지 않음)

```bash
# .streamlit 디렉토리 생성
mkdir -p .streamlit
```

`.streamlit/secrets.toml` 파일을 아래 내용으로 생성:

```toml
password = "highlink2026"
its_api_key = "여기에_ITS_API_키_입력"
```

**이 파일은 `.gitignore`에 포함되어 있어 절대 Git에 커밋되지 않습니다.**
환경변수(`set HIGHLINK_ITS_API_KEY=...`)는 더 이상 필요 없습니다.

## 2단계: 변경사항 확인

```python
# 후처리 함수 테스트
from engine.detection.detector import reclassify_truck

# SUV (종횡비 낮고 면적 작음) → car
assert reclassify_truck([100, 100, 200, 250], 7, 720, 1280) == 2
# 실제 트럭 (가로 넓음) → truck 유지
assert reclassify_truck([100, 100, 500, 250], 7, 720, 1280) == 7
print("후처리 테스트 통과")
```

## 3단계: Streamlit 서버 재시작

### 기존 서버 중지

```bash
# 실행 중인 Streamlit 프로세스 찾기
# Windows
tasklist | findstr streamlit
taskkill /F /IM streamlit.exe

# 또는 Ctrl+C로 기존 터미널 종료
```

### 서버 재시작

```bash
source .venv/bin/activate   # Windows: .venv\Scripts\activate
set HIGHLINK_ITS_API_KEY=여기에_API_키

streamlit run app/streamlit/app.py --server.port 8501
```

## 4단계: 동작 확인

브라우저에서 `http://localhost:8501` 접속 후:

1. **모드 A (영상 업로드)**: `data/samples/mixed_traffic.mp4` 업로드
   - [ ] SUV/밴이 "승용차"로 표시되는지 확인
   - [ ] 대형 트럭은 "트럭"으로 유지되는지 확인
2. **모드 C (실시간 CCTV)**: CCTV 선택 → 감지 시작
   - [ ] 재분류 로직이 실시간에서도 정상 동작하는지 확인

## 5단계: Cloudflare Tunnel 재시작 (이미 실행 중이면 생략)

```bash
cloudflared tunnel --url http://localhost:8501
```
