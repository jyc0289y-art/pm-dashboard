# HighLink — Phase 1 2단계 분류기 + 모드C 웹캠버그 수정 + 데모 영상 재생성 의뢰서

> **이 문서는 회사 PC(RTX 4060)의 Claude Code가 읽고 실행하기 위한 자동화 의뢰서입니다.**

## 배경

### 1. 2단계 차종 분류기 (EfficientNet-B0)
Phase 1의 핵심 기능인 **2단계 차종 분류기(EfficientNet-B0)**가 파이프라인에 통합되었습니다.
기존 `reclassify_truck()` 휴리스틱(종횡비+면적)에 더해, YOLO가 truck(7)으로 분류한 객체를
EfficientNet-B0으로 재분류하여 SUV/밴/승용차를 정확히 구분합니다.

### 2. 모드 C 웹캠 버그 수정 (긴급)
모드 C에서 `streamlit-webrtc`의 `webrtc_streamer()`가 ITS CCTV URL이 아닌 **클라이언트 웹캠**을
활성화하는 구조적 버그가 수정되었습니다. WebRTC 블록을 완전히 제거하고,
서버 측에서 `cv2.VideoCapture(selected_cctv.url)`로 CCTV 스트림을 직접 수신하는 방식으로 변경했습니다.

**변경 파일:**
- `engine/detection/vehicle_classifier.py` — NEW: EfficientNet-B0 분류기 (ImageNet pretrained)
- `engine/detection/detector.py` — VehicleDetector에 분류기 통합
- `scripts/generate_demo_video.py` — 데모 영상에 분류기 적용
- `app/streamlit/app.py` — 모드C 웹캠버그 수정 + 모든 모드에 분류기 적용 + 에러 메시지 수정

## 2단계: EfficientNet-B0 모델 확인

EfficientNet-B0은 torchvision에 포함되어 있으므로 별도 다운로드 불필요합니다.
첫 실행 시 ImageNet 가중치가 자동 다운로드됩니다 (~20MB).

```python
# 동작 확인
from engine.detection.vehicle_classifier import VehicleClassifier
import numpy as np

clf = VehicleClassifier(device="cuda:0")

# 더미 이미지로 테스트
dummy = np.random.randint(0, 255, (224, 224, 3), dtype=np.uint8)
category, prob = clf.classify_crop(dummy)
print(f"분류 결과: {category}, 확신도: {prob:.3f}")
print("2단계 분류기 로딩 성공!")
```

```bash
    --output data/demo/highlink_demo_mixed.mp4 \
    --max-frames 300

    --input data/samples/mixed_traffic.mp4 \

    --input data/samples/highway_sample.mp4 \

    --max-frames 300
```
