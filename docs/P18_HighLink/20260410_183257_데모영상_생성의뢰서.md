# HighLink Phase 0 — 데모 영상 생성 의뢰서

> **이 문서는 Claude Code가 읽고 실행하기 위한 자동화 의뢰서입니다.**
> 모든 단계를 순서대로 실행하고, 마지막에 결과보고서를 지정된 양식으로 생성해 주세요.

## 배경

맥북(Apple Silicon MPS)에서 데모 영상을 생성했으나 **9.7 FPS**로 느립니다.
이 PC의 **RTX 4060**에서 동일 스크립트를 실행하면 28+ FPS가 나올 것으로 예상됩니다.

고속도로 영상에서 YOLO11이 차량(승용차/트럭/버스/오토바이/보행자)을 실시간으로
구분·감지하는 모습을 보여주는 데모 영상을 생성합니다. 이 영상은 **사업 파트너에게
기술력을 시연하는 용도**로, 코드 없이 결과물만 공유됩니다.

## 0단계: 테스트 영상 교체 (중요!)

현재 `data/samples/highway_sample.mp4`는 승용차만 나옵니다 (car 920, truck 1).
**차종 구분 시연이 안 되므로**, 트럭/버스/오토바이/보행자가 혼재된 영상으로 교체합니다.

```bash
pip install yt-dlp
```

아래 영상 중 **다운로드 가능한 것을 순서대로 시도**하여 `data/samples/`에 저장하세요.
각 영상은 트럭, 버스, 오토바이, 승용차가 혼재된 고속도로/도로 영상입니다.

### 후보 영상 (우선순위 순)

**후보 1**: 혼합 교통 (트럭+버스+승용차+오토바이)
```bash
yt-dlp -f "bestvideo[height<=720]+bestaudio/best[height<=720]" --download-sections "*0:00-0:30" --merge-output-format mp4 -o "data/samples/mixed_traffic.mp4" "https://www.youtube.com/watch?v=wqctLW0Hb_0"
```

**후보 2**: 고속도로 혼합 교통
```bash
yt-dlp -f "bestvideo[height<=720]+bestaudio/best[height<=720]" --download-sections "*0:10-0:40" --merge-output-format mp4 -o "data/samples/mixed_traffic.mp4" "https://www.youtube.com/watch?v=PJ5xXXcfuTc"
```

**후보 3**: 도시 교통 (다양한 차종)
```bash
yt-dlp -f "bestvideo[height<=720]+bestaudio/best[height<=720]" --download-sections "*0:00-0:30" --merge-output-format mp4 -o "data/samples/mixed_traffic.mp4" "https://www.youtube.com/watch?v=MNn9qKG2UFI"
```

### 영상 선정 기준
다운로드된 영상에서 YOLO11으로 빠르게 10프레임만 테스트하여 **3종 이상의 클래스가 감지되는 영상**을 선택:

```python
from ultralytics import YOLO
import cv2

model = YOLO("yolo11l.pt")
cap = cv2.VideoCapture("data/samples/mixed_traffic.mp4")
classes_found = set()

for i in range(10):
    ret, frame = cap.read()
    if not ret: break
    results = model.predict(frame, conf=0.4, classes=[0,1,2,3,5,7], device="cuda:0", verbose=False)
    for box in results[0].boxes:
        cls_names = {0:"person", 1:"bicycle", 2:"car", 3:"motorcycle", 5:"bus", 7:"truck"}
        classes_found.add(cls_names.get(int(box.cls.item()), "unknown"))

cap.release()
print(f"감지된 클래스: {classes_found}")
# 3종 이상이면 이 영상 사용
```

**3종 미만이면 다음 후보로 넘어가세요.** 어떤 후보도 안 되면 YouTube에서 "highway traffic truck bus motorcycle drone" 검색 후 720p 30초 영상을 직접 선정.

### 선정된 영상이 있으면
기존 `highway_sample.mp4`는 유지하고, **새 영상 `mixed_traffic.mp4`를 데모용으로 사용**합니다.

## 2단계: 한국어 폰트 확인

Windows에서 사용 가능한 한국어 폰트를 확인하고, 스크립트의 `FONT_PATH`를 수정해야 합니다.

```python
# 폰트 경로 확인 스크립트
import os
font_candidates = [
    "C:/Windows/Fonts/malgun.ttf",      # 맑은 고딕
    "C:/Windows/Fonts/NanumGothic.ttf",  # 나눔고딕
    "C:/Windows/Fonts/gulim.ttc",        # 굴림
]
for f in font_candidates:
    if os.path.exists(f):
        print(f"사용 가능: {f}")
```

**`scripts/generate_demo_video.py`의 40행 부근에서 `FONT_PATH`를 수정:**

```python
# macOS 전용 (현재 값)

# Windows용으로 변경
```

## 4단계: 영상 품질 검증

생성된 영상에서 아래 항목을 확인:

1. **한국어 라벨 정상 표시**: "승용차 92%", "트럭 87%" 등 한국어가 물음표 없이 렌더링되는지
2. **HUD 표시**: 상단에 "HighLink AI | Frame N" + FPS 표시
3. **클래스별 카운트**: 상단 바에 "승용차: N | 트럭: N" 등 집계 표시
4. **워터마크**: 하단 우측에 "HighLink PoC — Confidential" 표시
5. **FPS**: RTX 4060에서 예상 28+ FPS가 HUD에 표시되는지
6. **H.264 변환**: ffmpeg가 설치되어 있으면 자동 변환됨. 없으면 `pip install imageio[ffmpeg]` 후 ffmpeg 설치 권장

1. **한국어 폰트가 핵심**: 이전 MPS 테스트에서 OpenCV putText가 한국어를 렌더링하지 못해 물음표로 표시됨. 현재 스크립트는 PIL(Pillow)을 사용하므로 정상 동작해야 하나, Windows 폰트 경로 변경이 필수
