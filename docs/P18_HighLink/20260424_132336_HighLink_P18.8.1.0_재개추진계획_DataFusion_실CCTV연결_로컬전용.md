> **세션**: P18.8.1.0 (예정, 새 채팅) | **작성일**: 2026-04-24 | **공백**: 2026-04-16 → 2026-04-24 (8일)

## 1. Context

8일간 중단된 HighLink(P18) 프로젝트를 재개한다. 직전 세션(P18.7.2.0)에서 **로컬 MVP 전체(Phase 1~7)가 구현 완료**됐고, HTML 테스트 결과 보고서가 GitHub Pages에 배포된 상태다. 직전 대화의 최종 합의 내용:

> ⚠️ **로컬 전용 방침 (사용자 확정)**: 지난 시도에서 Cloudflare Tunnel 등 외부 접속 기능이 프로젝트 진행을 방해했다. 이번 재개에서는 **로컬 서버에서만 동작하는 기능 구현에 집중**한다. 터널링, 원격 접속, 외부 노출은 모두 **범위 외**. Phase 1에서 이미 터널/폴백 코드를 제거했으며, 이 로드맵 어디에도 외부 접속 관련 작업은 포함하지 않는다.


- **기술 스택**: YOLO11L + ByteTrack + Streamlit + EfficientNet-B0 2단계 분류기 (MVP)
- **라이선스 전략**: MVP 단계는 AGPL 포함 모든 라이선스 무제약. 프로덕션 전환 시 RT-DETR(Apache 2.0)로 교체하며 자연스레 해소.
- **파트너 구조**: 관납 파트너사가 관제시스템 접근·실 CCTV 네트워크·GS/CC 인증 담당, 자사는 AI 엔진·소프트웨어 담당.
- **프로덕션 전환 로드맵**: Python → Java Spring + React, PyTorch → TensorRT, MJPEG → WebRTC.

### 미해결 우선순위 작업 (P18.7.2.0 세션지침에서 이관)

1. **Git 커밋 정리** — Phase 1~6 신규 파일 30+개가 `git status`에 누적. 압축 시 유실 방지 위해 조기 커밋.

6. **F3/F4 테스트 케이스 확장** — 현재 3건 → 6건 이상.

## 3. 실행 순서 및 의존성

```
Phase A (재개 준비)
    │
    ▼
Phase B (DataFusion + 실 CCTV) ← API 키 필요, 사용자 확인 필수
    │
    ├─── 병렬 ──── Phase D (F5 통합 + 테스트)
    │
    ▼
Phase C (멀티 카메라 확장) ← Phase B의 단일 스트림 안정화 이후
    │
    ▼
Phase E (파트너사 데모 패키지) ← Phase B+C+D 완료 후
```

**크리티컬 패스**: A → B → C → E (총 ~9시간)
**병렬 가능**: D는 B와 병행 가능

## 6-1. 범위 외 (하지 않는 것)

사용자 지시에 따라 이번 로드맵에서 **명시적으로 배제**:

- ❌ Cloudflare Tunnel, Tailscale, ngrok 등 모든 원격 터널링
- ❌ 외부 도메인/서브도메인 노출
- ❌ 스마트폰·원격 PC에서 관제 대시보드 접속
- ❌ Cross-origin 이슈 해결 작업
- ❌ 인증서/HTTPS 설정 (로컬은 HTTP로 충분)
- ❌ 방화벽·포트 포워딩 설정

이전 P18.4wc 세션에서 MJPEG 터널링 이슈(TRAP-002)로 많은 시간을 소비했다. 동일 실수 방지를 위해 **관제 대시보드 접속은 `localhost` 또는 같은 LAN 내부에서만** 허용한다. 모든 기능 검증은 맥 로컬(`localhost:8501`)에서 완결한다.
