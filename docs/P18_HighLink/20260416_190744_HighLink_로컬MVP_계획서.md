# HighLink 로컬 MVP 계획서

> **세션**: P18.7.1.0 | **작성일**: 2026-04-16 | **버전**: v4

## 1. Context

파트너사(관급 영업)가 요구한 **F1~F6 핵심 기능**을 로컬 서버에서 모두 동작하도록 구현하는 것이 목표.
현재 F1~F3+F6만 구현 완료, **F4(정체/사고)**와 **F5(악천후 대응)**가 미착수.
ITS 오픈데이터 API 중 CCTV만 사용 중이고, VDS/돌발정보/기상청/공사정보 API는 config 플래그만 존재.
원격 터널링(Cloudflare, Tailscale)은 **보류** — 로컬 서버 기능 구현에 집중.

### 파트너 요구 기능 (F1~F6) 현황

| # | 기능 | 설명 | 현재 상태 |
|---|------|------|----------|
| F1 | 양방향 통행 인식 | 차량 추적→차선별 주행 방향 식별 | ✅ WrongWayDetector 캘리브레이션 |
| F2 | 역주행 감지 | 정상 흐름 반대 이동 차량 탐지+알림 | ✅ 구현+테스트 완료 |
| F3 | 보행자/이륜차 인지 | 고속도로 보행자, 자전거, 오토바이 감지 | ✅ 구현+과감지 수정 완료 |
| F4 | 정체/사고 인지 | 교통 정체 판별 및 사고 발생 감지 | ❌ 미착수 (config만) |
| F5 | 악천후 대응 | 야간/눈/비/안개에서도 F1~F4 유지 | ❌ 미착수 (기상청 API config만) |
| F6 | 이벤트 알림 | 사고/역주행/보행자 감지 시 실시간 알림 | ✅ Streamlit event_panel 완료 |

### ITS API 활용 현황

| 데이터 | 용도 | 상태 | config 키 |
|--------|------|------|----------|
| CCTV 영상 | AI 분석 주 입력 | ✅ `cctv_client.py` 구현 | `its_api.base_url` |
| 교통소통정보/VDS | 정체 ground truth, 교차검증 | ❌ config만 | `fusion.use_vds: false` |

| 전국표준노드링크 | CCTV→도로 방향 자동 매핑 | ❌ 미착수 | — |

## 2. 관제 대시보드 설계 — 업계 벤치마킹 기반

### 2.1 벤치마킹 대상

| 시스템 | 핵심 특징 |
|--------|----------|
| **한국도로공사** | 10,000+ CCTV, 24/7 운영, ROAD PLUS 앱 연동 |
| **KT AI Traffic** | AI 영상분석→관제 데이터 변환, C-ITS 긴급차량 우선 신호 |
| **Kapsch DYNAC-SCADA** | 4~6 키스트로크 내 명령 실행, 2~3초 응답 |

| **Cisco Smart+Connected** | "단일 유리판(single pane of glass)" 통합 관제 |

### 2.2 핵심 설계 원칙 (관제요원 피로도 최소화)

#### A. 3단계 알림 체계 + 에스컬레이션

```
┌─────────────────────────────────────────────────────┐
│  🔴 CRITICAL (위험)  — 즉시 개입 필요                  │
│     역주행, 보행자, 중대사고                           │
│     → 화면 중앙 팝업 + 고유 경고음 + 카메라 자동 풀업    │
│     → 30초 미확인 시 상위 관제사/교대조장에 에스컬레이션  │
├─────────────────────────────────────────────────────┤
│  🟡 WARNING (경고)   — 확인 필요                      │
│     경미한 사고, 정체 시작, 이륜차                      │
│     → 사이드 패널 하이라이트 + 부드러운 알림음           │
│     → 60초 미확인 시 에스컬레이션                       │
├─────────────────────────────────────────────────────┤
│  🔵 INFO (정보)     — 참고용                          │
│     정체 해소, 날씨 변화, 시스템 상태                   │
│     → 타임라인에 기록, 별도 알림음 없음                  │
└─────────────────────────────────────────────────────┘
```

#### B. 주의력 관리 (Attention Management)

- **자동 카메라 풀업**: 이벤트 발생 시 해당 CCTV가 자동으로 메인 화면에 표시
- **배경 투명도 변화**: 비핵심 요소는 서서히 페이드하여 핵심 알림에 집중 유도
- **최소 자극 원칙**: 깜빡임/맥동은 CRITICAL에만 사용, 남용 시 "양치기 소년" 효과
- **인지 부하 분산**: 한 화면에 최대 6개 카메라 피드, 나머지는 이벤트 기반 자동 전환

#### C. 정보 밀도 vs 인지 부하

- **접이식 패널**: 상세 정보는 클릭 시 펼침, 기본은 요약만
- **컨텍스트 표시**: 현재 작업에 필요한 데이터만 동적 표시
- **구간 기반 우선순위**: 교통량/사고 이력 기반으로 감시 구간 자동 정렬

#### D. 24/7 운영 최적화

- **다크 모드 기본**: 장시간 근무 시 눈 피로 감소 (조도 500~700 lux 기준)
- **교대 인수인계**: 활성 이벤트 목록 + 미처리 항목 체크리스트 자동 생성
- **이벤트 타임라인**: 시간순 이벤트 로그로 교대 시 맥락 전달

### 2.6 관제시스템 기본 기능 (멀티 CCTV 관제)

업계 벤치마킹 결과, 관제시스템이 갖춰야 할 **기본 기능**을 아래와 같이 정의한다.

#### A. 멀티 카메라 동시 모니터링

```
┌─────────────────────────────────────────────────┐
│  카메라 그리드 (기본 2x3, 최대 4x4 전환 가능)     │
│  ┌────────┐ ┌────────┐ ┌────────┐               │
│  │ CAM-01 │ │ CAM-02 │ │ CAM-03 │               │
│  │ 경부 수원│ │ 경부 오산│ │ 영동 여주│               │
│  │ 🟢 정상 │ │ 🔴 역주행│ │ 🟢 정상 │               │
│  └────────┘ └────────┘ └────────┘               │
│  ┌────────┐ ┌────────┐ ┌────────┐               │
│  │ CAM-04 │ │ CAM-05 │ │ CAM-06 │               │
│  │ 중부 충주│ │ 서해안  │ │ 남해 순천│               │
│  │ 🟡 정체 │ │ 🟢 정상 │ │ 🟢 정상 │               │
│  └────────┘ └────────┘ └────────┘               │
│                                                  │
│  [1x1] [2x2] [2x3] [3x3] [4x4]  ← 그리드 전환   │
└─────────────────────────────────────────────────┘
```

**벤치마킹 근거**:
- Kapsch DYNAC-SCADA: 관제요원당 4~6개 카메라 동시 모니터링
- 한국도로공사: 구간별 CCTV 그룹핑 + 이벤트 시 자동 전환
- Cisco: 타일 레이아웃 + 클릭→풀스크린 확대

**구현 사양**:
- ITS API에서 CCTV 목록 자동 수집 → 지역/고속도로별 필터링
- 그리드 레이아웃 전환: 1x1(전체), 2x2, 2x3(기본), 3x3, 4x4
- 각 타일에 카메라명 + 상태 인디케이터(🟢정상/🟡경고/🔴위험) 표시
- 각 타일 MJPEG 스트림 (Tornado 핸들러로 카메라별 독립 스트림)

#### B. 전체화면 확대 (Single Camera Focus)

- 그리드 내 카메라 타일 **클릭 → 해당 카메라 전체화면 확대**
- 전체화면에서:
  - AI 감지 결과 오버레이 (바운딩 박스, 트랙 ID, 이벤트 경고)
  - FPS/모델 정보/신뢰도 HUD
  - PTZ 제어 (지원 카메라의 경우)
  - [ESC] 또는 [X]로 그리드 복귀
- **이벤트 기반 자동 풀업**: CRITICAL 이벤트 발생 시 해당 카메라 자동 전체화면
  - 관제요원이 다른 카메라 확인 중이면 자동 풀업 대신 알림 패널에 강조
  - 설정에서 자동 풀업 ON/OFF 가능

#### C. 지속 감시 지정 (Watch List)

- 관제요원이 특정 카메라를 **"지속 감시"로 지정** 가능
- 지속 감시 카메라는:
  - 그리드에서 항상 상단 고정 (다른 카메라가 자동 교체되어도 유지)
  - 테두리 강조 (파란색 아웃라인)
  - AI 감지 sensitivity 자동 상향 (conf_threshold 하향)
  - 이벤트 기록 강화 (INFO 등급도 기록)
- 최대 3개까지 지속 감시 지정

#### D. 카메라 순찰 (Auto-Patrol)

- 지정된 카메라 그룹을 **자동 순회** (N초 간격 전환)
- 순찰 중 이벤트 발생 시 자동 정지 → 해당 카메라 고정
- 순찰 경로: 고속도로 구간순, 위험도순, 커스텀 순서
- Iteris ClearGuide의 Route Alert 개념 차용

#### E. 이벤트 기반 카메라 전환 (Smart Switching)

- AI 이벤트 발생 → 그리드 내 비활성 슬롯에 해당 카메라 자동 배치
- 관제요원이 확인(✅) 누르면 원래 카메라로 복귀
- 복수 이벤트 동시 발생 시 우선순위 기반 큐잉
- Kapsch 방식: 4~6 키스트로크 내 이벤트 대응 완료

#### F. 카메라 관리

- ITS CCTV API에서 카메라 목록 실시간 갱신 (5분 주기)
- 카메라 상태 모니터링: 연결 끊김, 영상 품질 저하, 야간 모드 전환
- 카메라 그룹핑: 고속도로별, 구간별, 커스텀 그룹
- 즐겨찾기: 자주 확인하는 카메라 빠른 접근

#### G. 이벤트 대응 워크플로우 (벤치마킹: Kapsch + 도로공사)

```
이벤트 발생
    │
    ▼
[자동] 알림 표시 + 카메라 풀업 + 이벤트 클립 녹화 시작
    │
    ▼
[관제요원] 확인 (✅ 실제 이벤트 / ❌ 오탐)
    │
    ├── 오탐 → 피드백 기록 (AI 정확도 개선용) → 종료
    │
    └── 실제 이벤트
         │
         ▼
    [관제요원] 심각도 분류 (자동 분류 수정 가능)
         │
         ▼
    [관제요원] 대응 조치 선택
         ├── 🚨 사고 보고 (상위 기관 전파)
         ├── 🚑 출동 요청
         ├── 📢 전광판/VMS 메시지 표출
         ├── 🚧 차로 통제 요청
         └── 📝 메모 추가
         │
         ▼
    [자동] 이벤트 이력 저장 + 영상 클립 첨부
         │
         ▼
    [관제요원] 이벤트 종료 처리
```

### 2.7 디자인 테마 방침

> **⚠️ MVP 제외 항목**: 디자인 테마(색상 팔레트, 타이포그래피, 컴포넌트 스타일)는 나중에 테마 시스템으로 일괄 변경할 수 있도록 **CSS 변수/테마 파일로 분리**하되, MVP에서는 기본 다크 테마만 적용.

> - 모든 색상 값은 CSS 변수(`--color-critical`, `--color-warning` 등)로 관리
> - 컴포넌트 스타일은 테마 파일(`theme.css` 또는 Streamlit custom theme)에서 일괄 정의
> - MVP에서는 기능 동작이 우선, 시각적 완성도는 후속 작업
