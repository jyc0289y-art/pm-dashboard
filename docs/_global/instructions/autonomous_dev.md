# 자율개발모드 (Autonomous Development Mode)

사용자가 "자율개발모드", "자율개발", "묻지말고 진행", "autonomous dev" 등을 요청하면 이 모드를 활성화한다.

## 핵심 원칙

1. **질문하지 않고 진행**: 사용자에게 확인/허가를 구하지 않고, 스스로 판단하여 최선의 구현을 선택
2. **할 일을 스스로 찾아서 진행**: 명시적 지시가 없어도 프로젝트 개선점을 탐색하고 구현
3. **주기적 커밋 & 푸시**: 의미 있는 단위로 git commit + push하여 모바일에서 확인 가능하게
4. **프로젝트 폴더 외부 수정 금지**: 작업 대상 프로젝트 디렉토리 외의 파일은 수정하지 않음

## 병렬모드 (Parallel Mode)

사용자가 "자율개발병렬모드", "병렬로 진행", "parallel autonomous" 등을 요청하면 활성화.

### 병렬 작업 흐름
1. **작업 분할**: 독립적으로 수행 가능한 작업들을 식별
2. **에이전트 분배**: 각 작업을 별도 Agent로 할당 (최대 7개 동시)
3. **호이스팅 방지**: 모든 에이전트에게 arrow function / setTimeout 패턴 사용 지시
4. **중간 검증**: 에이전트 완료 시마다 console error 확인 → 즉시 수정
5. **일괄 커밋**: 모든 에이전트 완료 후 빌드 검증 → 커밋 & 푸시
6. **반복**: 다음 라운드 에이전트 실행

### 에이전트 지시 템플릿
각 에이전트에 반드시 포함할 내용:
- "Read each file BEFORE editing"
- "Use arrow functions `() => myFunc()` for ALL event listener bindings to avoid hoisting issues"
- "Keep existing features working"
- "After editing, run `npx vite build` to verify no build errors"
- "Do NOT commit or push"

### 충돌 방지
- 에이전트 간 파일 수정 충돌 최소화: 가능하면 각 에이전트가 담당 모듈만 수정
- index.html은 여러 에이전트가 수정할 수 있으므로, 완료 후 충돌 해결 필요
- 에이전트 완료 후 빌드 실패 시 호이스팅 에러가 가장 흔한 원인 → typeof 체크 또는 setTimeout 패턴으로 해결

## 작업 우선순위

1. 사용자가 명시적으로 요청한 기능/수정 (최우선)
2. 기존 코드의 버그 수정
3. 미구현 기능 완성
4. 새로운 기능 추가
5. UX/UI 개선
6. 성능 최적화

## 작업 흐름 (단일 모드)

1. **탐색**: 코드베이스를 분석하여 미구현·버그·개선점 발견
2. **구현**: 묻지 않고 바로 구현
3. **검증**: preview 도구로 동작 확인 (빌드 에러, 런타임 에러 체크)
4. **커밋**: 의미 있는 단위로 커밋 메시지 작성 후 commit + push
5. **보고**: 완료된 작업을 간결하게 사용자에게 보고
6. **반복**: 다음 작업으로 이동

## 주의사항

- 기존 기능이 동작하는 코드를 깨뜨리지 않도록 주의
- 대규모 아키텍처 변경은 사용자 확인 후 진행 (이것만 예외)
- 커밋 메시지는 영어로, 변경 내용을 명확하게 기술
- 빌드 실패 시 즉시 수정 후 재커밋
