# AI 작업의 구조적 한계와 측정 기반 개선 원칙

> **배경**: P21 FEP 영상 제작 과정에서 반복적으로 발견된 AI 구조적 한계와 개선 원칙을 전역 지침으로 격상한다. (2026-04-17)

## 🚨 AI 구조적 한계 (매 세션 인식 필수)

AI(Claude)는 다음을 할 수 없다:
1. **오디오를 들을 수 없다** → 대사/음성 타이밍은 반드시 whisper 등 전사 도구로 텍스트화 후 매칭
2. **영상 재생을 실시간 관찰할 수 없다** → 정적 프레임 추출 + side-by-side 비교만 가능
3. **시각적 "자연스러움"을 평가할 수 없다** → 정량 지표는 참고, 사용자 시각 검증이 최종 판정
4. **멀티모달 감각적 품질 판단 불가** → "메트릭 통과 ≠ 품질 통과"

## ⛔ 반복 실수 패턴 (영구 금지)

### Anti-Pattern 1: "정량 지표 100%" 자만
- **증상**: 메트릭이 녹색이지만 사용자는 "전혀 안 됨" 판정
- **원인**: 메트릭 기준이 실제 품질 기준과 불일치
- **사례 (FEP)**: silencedetect "nearest timing" 매칭이 21/21 "100% 정합" 보고했으나 실제로는 14~19초 오차. 내레이터/대사 혼재 오디오에서 "가장 가까운 발화"가 내레이터 발화에 매칭됨.
- **대응**: "개선됨" 선언 금지 → "개선 시도(미검증)" 표기 → 사용자 시각 검증 통과 후 "개선됨" 표기

### Anti-Pattern 2: "코드 반영 = 시각 구현" 착각
- **증상**: 벤치마킹 규칙을 코드에 구현했으나 프레임에서 규칙이 지각되지 않음
- **원인**: 에셋 한계, 좌표 조합, 렌더링 중첩 등
- **사례 (FEP)**: 말풍선 꼬리→입 로직 코드에 있었으나 에셋 꼬리 + 추가 꼬리 중첩으로 "드래그앤드롭 모양새"
- **대응**: 코드 수정 후 반드시 대표 프레임 렌더링 검증

### Anti-Pattern 3: 텍스트 벤치마킹 → 실측 없이 구현
- **증상**: 블로그/설명 기반 학습으로 얻은 규칙을 추정값으로 구현 → 방향 오류 발생
- **사례 (FEP)**: "자막은 대사 직후" 학습 → 측정 없이 "대사 0.4초 전"으로 반대 방향 구현 (500ms+ 역방향 오류)
- **대응**: 신규 규칙 도입 시 반드시 리버스 엔지니어링(yt-dlp + whisper + ffmpeg scdet)으로 실측 확보

### Anti-Pattern 4: 같은 이슈 3번 이상 같은 방법으로 수정 시도
- **증상**: 사용자가 동일 문제를 반복 지적하는데 같은 접근 고집
- **원인**: 메트릭/도구/접근 자체의 한계 인정 부족
- **대응**: **2회 재발 시점에서 접근 교체 강제**. 다른 측정 도구/다른 로직/다른 원리.

## ✅ 측정 기반 개선 원칙

### 1. 측정 파이프라인 우선
신규 규칙/상수 도입 시 반드시:
```
yt-dlp → 레퍼런스 영상 다운로드
faster-whisper → 한국어 word-level 전사
ffmpeg scdet → 씬 컷 감지
OpenCV → 자막 프레임 감지
VTT 파서 → 수동 자막 타이밍 (가용 시)
SequenceMatcher → 텍스트 내용 매칭 (단순 타이밍 매칭 금지)
```

### 2. 매칭은 "내용 기반"만 허용
- ❌ silencedetect + "nearest timing" 단독 매칭 → 화자 구분 불가 → 내레이터 오매칭
- ✅ whisper 전사 텍스트 + manifest 텍스트 SequenceMatcher 매칭
- ✅ 합쳐진 긴 세그먼트는 word-level sliding window로 phrase 시작점 추출

### 3. side-by-side 비교 필수
- 개선 전/후 **same-timestamp** 프레임 나란히 추출
- 정량 메트릭 + 시각 대조를 **쌍으로** 제시
- 단일 프레임 스팟 체크는 부분 검증에 불과

### 4. 사용자 검수 이전 자체 QA 절대 필수
- Layer 1 (기술 스펙) → Layer 2 (무결성) → Layer 3 (지각 품질) → Layer 4 (서사) → Layer 5 (회귀) 모두 통과
- Known Issues 리스트 명시
- 집중 검수 포인트 3~5개 제시

### 5. 사용자 피드백은 항상 옳다
- 사용자가 "안 됨" 지적 → 내 메트릭이 불완전하다는 증거
- 반박 금지, 접근 재검토 즉시 시작
- 같은 방법으로 "이번엔 진짜" 주장 금지

## 🎯 적용 대상

이 원칙은 다음 작업 전반에 적용된다:
- 영상/오디오/음성 관련 모든 작업 (P21 FEP, P16 CineLink 등)
- 시각적 품질 판단이 필요한 UI/그래픽 작업
- 타이밍/동기화가 중요한 모든 멀티미디어 작업
- 사용자 체감 품질이 최종 판정인 모든 작업
