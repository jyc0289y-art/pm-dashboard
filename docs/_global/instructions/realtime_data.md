# 실시간 데이터 작업 규칙 (항공권, 호텔, 환율 등)

**원칙: LLM은 실시간 가격/재고/시세를 절대 "기억"으로 답하지 않는다.**

모든 실시간 데이터 작업은 아래 파이프라인을 따른다:

**Step 1 — Worker가 조사 구조화 (할루시네이션 안전)**
- `ask_worker` 또는 `gemini_ask`에게: 검색해야 할 사이트 목록, 비교 항목 틀, 체크리스트 생성
- 예: "인천→도쿄 항공편 비교 시 확인할 사이트와 항목 정리해줘"
- Worker는 구조/템플릿만 만들고, 가격 숫자는 절대 생성 금지

**Step 2 — Claude가 실제 데이터 수집 (브라우저 또는 웹검색)**
- Chrome MCP(`computer`, `navigate`, `read_page`) 또는 `WebSearch`로 실제 사이트 조회
- Google Flights, Skyscanner, 네이버 항공권 등에서 실시간 가격 확인
- 스크린샷으로 가격 증거 확보

**Step 3 — Worker가 결과 정리 (할루시네이션 안전)**
- Claude가 수집한 실제 데이터를 `ask_worker`에게 넘겨 비교표/정리표 생성
- Worker는 주어진 데이터만 정리, 새 데이터 추가 금지

**Step 4 — Claude가 분석/추천**
- 실제 데이터 기반으로 최적 옵션 분석, 사용자에게 추천

**절대 금지:**
- Claude/Worker 어느 쪽이든 "기억"이나 "학습 데이터"로 가격/시세 답변
- 실시간 검증 없이 숫자가 포함된 답변 제공
- "대략 ~원 정도입니다" 식의 추정 가격 제시 (검증 불가)

**허용:**
- "일반적으로 성수기가 비수기보다 비쌉니다" 같은 상식적 경향 설명
- "이 노선은 LCC가 운항합니다" 같은 구조적 정보 (가격 제외)
