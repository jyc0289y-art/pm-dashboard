## 핵심 결정사항
1. **사업 구조**: SL Corporation = 개발사, 파트너사 = 관급 납품/영업
2. **LLM 제외**: 이벤트 메시지는 템플릿 기반
3. **듀얼 모드**: engine/(공통) + app/(독립제품). 모드B는 gRPC만 노출
4. **핵심 기능**: F1~F6 (양방향, 역주행, 보행자, 정체/사고, 악천후, 알림)
5. **Phase별 언어 전환**:
   - Phase 0-1: Python 전체 (PoC 검증)
   - Phase 2: DeepStream(추론) + Java Spring Boot(백엔드) 도입
   - Phase 3: C++ DeepStream + Java eGovFrame (관급 표준 완전 준수)
6. **관급 표준**: eGovFrame, GS인증, CC인증, Tibero 호환 고려
7. **Apple Silicon MPS 한계**: PyTorch MPS 백엔드 미성숙 → 개발용으로만 사용, 운영은 CUDA GPU 필수

## Phase 0 사전 테스트 결과 (2026-04-10)
- **환경**: Apple Silicon M5, MPS, YOLO11L, ultralytics 8.4.36, torch 2.11.0
- **이미지 테스트**: person 감지 conf 0.67~0.91, 소형객체 0개
- **비디오 FPS**: 9.7 FPS (avg 102.7ms, p95 119.0ms) — 목표 15fps 미달
- **감지 품질**: 100프레임에서 921건 (9.2/프레임), car 920 + truck 1
- **소형 객체**: 306/921 (33%) → SAHI 도입 권장
- **결론**: MPS에서 목표 FPS 미달 → CUDA GPU 필수, SAHI Phase 2에서 도입
- **MPS 느린 이유**: PyTorch MPS 미성숙 + ANE 미활용 (CoreML 변환 시 25-35fps 가능하나 운영은 CUDA이므로 불필요)
