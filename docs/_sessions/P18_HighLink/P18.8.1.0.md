## TRAP-011 — ITS API 엔드포인트 오류 (이번 세션 발견)
- **현상**: P18.7.2.0 구현의 `incident_client.py`와 `vds_client.py`가 잘못된 엔드포인트 사용 → 실 API 호출 시 401/4004 오류
- **원인**: 문서 없이 추측 기반으로 엔드포인트명 결정 (`/incidentInfo`, `/trafficInfo`) — 실제는 `/eventInfo`, `/vdsInfo`
- **수정**: 실 API 요청/응답 probing으로 정확한 엔드포인트와 필드명 확정 → 코드 교정
- **예방**:
  - ✅ ITS 오픈데이터 포털 목록에서 "차량검지정보" = `/vdsInfo`, "돌발상황정보" = `/eventInfo`
  - ✅ 새 API 클라이언트 구현 시 먼저 `curl`로 실 호출해서 응답 구조 확보 후 파싱 작성
  - ✅ 응답 필드 매핑은 한국어 이름 그대로 사용하는 경우가 많음 ("사고"/"공사"/"기상")

### 기상청 API 동작 특성
- 16:16에 호출 → 15:00 발표 데이터를 반환 (40분 룰 적용됨 = 16:40 이후 16:00 데이터 가능)
- `fetch_weather(nx, ny)` 호출이 캐시 채움 → `get_weather_for_camera()`는 캐시 조회용
- 포인트 추가는 `add_watch_point()` 필요

### CCTV 스트림 특성
- URL 형식: `http://cctvsec.ktict.co.kr/{번호}/{인코딩경로}`
- 해상도: **720x480 (SD)** — Full HD가 아님 (파트너사 CCTV는 1080p 예상)
- 프로토콜: MJPEG over HTTP (별도 auth 불필요, URL 자체에 토큰 내장)
- URL 유효시간: `its_api.refresh_interval_sec: 300` (5분 주기 갱신 필요)
