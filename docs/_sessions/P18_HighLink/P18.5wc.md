## 작업 요약
PM.2wc-REQ-001 (F3 bicycle 과감지 수정) 구현 완료.
자전거 라이더가 person + bicycle/motorcycle 으로 이중 감지되어 중복 이벤트가 발생하는 문제를 3가지 방법으로 해결:
1. IoU 기반 중복 감지 억제 (person과 이륜차 bbox가 겹치면 person 억제)
2. bicycle 클래스에 별도 높은 confidence threshold (0.6) 적용
3. detection classes에 bicycle(1) 추가하여 명시적 감지 후 필터링

## 주요 결과물
- `engine/rules/pedestrian.py` — `_compute_iou()` 함수 추가, `_suppress_cyclist_duplicates()` 메서드 추가, `update()` 3단계 파이프라인으로 리팩토링
- `engine/config.yaml` — `bicycle_min_confidence: 0.6`, `overlap_iou_threshold: 0.3` 설정 추가, detection classes에 bicycle(1) 추가

## 핵심 결정사항
1. **IoU threshold 0.3**: person bbox와 bicycle/motorcycle bbox가 30% 이상 겹치면 동일 대상으로 판단하고 person 이벤트 억제
2. **bicycle confidence 0.6**: 기본 0.5보다 높게 설정하여 오탐(일반 차량을 bicycle로 오인) 방지
3. **person 억제, 이륜차 유지**: 고속도로에서 이륜차 자체가 위험 이벤트이므로, 라이더(person)까지 별도 이벤트 발생 불필요
4. **detection classes에 bicycle(1) 추가**: 기존 [2,3,5,7,0]에서 1(bicycle) 누락 → [0,1,2,3,5,7]로 보정
