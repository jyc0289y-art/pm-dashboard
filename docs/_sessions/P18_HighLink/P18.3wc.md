# P18.3wc — 트럭 오분류 수정 + F1/F2/F3 룰엔진 통합 테스트 + 이벤트 경고 시각화

## 메타데이터
- 생성: 2026-04-10
- 포크원: P18.2wc
- 자식: P18.4wc

## 작업 요약
P18.2wc에서 포크. 회사 PC용 P번호 작성지침을 프로젝트 내 배치·푸시. CCTV 3대 이슈(프레임 드랍, 소형 객체 미감지, 대형 트럭→승용차 오분류) 수정. 프로젝트 방향을 기획문서(F1~F6) 기준으로 재정렬. F1+F2(역주행 감지) 및 F3(보행자/이륜차 감지) 룰엔진 통합 테스트 스크립트 작성·실행. 이벤트 경고 박스 시각화 강화. F3 테스트용 YouTube 영상 3건 다운로드 완료(미실행).

## 주요 결과물
- `.claude/CLAUDE.md` — 프로젝트 내 P번호 작성지침 (회사 PC 접근 가능)
- `engine/detection/detector.py` — `reclassify_truck()` 임계값 수정 (area<0.003, ratio<1.2)
- `engine/detection/vehicle_classifier.py` — pickup→truck 매핑 + bbox_ratio 가드 추가
- `app/streamlit/app.py` — 모드C 스트리밍 세션스테이트 기반 정지 제어 + 재연결 제한
- `scripts/test_rules.py` — F1+F2+F3 통합 테스트 스크립트 (YOLO11+ByteTrack+룰엔진)
  - 이벤트 경고 박스: 이중 빨간 테두리 + 깜빡임 + 상단 배너 + HUD 경고 표시
  - F3 타겟(보행자/자전거/이륜차) 주황색 하이라이트
  - 결과 파일명 타임스탬프: `{YYYYMMDD_HHMMSS}_{test_type}_{source}.mp4`
- `data/test_videos/` — F3 테스트 영상 3건 다운로드 완료
  - `pedestrian_dashcam.mp4` (640x360, 25fps, 64s)
  - `highway_bicycle.mp4` (640x360, 30fps, 133s)
  - `highway_tunnel_cctv.mp4` (640x360, 30fps, 134s)

## 핵심 결정사항
1. **Phase 용어 통일**: Phase 0/1/2/3은 기획문서(`HIGHWAY_AI_SURVEILLANCE_PROJECT.md`)의 정의만 사용. 세션 내부 Phase 구분 금지.
2. **데모영상 맥에서 직접**: 회사 PC에 데모 영상 요청 금지. 맥 MPS로 로컬 테스트 후 iCloud에 저장. 회사 PC = 서버 역할만.
3. **프로젝트 방향 재정렬**: 차종 분류 사이드트랙 → F1~F6 핵심 요구사항 중심으로 복귀
4. **대형 트럭 오분류 원인**: `reclassify_truck()`의 `relative_area < 5%` 조건이 CCTV에서 대형 트럭도 포착 → 0.3%로 대폭 축소
5. **뉴스 클립 테스트 한계**: CCTV 구간이 짧아 보정 벡터 부족 → 실제 ITS CCTV 연속 스트림 필요
