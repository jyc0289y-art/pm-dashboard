## 프로젝트 전체 맥락 (P10.1wc~P10.9wc 누적)

### Hodlum이란?
- **Ludlum Model 3030-2** Alpha-Beta Sample Counter용 커스텀 데이터 로깅/보고서 소프트웨어
- 장비와 RS-232 (USB-to-Serial) 통신으로 파라미터 읽기 + 측정 실행 + 결과 저장/보고
- 개발사: SL Corporation / 브랜드명: Hodlum (회사명 + Ludlum)
- **장비 설정값 변경 금지**: READ + COUNT 명령만 사용. D0-D3, B0/B1, X0/X1, S* 등 설정 변경 명령 차단

### 기술 스택
- **Python 3.10+**, PyQt6 (데스크톱 GUI), PyQt6-WebEngine (보고서 미리보기)
- pyserial (RS-232 통신), SQLite (측정 데이터 저장)
- Jinja2 + WeasyPrint (HTML/PDF 보고서), PyYAML (설정 파일)
- PyInstaller (배포 빌드)
- **디자인**: HOBIS Design System (사이버펑크/테란 테마, Orbitron + Share Tech Mono 폰트)

### 보고서 테마 시스템 상세 (이번 작업의 핵심)
- **위치**: `gui/report/generator.py` (line 36~101)
- **런타임 경로 분기**:
  ```python
  if getattr(sys, "frozen", False):
      THEMES_DIR = Path(sys._MEIPASS) / "gui" / "report" / "themes"
  else:
      THEMES_DIR = Path(__file__).parent / "themes"
  ```
- **_scan_themes()**: `THEMES_DIR.glob("*.css")` → 각 CSS 상단 `@theme-name`, `@theme-description` 메타데이터 파싱 → `REPORT_THEMES` dict 구성
- **_get_theme_css()**: CSS 파일 읽기 → `string.Template.safe_substitute()` 로 `${green}`, `${cyan}`, `${font_import}` 등 HOBIS 컬러 토큰 치환
- **reload_themes()**: 런타임 재스캔 API
- **config.yaml 로딩** (`hodlum.py:28-37`): `--config` 기본값 `"config.yaml"` (CWD 기준 상대경로)
- **폰트 로딩** (`main_window.py:48-66`): `Path(__file__).parent / "fonts"` → tempdir 복사 → QFontDatabase 등록

## 이 세션(P10.9wc)의 작업 요약

P10.4wc의 Windows 배포 파이프라인을 검토한 결과:
- **현재**: `--onedir` 모드 → `dist/Hodlum/` 폴더 (exe + 수백 개 DLL)
- **문제**: 빌드 자체가 Windows에서 Python 설치 → pip 의존성 → PyInstaller 필요. 사용자에게 배포 불가.
- **목표**: 단일 `Hodlum.exe` 파일만으로 Python 없이 바로 실행 가능

## 핵심 결정사항

1. **--onefile 모드 전환**: `hodlum.spec`에서 모든 바이너리/데이터를 exe에 포함. COLLECT 단계 제거.
2. **리포트 테마 이중 탐색**: 내장(exe 번들) + 외부(exe 옆 `themes/` 폴더) 병행. 외부가 우선(동일 stem).
3. **config.yaml 외부화**: exe 옆에 배치. 없으면 하드코딩 기본값 사용.
4. **최종 배포 형태**: `Hodlum.exe` + `themes/` 폴더 + `config.yaml` (3개만 전달하면 실행 가능)

## 총평 — 단일 exe 배포를 위한 아키텍처 분석과 설계

### 이번 세션의 의의

P10.9wc는 코드를 한 줄도 변경하지 않았지만, 6개 파일의 정확한 변경 내용과 이유를 사전 문서화하여 P10.10wc의 구현을 단순 실행 문제로 축소시킨 **설계 전용 세션**이다.

핵심 설계 결정은 **테마 이중 탐색**(내장 _MEIPASS + 외부 exe옆)으로, 배포 편의성과 확장성을 동시에 달성했다. 프로젝트 전체 맥락(P10.1wc~P10.9wc)을 완전히 정리하여 이후 포크 시 이 파일만으로 전체 파악이 가능하다.

### 작업 결과 정량 요약

| 항목 | 수치 |
|------|------|
| 분석 대상 | 6개 파일 |
| 설계 결정 | 4건 (onefile, 테마 이중탐색, config 외부화, 배포 구조) |
| 코드 변경 | 0줄 (설계만) |

### 전략적 제언

**단기**: 설계대로 구현하면 된다(P10.10wc에서 실현됨).

**중기**: "분석 → 설계 → 구현 분리" 패턴은 복잡한 변경의 리스크를 줄이는 워크플로우로 다른 프로젝트에도 적용 가능.

**장기**: onefile 빌드의 시작 시간 최적화(스플래시 스크린 등)를 고려할 수 있다.

## 다음 작업: 수정 대상 파일 및 변경 내용

### 1. `hodlum.spec` — onefile 모드 전환

**현재** (line 94~123):
```python
exe = EXE(
    pyz, a.scripts, [],
    exclude_binaries=True,       # ← onedir용: 바이너리 분리
    name='Hodlum', console=False, ...
)
coll = COLLECT(exe, a.binaries, a.zipfiles, a.datas, ...)  # ← onedir용
```

**변경**:
```python
exe = EXE(
    pyz, a.scripts,
    a.binaries, a.zipfiles, a.datas,  # ← 모두 exe에 포함
    exclude_binaries=False,             # ← onefile
    name='Hodlum', console=False, ...
)
# COLLECT 삭제 (onefile이므로 불필요)
```

- macOS BUNDLE 블록은 유지 (macOS는 여전히 onedir + .app 번들)
- `sys.platform` 분기: Windows만 onefile, macOS는 기존 유지

### 2. `gui/report/generator.py` — 테마 이중 탐색 (line 36~101)

**현재**:
```python
if getattr(_sys, "frozen", False):
    THEMES_DIR = Path(_sys._MEIPASS) / "gui" / "report" / "themes"
else:
    THEMES_DIR = Path(__file__).parent / "themes"
```

**변경 방향**:
```python
def _get_themes_dirs():
    """내장 + 외부 테마 디렉토리 목록 반환."""
    dirs = []
    if getattr(_sys, "frozen", False):
        # 내장 테마 (exe 번들 내부, _MEIPASS 임시 폴더)
        dirs.append(Path(_sys._MEIPASS) / "gui" / "report" / "themes")
        # 외부 테마 (exe 옆 themes/ 폴더 — 사용자가 CSS 추가 가능)
        exe_dir = Path(_sys.executable).parent
        external = exe_dir / "themes"
        if external.is_dir():
            dirs.append(external)
    else:
        dirs.append(Path(__file__).parent / "themes")
    return dirs
```

`_scan_themes()` 수정: 모든 dirs를 순회하며 CSS 수집. 동일 stem이면 외부(나중 dir)가 우선.

### 3. `hodlum.py` — config.yaml 경로 수정 (line 28~37)

**현재**: `Path(args.config)` — CWD 기준 상대경로. exe 더블클릭 시 CWD가 exe 폴더가 아닐 수 있음.

**변경**:
```python
if getattr(sys, "frozen", False):

parser.add_argument("--config", default=config_default, help="Config file path")
```

## 주의사항

- **폰트 로딩** (`main_window.py:48-66`): `Path(__file__).parent / "fonts"` 사용. onefile frozen 모드에서 `__file__`은 `_MEIPASS` 하위를 가리키므로 정상 동작 (폰트는 exe에 내장, 외부화 불필요)
- **WeasyPrint (PDF)**: Windows에서 GTK3 런타임 필요. HTML/MD만 사용 시 불필요. hiddenimport는 유지하되 런타임 에러는 graceful 처리됨
- **USB-to-Serial 드라이버**: Windows에서 별도 설치 필요 (FTDI/CH340). config.yaml의 `serial.port: auto`로 자동 검색 가능
- **--mock 모드**: `Hodlum.exe --mock`으로 하드웨어 없이 GUI 테스트 가능
