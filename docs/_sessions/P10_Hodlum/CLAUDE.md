# Hodlum 프로젝트 지침

## 세션 컨텍스트 포크 시스템

이 프로젝트는 세션 간 컨텍스트를 이어받기 위한 포크 시스템을 사용합니다.

### 세션지침 파일 위치
- **레지스트리**: `memory/sessions/registry.md` — 전체 세션 목록 + 포크 관계
- **개별 세션**: `memory/sessions/P{번호}.md` — 세션별 컨텍스트

### 포크 방법
사용자가 "P10.9wc 포크해줘" 등으로 요청하면:
1. `memory/sessions/registry.md` 읽어서 다음 순번 확인
2. `memory/sessions/P10.9wc.md` (포크원 세션지침) 읽기
3. 새 P번호 부여 (포크원의 "포크 가이드" 섹션에 지시가 있으면 따를 것)
4. `memory/sessions/registry.md`에 신규 항목 추가
5. `memory/sessions/P{새번호}.md` 생성
6. 포크원 컨텍스트 요약을 사용자에게 보고 후 작업 재개

### P번호 체계
- 형식: `P{프로젝트}.{순번}{정책태그}`
- 이 프로젝트: P10 (Hodlum)
- 정책 태그: `w` = Worker LLM 위임, `c` = 컨텍스트 포크 시스템
- 예: P10.10wc = 프로젝트10, 세션10, Worker+Context fork 정책

### 새 세션 시작 시
1. `memory/sessions/registry.md` 확인
2. 다음 번호 부여 + 정책 태그(wc) 포함
3. 첫 응답에 P번호 안내: "이 세션 번호: `[P10.Xwc]`"
4. registry에 신규 항목 추가
5. 세션지침 파일 생성

### 세션지침 표준 구조
```markdown
# P{번호} — {제목}
## 메타데이터
- 생성: YYYY-MM-DD
- 포크원: {부모 P번호}
- 자식: {자식 P번호 목록}
## 작업 요약 (3~5줄)
## 주요 결과물
## 핵심 결정사항
## 다음 작업을 위한 컨텍스트
```
