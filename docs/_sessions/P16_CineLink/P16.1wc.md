## 핵심 결정사항

1. **브랜드 구조**: SL Corporation → SeouLink(대외 배포 브랜드) → CineLink(내부 제작 엔진, 장기적으로 독립 제품화). 크레딧 표기: `Produced with CineLink | SeouLink`.

2. **파이프라인 6 Phase 확정**:
   - Phase 1 분석 (ffprobe + 1fps 프레임 + Whisper 전사 + Claude 비전 태깅)
   - Phase 2 색보정 (LOG→Rec.709 LUT → 톤 매칭 → 크리에이티브 룩, 이 순서 고정)
   - Phase 3 사진 가공 (RAW 현상 → 자동보정 → 스마트 크롭 → Ken Burns)
   - Phase 4 나레이션 (대본 → Typecast TTS, 대안 ElevenLabs/Edge)
   - Phase 5 BGM/SFX (나레이션 0dB 기준, BGM -15~-20dB, 덕킹/페이드)
   - Phase 6 편집·합성 (타임라인 JSON → 트리밍 → xfade → CJK 오버레이 → 3트랙 amix → 워터마크 → 인트로 → 엔딩 → `+faststart` 인코딩)

3. **핵심 4대 원칙**: 분석 우선 / 비파괴 (원본 불변, 중간물 CRF18) / 단계별 검증 (2~3 샘플 선행) / 맥락 기반 자동화.

4. **토큰 전략**:
   - 1080p 프레임 ≈ 1.6K tok, 10분@1fps ≈ 960K tok → **fps는 0.2~0.5 표준**, 구간 지정 필수
   - 분석=Sonnet, 판단/대본=Opus 역할 분리
   - ffmpeg/whisper/ffprobe는 토큰 0 (로컬), 분석 결과 디스크 캐싱

5. **모델 자동 전환 우회로**:
   - 어시스턴트는 세션 내 자가 모델 전환 불가
   - **방법 1**: 사용자가 `/model opusplan` 한 번 입력 → 계획=Opus, 실행=Sonnet 자동 분리
   - **방법 2**: `.claude/agents/` 서브에이전트 frontmatter에 model 지정 → 메인이 자동 위임
   - CineLink용 5종 에이전트 설계 확정: `frame-analyzer`(Sonnet) / `transcript-processor`(Haiku) / `ffmpeg-runner`(Haiku) / `narration-writer`(Opus) / `editor-judge`(Opus)

6. **장기 로드맵**: Claude로 2~3편 실전검증(현재 단계) → Gemma 4 품질 비교 → Tauri/Electron 데스크탑 앱 → 모바일 컴패니언.

## 사용자 피드백 (중요 — 자식 세션이 반드시 반영할 것)

1. **자동화 권한 원칙**: "자동화 때문에 권한을 준 건데 그걸 사용자가 해야 되는 건 이상하다." → 서브에이전트 호출 등 어시스턴트가 스스로 할 수 있는 일은 사용자에게 떠넘기지 말 것. 단 `/model opusplan`처럼 구조적으로 불가능한 것만 사용자에게 안내.

2. **작업 크기 관리**: 서브에이전트에 넘기는 작업이 과대하면 중단될 수 있음. 분할·체크포인트·재시도 대응책을 사전에 마련할 것.

3. **세션 종료 판단**: 세션 내 진행 정체/대화 표시 이상 → 즉시 세션 마무리 후 포크로 재개.

## 총평

**세션의 의의**: CineLink(P16) 프로젝트의 정체성·파이프라인·기술 스택·장기 로드맵을 **문서 기반으로 확정**한 설계 세션. 코드 산출물은 없지만 이후 모든 작업의 레일을 깔았다.

**정량 요약**:
- 신규 프로젝트 등록: 1건 (P16)
- 플랜 확정: 1건 (toasty-wondering-rabbit.md)
- 디렉토리 스캐폴딩: 부분 완료 (brace expansion 오류 잔재는 자식 세션에서 정리)
- 미완 작업: 6건 (자식 세션으로 캐리오버)
- 대화 이슈: 중간 표시 이상 발생 → 세션 조기 종료

**전략적 제언**:
- **단기**: 자식 세션에서 5종 서브에이전트부터 만들어라. 이게 있어야 이후 모든 Phase 작업에서 모델 자동 분리가 작동한다.
- **중기**: 첫 실제 영상 1편을 **짧게** (2~3분) 잡아 전체 파이프라인을 end-to-end로 돌려라. 긴 영상부터 시작하면 토큰 한도에 걸린다.
- **장기**: Gemma 4 온디바이스 전환은 Claude로 품질 기준선을 확보한 후에. 조급하게 로컬 모델부터 시도하지 말 것.

**세션 운영 교훈**:
- 사용자가 "자동화 권한" 원칙을 강조함 → 앞으로 "X를 해주세요"로 사용자에게 작업을 넘기는 응답 패턴을 지양하고, 가능한 모든 것은 직접 실행.
- Plan 작성 후 사용자 승인을 받았음에도 실행 단계에서 세션 표시 이상이 발생 → **다음 세션에서는 승인 직후 실행 단계를 잘게 쪼개 조기 체크포인트 확보**.
- 대형 문서(23K+ tok)는 메인 컨텍스트에서 직접 읽지 말고 서브에이전트 요약으로 우회 — 이번 세션에서 잘 작동함.
