## 2. 용어 점검 결과

| 사용자 표현 | 업계 표준 | 판정 | 비고 |
|-------------|----------|------|------|
| **컬러그레이드** | **컬러 그레이딩 (color grading)** | ⚠️ 수정 권장 | 명사=그레이딩, 동사=grade. 1차 보정(color correction)과 크리에이티브 톤(color grading) 구분. |
| **LUT** | **LUT (Look-Up Table)** | ✅ 정확 | 일반적으로 **3D LUT** (`.cube`, `.3dl`). 1차 보정 LUT(LOG→Rec.709)와 크리에이티브 LUT 2종. |
| **LUT 적용 썸네일** | **그레이딩 프리뷰 (grading preview)** 또는 **LUT 프리뷰 그리드** | ⚠️ 수정 권장 | "썸네일"은 일반 축소 이미지. N개 LUT 후보를 작게 미리 보여주는 건 "LUT 프리뷰 그리드"(FCP/Resolve는 Still Gallery/Preview Strip). |

**추가 표준어** (본 기획안·CineLink 문서에서 일관 사용):
- **프록시(proxy)**: 원본 4K → 1080p/720p 편집용 저해상도 복사본
- **씬 디텍션(scene detection)**: 컷 전환점 자동 감지 (PySceneDetect 등)
- **J-cut / L-cut**: 오디오·비디오 경계를 어긋나게 배치하는 대화 편집 기법
- **LUFS (Loudness Units Full Scale)**: 라우드니스 표준 (YouTube -14, 방송 -23)
- **EDL / OTIO / FCPXML**: NLE 간 타임라인 교환 포맷
- **B-roll**: 메인 외 보조 영상 (컷어웨이용)

## 5. 3대 편집 장애 해결 — 상세 설계

### 장애 1: 영상 훑어보기 귀찮음 → **VideoIndex 파이프라인**

영상을 "읽을 수 있는 텍스트"로 전환. MD 파일 훑어보기만으로 원하는 구간 식별.

#### 1-1. 음성 정보 텍스트화
- **도구**: faster-whisper large-v3 + pyannote.audio (화자 분리) + YAMNet (521 AudioSet 클래스 배경음 분류)
- **산출물**: `{episode}/index/transcript.json` + `transcript.srt`
- **핵심**: 대사는 화자 ID로, 배경음(경적/파도/바람/박수)은 이벤트로 분리

#### 1-2. 영상 정보 텍스트화 (HighLink 통합)
- **부하 감소**: 1 fps 샘플링, 720p 다운스케일, 배치 8~16
- **도구**:
  1. YOLO11 + ByteTrack ← HighLink 포팅
  2. OCR (PaddleOCR/EasyOCR) — 간판·표지판·자막
  3. Places365 (ResNet50) 또는 Qwen3-VL — 장소 분류 365종
  4. SlowFast 또는 Qwen3-VL — 행동 분석
  5. Qwen3.5-122B — 1문단 상황 요약
- **산출물**: `{episode}/index/vision.json`

#### 1-3. 텍스트 컨텍스트화 (컷 포인트 결정)
컷 포인트 = **4대 신호 가중 결합**:
1. PySceneDetect (ContentDetector) — 화면 전환
2. silencedetect — 장기 무음 (CL-TRAP-010 재활용)
3. 화자 변경 (pyannote)
4. 장소 변경 (Places365 연속 프레임 비교)

**클립 생성**: FFmpeg stream copy (재인코딩 0)
**파일명**: `{YYYYMMDD_HHMMSS_SS}_{키워드1}_{키워드2}_{키워드3}.mp4`
예: `20260423_172112_15_카페_대화_커피.mp4`

#### 1-4. 컨텍스트 전달
- **클립별 MD**: `{clipname}.md` (동일 이름 페어) — 타임코드·객체·대사·배경음·요약
- **에피소드 INDEX.md**: 전체 클립 목록 + 인물·장소 집계 + 키워드
- **웹 뷰어**: HighLink `index.html` 포팅한 `viewer.html.j2` — 클립 + 영상 동기화

#### ⚠️ 업계 관행 보완 (사용자가 놓친 부분)
1. **프록시 워크플로우**: 4K 원본은 편집 불편 → 1080p proxy 자동 생성 (H.264 CRF23)
2. **씬 디텍션 보조**: PySceneDetect는 YOLO 없이도 cheap한 1차 필터
3. **멀티캠 동기화**: 2대+ 카메라 시 오디오 웨이브폼 매칭 또는 LTC (Phase C)
4. **메타데이터 표준화**: exiftool로 XMP/EXIF 촬영 시각 추출 (파일명 타임스탬프 신뢰도)
5. **B-roll 자동 구분**: 대사 없는 장면 = B-roll 후보 태깅

### 장애 2: 편집 스킬 부족 → **SoloMode 편집 도구 모음**

파이널 컷 GUI 조작을 Claude 명령 + CineLink 함수로 대체.

#### 2-1. 컷 편집 → 텍스트 기반 선택
- VideoIndex에서 클립 선택 → `ffmpeg_lib.concat_clips([ids])`
- 자연어 쿼리: "15초 릴스, 카페 장면 2개 + 커피 근접 + 웃음 반응" → Skill 매칭 → 타임라인
- **신규 Skill**: `cut-assembler` (`channels/_shared/skills/`)

#### 2-2. 컬러 그레이딩
- **분석**:
  1. LOG 감지 (히스토그램 + 카메라 메타데이터: C-Log/S-Log/V-Log)
  2. 색감 프로파일: RGB 히스토그램·평균 색온도(K)·saturation·contrast → JSON
  3. 클립 간 편차 수치화
- **매칭 모드 2종**:
  - (a) **톤 매칭**: 기준 클립에 다른 클립 색감 맞춤 (auto-color-match)
  - (b) **크리에이티브 LUT**: 인기 LUT 라이브러리 추천 (orange-teal, film emulation)
- **그레이딩 프리뷰 그리드**: N개 LUT 후보를 대표 프레임에 적용 → 3×3 or 4×4 썸네일 그리드 → 클릭 선택 → 전체 적용
- **HighLink 통합**: WeatherAdapter 기법으로 저조도/고조도 자동 보정
- **신규 파일**: `cinelink/shared/scripts/color_lib.py` + `shared/luts/` LUT 팩

#### 2-3. 자막 (선택적 삽입)
- 대사 (화자 구분, 색상 차별)
- 배경음 캡션 (`[파도 소리]`, `[자동차 경적]`)
- 장소 태그 (`[카페 - 스타벅스 강남점]`)
- 번역 자막 (Whisper translate + Claude 문화 조정)
- silencedetect auto-snap (CL-TRAP-010) 필수 적용

#### 2-4. 효과음·BGM·TTS
- **추천**:
  1. VideoIndex 컨텍스트 → 키워드 추출
  2. 무료: Freesound.org, Pixabay, YouTube Audio Library, Zapsplat
  3. 유료: Artlist, Epidemic Sound, Musicbed
  4. 미리듣기 URL 리스트
- **로그**: `LICENSE_LOG.md`에 트랙/출처/라이선스 자동 기록
- **TTS**:
  - 무료: Edge TTS (기본), gTTS
  - 유료: Typecast (사용자 로그인 필요), ElevenLabs
  - 캐릭터 추천: 대사 톤·성별·연령 → LLM → Typecast 캐릭터 ID 매칭
- **신규 Command**: `/project:find-audio [무드]`

#### ⚠️ 업계 관행 보완
1. **오디오 라우드니스 정규화**: YouTube 업로드 전 -14 LUFS (ffmpeg `loudnorm` 2-pass), 방송 -23 LUFS
2. **피크 리미터**: True Peak -1.0 dBTP 이하 (클리핑 방지)
3. **J-cut / L-cut**: 대화 편집 자연스러움 (Phase D 자동화 후보)
4. **색공간 관리**: 원본 Rec.709 vs HDR(HLG/PQ) 구분 (ACES는 과투자)
5. **안전 영역 (title safe)**: 자막은 화면 90% 이내
6. **프로젝트 버전 관리**: EDL/FCPXML/OTIO 내보내기 필수
7. **백업 3-2-1 규칙**: 3 카피 / 2 저장소 / 1 원격

  - match_reference: c005
  ## 자막

  - bg_events: selective (파도, 경적만)

  - narration: typecast voice_id=nari_calm

- **실행 엔진**: MD 파서 → `ffmpeg_lib` 함수 시퀀스 → 최종 mp4
